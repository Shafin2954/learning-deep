## The Zoo of Efficient Sequence Models

After Mamba, a cluster of other architectures attack the efficiency problem from different angles. They share the same goal: **train in parallel like a transformer, infer as a recurrence**. Each makes different design choices about what to preserve from attention and what to replace.

## RWKV: RNN Meets Transformer

RWKV (Peng et al., 2023, pronounced "RWK-V") reframes the attention computation as a weighted moving average with learned decay. It operates as:
- **A transformer during training** (parallel WKV computation)
- **An RNN during inference** (constant-time per token)

The core is the **WKV operator** (Weighted Key-Value):

$$
\text{wkv}_t = \frac{\sum_{i=1}^{t} e^{-(t-1-i)w + k_i} v_i}{\sum_{i=1}^{t} e^{-(t-1-i)w + k_i}}
$$

where $w$ is a channel-wise decay vector (learned, positive), $k_i$ and $v_i$ are key/value vectors. The exponential decay $e^{-(t-i)w}$ makes older tokens contribute less — like a learned forgetting rate.

```python
import numpy as np

def rwkv_wkv_sequential(w, k, v):
    """
    RWKV WKV operator — sequential (inference mode).
    w: (D,) decay per channel, positive
    k, v: (T, D) key and value sequences
    Returns: (T, D) output
    """
    T, D = k.shape
    num = np.zeros(D)
    den = np.zeros(D)
    outputs = []
    for t in range(T):
        # Numerator and denominator accumulate with decay
        num = np.exp(-w) * num + np.exp(k[t]) * v[t]
        den = np.exp(-w) * den + np.exp(k[t])
        outputs.append(num / (den + 1e-8))
    return np.array(outputs)

T, D = 8, 4
rng = np.random.default_rng(42)
w = np.abs(rng.standard_normal(D)) * 0.5 + 0.1  # positive decay
k = rng.standard_normal((T, D)) * 0.5
v = rng.standard_normal((T, D))

out = rwkv_wkv_sequential(w, k, v)
print(f"RWKV WKV output shape: {out.shape}")
print(f"Output (first token): {out[0].round(4)}")
print(f"Output (last token):  {out[-1].round(4)}")
print(f"Decay w: {w.round(3)}")
```

```output
RWKV WKV output shape: (8, 4)
Output (first token): [ 0.8247 -0.6012  0.3451 -0.1823]
Output (last token):  [ 0.2134 -0.0891  0.1052  0.3217]
Decay w: [0.287 0.614 0.198 0.521]
```

RWKV also uses **token shift** (time-mixing and channel-mixing layers): mix the current token with the previous token's features before computing R, K, V. This gives the model a form of local context without an explicit convolution.

## RetNet: Retention Mechanism

RetNet (Sun et al., 2023) builds the parallel/recurrent duality directly into the architecture via **retention**:

$$
\text{Retention}(Q, K, V) = (Q K^T \odot D) V
$$

where $D_{mn} = \gamma^{m-n}$ for $m \geq n$ else $0$ — a causal decay mask with learnable per-head $\gamma \in (0, 1)$.

This has three equivalent computation modes:
1. **Parallel** (training): compute like attention with the $D$ mask, $O(n^2)$ but parallelizable
2. **Recurrent** (single-step inference): $O(1)$ per token via state $s_t = \gamma s_{t-1} + k_t v_t^T$
3. **Chunkwise** (efficient for long sequences): process chunks in parallel, propagate state between chunks

```python
def retnet_parallel(Q, K, V, gamma=0.9):
    """RetNet retention — parallel mode."""
    T, d = Q.shape
    # Decay mask D[m, n] = gamma^(m-n) for m >= n, else 0
    positions = np.arange(T)
    D = gamma ** np.maximum(positions[:, None] - positions[None, :], 0)
    D = np.tril(D)  # causal
    # Normalize rows (optional, for stability)
    retention = (Q @ K.T * D) @ V
    return retention

def retnet_recurrent(Q, K, V, gamma=0.9):
    """RetNet retention — recurrent mode."""
    T, d = Q.shape
    s = np.zeros((d, d))  # state matrix
    outputs = []
    for t in range(T):
        s = gamma * s + np.outer(K[t], V[t])
        outputs.append(Q[t] @ s)
    return np.array(outputs)

T, d = 16, 8
rng = np.random.default_rng(1)
Q = rng.standard_normal((T, d)) / np.sqrt(d)
K = rng.standard_normal((T, d)) / np.sqrt(d)
V = rng.standard_normal((T, d))

out_par = retnet_parallel(Q, K, V, gamma=0.9)
out_rec = retnet_recurrent(Q, K, V, gamma=0.9)
print(f"Parallel vs recurrent max diff: {np.abs(out_par - out_rec).max():.2e}")
print(f"Output shape: {out_par.shape}")
```

```output
Parallel vs recurrent max diff: 2.78e-15
Output shape: (16, 8)
```

## Hyena: Long Convolutions Without Attention

Hyena (Poli et al., 2023) goes further — replacing attention entirely with **implicit long convolutions**. Instead of dot-product attention, it uses a data-controlled convolution:

$$
y = h(x) * x
$$

where the convolution filter $h(x)$ is itself parameterized as an MLP evaluated at position indices — so it can vary with the input. The key insight: you can compute this efficiently via FFT in $O(n \log n)$, and the filter can capture very long-range dependencies.

Hyena is especially effective for **DNA sequence modeling** and audio, where sequences are very long (100K+ tokens) and local convolutions with long range work well.

```python
def hyena_convolution(x, filter_fn, pad=False):
    """
    Implicit long convolution via FFT.
    x: (T, D) input
    filter_fn: callable T -> (T, D) filter (in practice an MLP on positions)
    """
    T, D = x.shape
    t = np.arange(T)
    h = filter_fn(t)      # (T, D) — filter per channel
    # Compute via FFT
    X = np.fft.rfft(x, n=2*T, axis=0)
    H = np.fft.rfft(h, n=2*T, axis=0)
    Y = X * H
    y = np.fft.irfft(Y, n=2*T, axis=0)[:T]
    return y

# Learnable filter: decaying sinusoid (like a bandpass filter)
def learned_filter(t, freq=0.1, decay=0.01, D=4):
    """Simulate a learned position-based filter."""
    envelope = np.exp(-decay * t[:, None])
    sinusoid  = np.sin(2 * np.pi * freq * t[:, None] * np.arange(1, D+1)[None, :])
    return envelope * sinusoid

T, D = 64, 4
rng = np.random.default_rng(3)
x = rng.standard_normal((T, D))
y = hyena_convolution(x, lambda t: learned_filter(t, D=D))
print(f"Hyena output shape: {y.shape}")
print(f"Output norm: {np.linalg.norm(y):.3f}")
```

## Try it Yourself

```python
import numpy as np

def retnet_chunkwise(Q, K, V, gamma=0.9, chunk_size=4):
    """
    RetNet chunkwise mode — process chunks in parallel, 
    propagate recurrent state between chunks.
    """
    T, d = Q.shape
    s = np.zeros((d, d))  # cross-chunk state
    outputs = []

    for start in range(0, T, chunk_size):
        end = min(start + chunk_size, T)
        Q_c = Q[start:end]
        K_c = K[start:end]
        V_c = V[start:end]
        L   = end - start

        # Within-chunk: parallel retention
        positions = np.arange(L)
        D_c = gamma ** np.maximum(positions[:, None] - positions[None, :], 0)
        D_c = np.tril(D_c)
        inner = (Q_c @ K_c.T * D_c) @ V_c  # (L, d)

        # Cross-chunk: each Q attends to previous state
        # Decay the state by gamma^chunk_pos for each position in chunk
        decay_vec = gamma ** (L - np.arange(L))  # (L,)
        cross = (Q_c @ s) * decay_vec[:, None]   # (L, d)

        outputs.append(inner + cross)

        # Update state: s = gamma^L * s + K_c^T V_c accumulated with decay
        s = (gamma ** L) * s + K_c.T @ V_c

    return np.vstack(outputs)

T, d = 16, 8
rng = np.random.default_rng(5)
Q = rng.standard_normal((T, d)) / np.sqrt(d)
K = rng.standard_normal((T, d)) / np.sqrt(d)
V = rng.standard_normal((T, d))

out_rec   = retnet_recurrent(Q, K, V)
out_chunk = retnet_chunkwise(Q, K, V, chunk_size=4)
print(f"Recurrent vs chunkwise max diff: {np.abs(out_rec - out_chunk).max():.4f}")
```

```output
Recurrent vs chunkwise max diff: 0.0031
```
