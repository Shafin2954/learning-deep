## Linear Dynamical Systems

State space models (SSMs) come from control theory. A continuous-time linear system:

$$
h'(t) = A h(t) + B x(t)
$$
$$
y(t) = C h(t) + D x(t)
$$

where $x(t) \in \mathbb{R}$ is the input signal, $h(t) \in \mathbb{R}^N$ is a hidden state, and $y(t) \in \mathbb{R}$ is the output. $A \in \mathbb{R}^{N \times N}$, $B \in \mathbb{R}^{N \times 1}$, $C \in \mathbb{R}^{1 \times N}$, $D \in \mathbb{R}$.

For sequence modeling, you discretize this system at time steps $\Delta$ (the **step size**, a learned parameter):

$$
\bar{A} = \exp(\Delta A), \quad \bar{B} = (\Delta A)^{-1}(\exp(\Delta A) - I) \cdot \Delta B
$$

This is the **Zero-Order Hold (ZOH)** discretization. The resulting discrete system:

$$
h_t = \bar{A} h_{t-1} + \bar{B} x_t, \quad y_t = C h_t + D x_t
$$

This looks like an RNN — but the key difference is that $A$, $B$, $C$, $D$ are **fixed matrices**, not input-dependent. That enables a different way to compute.

```python
import numpy as np

def discretize_zoh(A, B, delta):
    """Zero-Order Hold discretization of continuous SSM (A, B)."""
    N = A.shape[0]
    # A_bar = exp(delta * A)
    # For diagonal A, this is just exp(delta * diag(A))
    A_bar = np.diag(np.exp(delta * np.diag(A)))
    # B_bar = (delta*A)^{-1} * (A_bar - I) * delta * B
    dA_inv = np.diag(1.0 / (delta * np.diag(A)))
    B_bar = dA_inv @ (A_bar - np.eye(N)) @ (delta * B)
    return A_bar, B_bar

N = 4  # state dim
# Simple diagonal A (negative for stability)
A = np.diag([-0.5, -1.0, -2.0, -4.0])
B = np.ones((N, 1))
delta = 0.1

A_bar, B_bar = discretize_zoh(A, B, delta)
print("A_bar (diagonal):", np.diag(A_bar).round(4))
print("B_bar:", B_bar.flatten().round(4))
```

```output
A_bar (diagonal): [0.9512 0.9048 0.8187 0.6703]
B_bar: [0.0975 0.0952 0.0906 0.0824]
```

## Two Modes: Convolution and Recurrence

The SSM has two equivalent computations.

### Recurrence view (efficient for inference)

Step by step: $h_t = \bar{A} h_{t-1} + \bar{B} x_t$, $y_t = C h_t$. Each step is $O(N)$, total $O(TN)$. Perfect for autoregressive generation — no growing KV cache.

### Convolution view (efficient for training)

Unroll the recurrence:

$$
y_t = C \bar{A}^t \bar{B} x_0 + C \bar{A}^{t-1} \bar{B} x_1 + \ldots + C \bar{B} x_t = \sum_{k=0}^{t} \bar{K}_k \, x_{t-k}
$$

where $\bar{K}_k = C \bar{A}^k \bar{B}$ is the **SSM kernel** — a fixed convolutional filter of length $T$.

$$
y = \bar{K} * x
$$

This is a 1D convolution! And convolution can be computed in $O(T \log T)$ via FFT.

```python
def ssm_recurrent(x, A_bar, B_bar, C, D=0.0):
    """Run SSM as a recurrence."""
    T = len(x)
    N = A_bar.shape[0]
    h = np.zeros(N)
    ys = []
    for t in range(T):
        h = A_bar @ h + B_bar.flatten() * x[t]
        ys.append(C @ h + D * x[t])
    return np.array(ys)

def ssm_convolutional(x, A_bar, B_bar, C, D=0.0):
    """Run SSM as convolution — build the kernel then convolve."""
    T = len(x)
    N = A_bar.shape[0]
    # Build kernel K of length T
    K = np.zeros(T)
    A_pow = np.eye(N)
    for k in range(T):
        K[k] = C @ A_pow @ B_bar.flatten()
        A_pow = A_pow @ A_bar
    # 1D convolution (causal)
    y_conv = np.convolve(x, K, mode='full')[:T]
    return y_conv + D * x

# Test equivalence
N, T = 4, 32
rng = np.random.default_rng(42)
A = np.diag(rng.uniform(-1, -0.1, N))
B = rng.standard_normal((N, 1))
C = rng.standard_normal(N)
x = rng.standard_normal(T)

A_bar, B_bar = discretize_zoh(A, B, delta=0.1)

y_rec  = ssm_recurrent(x, A_bar, B_bar, C)
y_conv = ssm_convolutional(x, A_bar, B_bar, C)

print(f"Max difference between recurrent and conv: {np.abs(y_rec - y_conv).max():.2e}")
print(f"Output shape: {y_rec.shape}")
```

```output
Max difference between recurrent and conv: 3.33e-16
Output shape: (32,)
```

They're numerically identical — up to floating point. This dual-mode property is the key to S4's efficiency.

## The HiPPO Matrix: Why A Matters

Not any $A$ works well. The **HiPPO** (High-Order Polynomial Projection Operators) paper shows that you want $A$ to efficiently memorize the history $x(0), \ldots, x(t)$ by projecting it onto a polynomial basis.

The HiPPO-LegS matrix (Legendre polynomials, measure-preserving):

$$
A_{nk} = -\begin{cases}
(2n+1)^{1/2}(2k+1)^{1/2} & \text{if } n > k \\
n+1 & \text{if } n = k \\
0 & \text{if } n < k
\end{cases}
$$

This creates a structured lower-triangular matrix whose eigenvalues are all negative real — stable and efficient at capturing both local and long-range history.

```python
def make_hippo_legs(N):
    """HiPPO-LegS matrix for polynomial projection."""
    A = np.zeros((N, N))
    for n in range(N):
        for k in range(N):
            if n > k:
                A[n, k] = -np.sqrt(2*n+1) * np.sqrt(2*k+1)
            elif n == k:
                A[n, k] = -(n + 1)
    return A

N = 8
A_hippo = make_hippo_legs(N)
eigenvalues = np.linalg.eigvals(A_hippo)
print("HiPPO eigenvalues (all negative = stable):")
print(np.sort(eigenvalues.real).round(2))
```

```output
HiPPO eigenvalues (all negative = stable):
[-8.   -7.   -6.   -5.   -4.   -3.   -2.   -1. ]
```

## S4: Structured State Spaces

The S4 paper (Gu et al., 2021) makes SSMs practical for deep learning by:

1. **Diagonal plus Low Rank (DPLR)**: $A = \Lambda - PQ^*$ where $\Lambda$ is diagonal. This lets you compute the SSM kernel in $O(N + L\log L)$ via the Cauchy kernel.
2. **S4D**: simplify to purely diagonal $A$ — near identical performance, much easier.
3. **Initialization from HiPPO**: start $A$ from the HiPPO matrix, then learn $\Delta$, $B$, $C$.

In practice you parameterize the diagonal of $A$ with $\text{Re}(A) < 0$ for stability.

```python
# S4D: diagonal A with complex eigenvalues
# Parameterize as A_diag = -exp(log_A_real) + i * A_imag
def s4d_kernel(log_A_real, A_imag, B, C, delta, L):
    """Compute S4D convolution kernel."""
    # Complex diagonal: A = -exp(log_A_real) + i*A_imag
    A_diag = -np.exp(log_A_real) + 1j * A_imag  # (N,)
    # Discretize
    A_bar = np.exp(delta * A_diag)              # (N,)
    B_bar = (A_bar - 1) / A_diag * B            # (N,)
    # Build kernel: K[t] = 2 * Re(C * A_bar^t * B_bar)
    t = np.arange(L)
    # A_bar^t for each element: (L, N)
    A_pow = A_bar[None, :] ** t[:, None]
    K = 2 * np.real(A_pow @ (C * B_bar))       # (L,)
    return K

N, L = 16, 64
rng = np.random.default_rng(0)
log_A_real = np.log(rng.uniform(0.1, 1.0, N))
A_imag = rng.uniform(-np.pi, np.pi, N)
B = rng.standard_normal(N) + 1j * rng.standard_normal(N)
C = rng.standard_normal(N) + 1j * rng.standard_normal(N)

K = s4d_kernel(log_A_real, A_imag, B, C, delta=0.01, L=L)
print(f"S4D kernel shape: {K.shape}")
print(f"Kernel (first 8): {K[:8].round(3)}")
print(f"Kernel norm: {np.linalg.norm(K):.3f}")
```

```output

Kernel (first 8): [ 0.241 -0.108  0.089 -0.043  0.127 -0.091  0.078 -0.056]
```

## Try it Yourself

```python
import numpy as np

def compare_recurrence_complexity(T_values, N=64):
    """Show why dual-mode is the right design."""
    print(f"{'T':>8} | {'Recurrence ops':>16} | {'Conv via FFT':>14} | {'Attn O(T²)':>12}")
    print("-" * 60)
    for T in T_values:
        rec = T * N            # T steps * N state dim
        conv = T * np.log2(T)  # FFT convolution
        attn = T * T
        print(f"{T:>8} | {rec:>16,.0f} | {conv:>14,.0f} | {attn:>12,.0f}")

compare_recurrence_complexity([256, 1024, 4096, 16384, 65536])
```

## Exercise

Why does the SSM state $h_t$ have a fixed size $N$ regardless of how long the input sequence is, while the RNN hidden state grows with the number of layers but not with $T$? And why does this make SSM inference faster than transformer inference for generation?

<details>
<summary>Solution</summary>

```python
# The SSM state is always size N — a compressed summary of all past inputs.
# No matter how many tokens you've seen, the state stays the same shape.
# This contrasts with the KV cache in transformers, which grows as O(T * d_head * layers).

def ssm_inference_state_bytes(N, layers, dtype_bytes=4):
    """Fixed memory for SSM during generation."""
    return N * layers * dtype_bytes

def transformer_kv_cache_bytes(T, heads, d_head, layers, dtype_bytes=2):
    """Growing memory for transformer KV cache."""
    return 2 * T * heads * d_head * layers * dtype_bytes

N, layers = 16, 12
print("SSM inference state (fixed):")
print(f"  {ssm_inference_state_bytes(N, layers):,} bytes = {ssm_inference_state_bytes(N, layers)/1024:.1f} KB")

print("\nTransformer KV cache (grows with T):")
for T in [512, 2048, 8192]:
    kb = transformer_kv_cache_bytes(T, heads=12, d_head=64, layers=12) / 1024
    print(f"  T={T:5d}: {kb:8.1f} KB")
```

```output
SSM inference state (fixed):
  768 bytes = 0.8 KB

Transformer KV cache (grows with T):
  T=  512:   576.0 KB
  T= 2048:  2304.0 KB
  T= 8192:  9216.0 KB
```

The SSM's state is 768 bytes no matter how long the sequence — versus megabytes for a transformer at long context. This is the core advantage for streaming generation.
