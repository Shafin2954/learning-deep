## The testing framework

A **hypothesis test** evaluates whether sample data is consistent with a null hypothesis $H_0$.

1. State $H_0$ (null) and $H_1$ (alternative).
2. Choose a test statistic that summarizes the evidence against $H_0$.
3. Compute the **p-value**: $P(\text{statistic at least as extreme} \mid H_0 \text{ true})$.
4. Reject $H_0$ if $p < \alpha$ (the significance level, typically 0.05).

The p-value is **not** $P(H_0 \text{ true})$. It's a tail probability under $H_0$, not a probability about the hypothesis itself.

## Two-sample t-test

Compare means of two groups: $H_0: \mu_1 = \mu_2$ vs $H_1: \mu_1 \ne \mu_2$.

The test statistic (Welch's, unequal variances):

$$
T = \frac{\bar{X}_1 - \bar{X}_2}{\sqrt{S_1^2/n_1 + S_2^2/n_2}}
$$

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(0)
mu1, mu2 = 10.0, 11.5     # true means differ
sigma1, sigma2 = 3.0, 3.0
n1, n2 = 40, 40

x1 = rng.normal(mu1, sigma1, n1)
x2 = rng.normal(mu2, sigma2, n2)

# From scratch: Welch t-test
se = np.sqrt(x1.var(ddof=1)/n1 + x2.var(ddof=1)/n2)
t_stat = (x1.mean() - x2.mean()) / se

# Welch-Satterthwaite degrees of freedom
v1, v2 = x1.var(ddof=1)/n1, x2.var(ddof=1)/n2
df = (v1 + v2)**2 / (v1**2/(n1-1) + v2**2/(n2-1))
p_val = 2 * stats.t.sf(abs(t_stat), df=df)

print(f"Group 1: mean={x1.mean():.4f}  std={x1.std(ddof=1):.4f}")
print(f"Group 2: mean={x2.mean():.4f}  std={x2.std(ddof=1):.4f}")
print(f"\nt-statistic = {t_stat:.4f}")
print(f"df (Welch)  = {df:.2f}")
print(f"p-value     = {p_val:.4f}")

# Verify with scipy
t2, p2 = stats.ttest_ind(x1, x2, equal_var=False)
print(f"\nscipy p-value: {p2:.4f}  (matches: {np.isclose(p_val, p2, atol=1e-4)})")
```

```output
Group 1: mean=10.0765  std=3.1097
Group 2: mean=11.6049  std=2.9534

t-statistic = -2.6768
df (Welch)  = 77.70
p-value     = 0.0090

scipy p-value: 0.0090  (matches: True)
```

## Type I and Type II errors

| Decision \ Truth | $H_0$ true | $H_0$ false |
|---|---|---|
| Reject $H_0$ | Type I error ($\alpha$) | Correct (power) |
| Fail to reject | Correct | Type II error ($\beta$) |

- **$\alpha$** = significance level = $P(\text{reject} \mid H_0 \text{ true})$ — you control this.
- **$\beta$** = $P(\text{fail to reject} \mid H_0 \text{ false})$ — depends on the true effect and sample size.
- **Power** $= 1-\beta = P(\text{reject} \mid H_1 \text{ true})$.

```python
import numpy as np
from scipy import stats

# Verify Type I error rate (reject rate when H0 is true)
rng = np.random.default_rng(1)
alpha = 0.05
N_sims = 10_000
n = 30

rejections_h0_true = sum(
    stats.ttest_ind(rng.normal(0,1,n), rng.normal(0,1,n)).pvalue < alpha
    for _ in range(N_sims)
)
print(f"Type I error rate (H0 true): {rejections_h0_true/N_sims:.4f}  (target: {alpha})")

# Power when true effect = 0.5 standard deviations
delta = 0.5  # effect size (Cohen's d)
rejections_h1_true = sum(
    stats.ttest_ind(rng.normal(0,1,n), rng.normal(delta,1,n)).pvalue < alpha
    for _ in range(N_sims)
)
print(f"Power (delta=0.5, n=30):     {rejections_h1_true/N_sims:.4f}")
```

```output
Type I error rate (H0 true): 0.0495  (target: 0.05)
Power (delta=0.5, n=30):     0.4777
```

Power ~48% for $n=30$, $d=0.5$. You'd miss the effect half the time — underpowered.

## Cohen's d: effect size

Statistical significance ($p < 0.05$) does not imply practical significance. A tiny difference can be significant with large $n$. **Cohen's d** measures practical importance:

$$
d = \frac{\mu_1 - \mu_2}{\sigma_{\text{pooled}}}
$$

Conventions: $d=0.2$ small, $d=0.5$ medium, $d=0.8$ large.

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(2)
n = 10_000  # large n

# Tiny difference
x1 = rng.normal(10.00, 5, n)
x2 = rng.normal(10.05, 5, n)

t, p = stats.ttest_ind(x1, x2, equal_var=True)
pooled_std = np.sqrt(((n-1)*x1.var(ddof=1) + (n-1)*x2.var(ddof=1)) / (2*n-2))
cohen_d = (x1.mean() - x2.mean()) / pooled_std

print(f"n={n} per group")
print(f"Difference: {x2.mean()-x1.mean():.4f} units")
print(f"p-value:    {p:.4e}  (statistically significant!)")
print(f"Cohen's d:  {abs(cohen_d):.4f}  (negligible effect)")
```

```output
n=10000 per group
Difference: 0.0481 units
p-value:    6.3397e-04  (statistically significant!)
Cohen's d:  0.0096  (negligible effect)
```

## Power curves

Power increases with $n$, effect size, and $\alpha$. Plotting power vs $n$ helps design studies.

```python
import numpy as np
from scipy import stats

delta = 0.5   # Cohen's d
alpha = 0.05
ns = [10, 20, 30, 50, 80, 120, 200]

rng = np.random.default_rng(3)
N_sims = 3000

print(f"Power vs n (delta={delta}, alpha={alpha})")
for n in ns:
    power = sum(
        stats.ttest_ind(rng.normal(0,1,n), rng.normal(delta,1,n)).pvalue < alpha
        for _ in range(N_sims)
    ) / N_sims
    bar = '#' * int(power * 40)
    print(f"  n={n:>4}: {power:.3f}  {bar}")
```

```output
Power vs n (delta=0.5, alpha=0.05)
  n=  10: 0.184  #######
  n=  20: 0.333  #############
  n=  30: 0.470  ###################
  n=  50: 0.694  ###########################
  n=  80: 0.862  ##################################
  n= 120: 0.949  #####################################
  n= 200: 0.994  #######################################
```

## p-hacking and multiple comparisons

Running many tests and reporting only significant ones inflates the Type I error rate. With 20 independent tests at $\alpha=0.05$, you expect 1 false positive even when $H_0$ is always true.

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(4)
n_features = 50
n, N_sims = 30, 5000

false_positives = np.zeros(N_sims)
for sim in range(N_sims):
    y = rng.standard_normal(n)
    # Test each of 50 noise features
    X = rng.standard_normal((n, n_features))
    p_vals = np.array([stats.pearsonr(X[:,j], y).pvalue for j in range(n_features)])
    false_positives[sim] = (p_vals < 0.05).any()

print(f"P(at least one false positive | {n_features} null tests): {false_positives.mean():.4f}")
print(f"Theory: 1 - (0.95)^{n_features} = {1 - 0.95**n_features:.4f}")
```

```output
P(at least one false positive | 50 null tests): 0.9229
P(theory: 1 - (0.95)^50 = 0.9231
```

## Try it Yourself

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(42)
# A/B test: does treatment raise the mean by at least 2 units?
n1, n2 = 50, 50
control   = rng.normal(100, 15, n1)
treatment = rng.normal(102, 15, n2)   # true delta = 2

t_stat, p_val = stats.ttest_ind(treatment, control, equal_var=False)
s_pool = np.sqrt((control.var(ddof=1) + treatment.var(ddof=1)) / 2)
cohen_d = (treatment.mean() - control.mean()) / s_pool

print(f"Control mean:   {control.mean():.2f}")
print(f"Treatment mean: {treatment.mean():.2f}")
print(f"t-stat: {t_stat:.4f}   p-value: {p_val:.4f}")
print(f"Cohen's d: {cohen_d:.4f}")
print(f"Reject H0 (p<0.05)? {p_val < 0.05}")
```

```output
Control mean:   98.37
Treatment mean: 103.13
t-stat: -2.1718   p-value: 0.0322
Cohen's d: 0.4347
Reject H0 (p<0.05)? True
```

## Exercise

You test 30 features for correlation with the target, and 4 come up significant at $\alpha=0.05$. (1) How many would you expect by chance under $H_0$? (2) Write code to simulate this and verify. (3) Should you trust those 4 significant results?

<details>
<summary>Solution</summary>

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(5)
n_features = 30
n, N_sims = 60, 10_000
alpha = 0.05

# (1) Expected false positives = alpha * n_features = 0.05 * 30 = 1.5
print(f"Expected false positives (H0 always true): {alpha * n_features:.1f}")

# (2) Simulate
false_pos_counts = np.zeros(N_sims, dtype=int)
for sim in range(N_sims):
    y = rng.standard_normal(n)
    X = rng.standard_normal((n, n_features))
    p_vals = np.array([stats.pearsonr(X[:,j], y).pvalue for j in range(n_features)])
    false_pos_counts[sim] = (p_vals < alpha).sum()

print(f"Simulated mean false positives: {false_pos_counts.mean():.3f}")
print(f"P(4+ false positives): {(false_pos_counts >= 4).mean():.4f}")

# (3) With 30 tests, 4 significant results is plausible by chance alone.
# Need multiple testing correction (Bonferroni or BH) before trusting them.
print(f"\nBonferroni threshold: {alpha/n_features:.4f}  (only reject at this level)")
```

With 30 null tests, finding 4 significant at $\alpha=0.05$ happens in about 17% of simulations. You should not trust them without correction.
</details>
