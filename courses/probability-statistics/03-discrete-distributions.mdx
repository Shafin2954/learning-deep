## Why these distributions

A handful of discrete distributions appear everywhere in ML and statistics because they
arise naturally from simple processes: a coin flip, counting successes, counting events in
time. Knowing their shapes, parameters, and relationships lets you match a distribution to
a generative process instantly.

## Bernoulli$(p)$

A single binary trial with success probability $p \in [0,1]$.

$$
P(X = 1) = p, \quad P(X = 0) = 1-p
$$
$$
E[X] = p, \qquad \text{Var}(X) = p(1-p)
$$

The building block of everything that follows.

## Binomial$(n, p)$

Count of successes in $n$ **independent** Bernoulli$(p)$ trials.

$$
P(X = k) = \binom{n}{k} p^k (1-p)^{n-k}, \qquad k = 0, 1, \ldots, n
$$
$$
E[X] = np, \qquad \text{Var}(X) = np(1-p)
$$

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(3)
n, p = 20, 0.3

samples = rng.binomial(n, p, size=100_000)
print(f"Sample mean = {samples.mean():.4f}   true E[X] = {n*p:.4f}")
print(f"Sample var  = {samples.var():.4f}   true Var  = {n*p*(1-p):.4f}")

# PMF at k=6 (theoretical and empirical)
k = 6
pmf_true = stats.binom.pmf(k, n, p)
pmf_emp  = (samples == k).mean()
print(f"P(X=6)  true={pmf_true:.4f}  empirical={pmf_emp:.4f}")
```

```output
Sample mean = 6.0017   true E[X] = 6.0000
Sample var  = 4.2098   true Var  = 4.2000
P(X=6)  true=0.1916  empirical=0.1914
```

The Binomial approximates the Normal when $n$ is large and $p$ not too extreme (the CLT,
ch. 7). The **mode** is near $np$; the distribution is right-skewed for small $p$.

## Geometric$(p)$

Number of trials until the **first** success (inclusive).

$$
P(X = k) = (1-p)^{k-1} p, \qquad k = 1, 2, 3, \ldots
$$
$$
E[X] = \frac{1}{p}, \qquad \text{Var}(X) = \frac{1-p}{p^2}
$$

The Geometric is the **only discrete memoryless distribution**: $P(X > m+k \mid X > m) = P(X > k)$. Waiting times that "forget" their past are Geometric. This is why the number of requests until a server crash (if crashes are independent) is Geometric.

```python
import numpy as np

rng = np.random.default_rng(4)
p = 0.2
samples = rng.geometric(p, size=200_000)

print(f"E[X]   sample={samples.mean():.4f}   true={1/p:.4f}")
print(f"Var(X) sample={samples.var():.4f}   true={(1-p)/p**2:.4f}")

# Verify memoryless: P(X > 10 | X > 5) should equal P(X > 5)
P_gt5  = (samples > 5).mean()
P_gt10_given_gt5 = (samples > 10).mean() / (samples > 5).mean()
P_gt5_fresh = (samples > 5).mean()
print(f"P(X>5)           = {P_gt5:.4f}")
print(f"P(X>10 | X>5)    = {P_gt10_given_gt5:.4f}  ← should equal P(X>5)")
```

```output
E[X]   sample=5.0006   true=5.0000
Var(X) sample=20.1013   true=20.0000
P(X>5)           = 0.3278
P(X>10 | X>5)    = 0.3277  ← should equal P(X>5)
```

## Negative Binomial$(r, p)$

Number of trials until the $r$-th success. The Geometric is the special case $r = 1$.

$$
P(X = k) = \binom{k-1}{r-1} p^r (1-p)^{k-r}, \qquad k = r, r+1, \ldots
$$
$$
E[X] = \frac{r}{p}, \qquad \text{Var}(X) = \frac{r(1-p)}{p^2}
$$

In practice, the **NegBin distribution** is used for **overdispersed count data** — counts
where the variance exceeds the mean. Poisson requires Var = mean; NegBin allows Var > mean.
This is why NegBin appears in genomics (read counts) and text modeling (word counts).

## Poisson$(\lambda)$

Count of events in a fixed interval when events occur independently at rate $\lambda$.

$$
P(X = k) = \frac{\lambda^k e^{-\lambda}}{k!}, \qquad k = 0, 1, 2, \ldots
$$
$$
E[X] = \lambda, \qquad \text{Var}(X) = \lambda
$$

Mean equals variance — the defining constraint. If your count data has Var $\gg$ mean,
use NegBin; if Var $\approx$ mean, Poisson is appropriate.

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(6)
lam = 4.5

samples = rng.poisson(lam, size=200_000)
print(f"E[X]   sample={samples.mean():.4f}   true={lam}")
print(f"Var(X) sample={samples.var():.4f}   true={lam}  (mean ≈ var)")

# PMF shape: concentrated around lambda, right tail
for k in range(0, 11):
    bar = '#' * int(stats.poisson.pmf(k, lam) * 200)
    print(f"k={k:2d}  {bar}  {stats.poisson.pmf(k, lam):.4f}")
```

### Poisson as limit of Binomial

When $n \to \infty$, $p \to 0$, and $np \to \lambda$, Binomial$(n,p)$ → Poisson$(\lambda)$.
The Poisson is the "rare events" distribution.

## Try it Yourself

```python
import numpy as np
from scipy import stats

rng = np.random.default_rng(9)

# 1) A factory has a 0.5% defect rate per item. In a batch of 1000, what is
#    P(X >= 10)? Use both Binomial and Poisson (lambda = n*p = 5).
n, p = 1000, 0.005
lam = n * p   # = 5.0

print("P(X >= 10):")
print(f"  Binomial: {1 - stats.binom.cdf(9, n, p):.5f}")
print(f"  Poisson : {1 - stats.poisson.cdf(9, lam):.5f}")
print(f"  Simulation: {(rng.binomial(n, p, 500_000) >= 10).mean():.5f}")

# 2) NegBin: number of trials until 3rd success, p=0.4
r_nb, p_nb = 3, 0.4
x_nb = rng.negative_binomial(r_nb, p_nb, 300_000)
print(f"\nNegBin(r=3,p=0.4): E[X]={x_nb.mean():.3f} (true={r_nb/p_nb:.3f})")
```

```output
P(X >= 10):
  Binomial: 0.03183
  Poisson : 0.03183
  Simulation: 0.03198

NegBin(r=3,p=0.4): E[X]=7.499 (true=7.500)
```

## Exercise

Calls arrive at a call center at rate $\lambda = 8$ per hour. What is the probability that
in any given hour: (a) fewer than 5 calls arrive, (b) exactly 8 calls arrive, (c) more than
15 calls arrive? Use the Poisson PMF/CDF formula, then verify with simulation.

<details>
<summary>Solution</summary>

```python
import numpy as np
from scipy import stats

lam = 8.0
rng = np.random.default_rng(0)
sim = rng.poisson(lam, 500_000)

print(f"P(X < 5)  theory={stats.poisson.cdf(4, lam):.4f}  sim={(sim < 5).mean():.4f}")
print(f"P(X = 8)  theory={stats.poisson.pmf(8, lam):.4f}  sim={(sim == 8).mean():.4f}")
print(f"P(X > 15) theory={1-stats.poisson.cdf(15, lam):.4f}  sim={(sim > 15).mean():.4f}")
```

The Poisson CDF is $F(k) = \sum_{i=0}^k \lambda^i e^{-\lambda} / i!$; `scipy.stats.poisson.cdf`
computes it exactly. For $k \ge 4\lambda$, probabilities become very small — the tail
decays faster than exponential.
</details>
