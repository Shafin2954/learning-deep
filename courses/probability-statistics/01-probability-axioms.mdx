## What probability formalizes

Probability is a number between 0 and 1 assigned to an event. The assignment must be
consistent: if something certainly happens, it gets 1; if it certainly doesn't, it gets 0;
and the probability of two mutually exclusive events happening is the sum of their
individual probabilities. These three requirements — Kolmogorov's axioms — are all of
probability theory.

Formally: let $\Omega$ be the **sample space** (all possible outcomes). An **event** $A$
is a subset of $\Omega$. A **probability measure** $P$ satisfies:

$$
\text{1. } P(A) \ge 0 \quad \text{2. } P(\Omega) = 1 \quad \text{3. } P(A \cup B) = P(A) + P(B) \text{ if } A \cap B = \emptyset
$$

Everything else — conditional probability, independence, Bayes — follows from these three.

## Conditional probability

The conditional probability of $A$ given $B$ is the probability of $A$ restricted to
outcomes where $B$ already occurred:

$$
P(A \mid B) = \frac{P(A \cap B)}{P(B)}, \qquad P(B) > 0
$$

Rearranging: $P(A \cap B) = P(A \mid B) \cdot P(B)$. This is the **multiplication rule**.

```python
import numpy as np

rng = np.random.default_rng(0)
N = 100_000

# Roll a fair die. A = {even}, B = {> 3}
rolls = rng.integers(1, 7, N)           # 1..6 uniform
A = rolls % 2 == 0                       # {2, 4, 6}
B = rolls > 3                            # {4, 5, 6}

P_A     = A.mean()
P_B     = B.mean()
P_A_and_B = (A & B).mean()              # {4, 6}
P_A_given_B = P_A_and_B / P_B          # among {4,5,6}, fraction that are even

print(f"P(A)           = {P_A:.4f}   (true 3/6 = 0.5000)")
print(f"P(B)           = {P_B:.4f}   (true 3/6 = 0.5000)")
print(f"P(A and B)     = {P_A_and_B:.4f}   (true 2/6 = 0.3333)")
print(f"P(A|B)         = {P_A_given_B:.4f}   (true 2/3 = 0.6667)")
```

```output
P(A)           = 0.4994   (true 3/6 = 0.5000)
P(B)           = 0.4996   (true 3/6 = 0.5000)
P(A and B)     = 0.3330   (true 2/6 = 0.3333)
P(A|B)         = 0.6666   (true 2/3 = 0.6667)
```

## Independence

$A$ and $B$ are **independent** if knowing $B$ tells you nothing about $A$:

$$
P(A \mid B) = P(A) \quad \Leftrightarrow \quad P(A \cap B) = P(A) \cdot P(B)
$$

The second form (factorization) is the operational definition. Use it to test independence:
check whether the joint probability equals the product of the marginals.

## Law of total probability

If $B_1, B_2, \ldots, B_k$ partition $\Omega$ (mutually exclusive, exhaustive), then:

$$
P(A) = \sum_{i=1}^k P(A \mid B_i) \cdot P(B_i)
$$

This is how you **marginalize** a conditional probability into an unconditional one. It
appears in every Bayesian computation.

## Bayes' theorem

Invert the conditional: given the likelihood $P(A \mid B)$ and the marginals, find
$P(B \mid A)$.

$$
P(B \mid A) = \frac{P(A \mid B) \cdot P(B)}{P(A)} = \frac{P(A \mid B) \cdot P(B)}{\sum_i P(A \mid B_i) P(B_i)}
$$

In the ML context: $B$ is a hypothesis (model, class, parameter value); $A$ is data.
$P(B)$ is the **prior**; $P(A \mid B)$ is the **likelihood**; $P(B \mid A)$ is the
**posterior**. The denominator $P(A)$ is the marginal likelihood — it normalizes the
posterior.

```python
import numpy as np

# Classic medical test problem:
# Disease prevalence 1%, test sensitivity 99%, specificity 95%.
# Question: given a positive test, what is P(disease)?

P_disease   = 0.01                # prior P(D)
P_pos_given_disease   = 0.99      # sensitivity = P(+|D)
P_pos_given_no_disease = 0.05     # 1 - specificity = P(+|not D)

# Law of total probability for P(+)
P_pos = (P_pos_given_disease * P_disease
         + P_pos_given_no_disease * (1 - P_disease))

# Bayes
P_disease_given_pos = (P_pos_given_disease * P_disease) / P_pos

print(f"P(positive test)     = {P_pos:.4f}")
print(f"P(disease | positive) = {P_disease_given_pos:.4f}")
print(f"  → only {P_disease_given_pos*100:.1f}% chance of disease despite a 99% sensitive test")
```

```output
P(positive test)     = 0.0594
P(disease | positive) = 0.1667
  → only 16.7% chance of disease despite a 99% sensitive test
```

This counter-intuitive result — the **base-rate fallacy** — follows directly from Bayes.
The low prior (1% prevalence) overwhelms the strong likelihood. This is why ML models
trained on imbalanced classes with many false positives are hard to interpret without
calibration (ch. 15).

## Try it Yourself

```python
import numpy as np

# Simulate the classic Monty Hall problem and verify P(win|switch) = 2/3.
rng = np.random.default_rng(7)
N = 200_000

car   = rng.integers(0, 3, N)     # door hiding the car: 0, 1, or 2
first = rng.integers(0, 3, N)     # contestant's first pick

# Monty opens a door that is not the car and not the first pick.
# If you switch, you win if and only if your first pick was wrong.
win_if_switch = (first != car)
win_if_stay   = (first == car)

print(f"P(win | switch) = {win_if_switch.mean():.4f}  (true = 2/3 ≈ 0.6667)")
print(f"P(win | stay)   = {win_if_stay.mean():.4f}  (true = 1/3 ≈ 0.3333)")
```

```output
P(win | switch) = 0.6661  (true = 2/3 ≈ 0.6667)
P(win | stay)   = 0.3339  (true = 1/3 ≈ 0.3333)
```

## Exercise

A factory has three machines: M1 produces 50% of output with 2% defect rate, M2 produces
30% with 3% defect rate, M3 produces 20% with 5% defect rate. A randomly chosen item is
defective. What is the probability it came from M3?

<details>
<summary>Solution</summary>

```python
# P(defect | Mi) * P(Mi)
p = [0.50, 0.30, 0.20]   # machine fractions
d = [0.02, 0.03, 0.05]   # defect rates

# P(defective) by law of total probability
P_def = sum(pi * di for pi, di in zip(p, d))

# P(M3 | defective)
P_M3_given_def = (p[2] * d[2]) / P_def

print(f"P(defective)       = {P_def:.4f}")
print(f"P(M3 | defective)  = {P_M3_given_def:.4f}")
```

M3 produces 20% of items but contributes disproportionately to defects (5% rate), so the
posterior is higher than the prior — a direct demonstration of Bayes' theorem in quality
control.
</details>
