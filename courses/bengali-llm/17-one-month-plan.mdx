## The governing rules

This chapter ties the whole track into an execution plan for the month before July 25.
The plan encodes the discipline the chapters taught: **submit a baseline early, decide on
a trustworthy CV, escalate only on evidence, freeze before the deadline.**

1. **Get a submitted baseline in the first 2–3 days.** A submission on the board beats a
   perfect plan (ch. 8). It validates your whole pipeline end to end.
2. **Lock the metric and CV before optimizing** (ch. 14–15). You can't improve what you can't trust.
3. **Escalate by evidence** (ch. 8): prompt → RAG → fine-tune, each step only if the CV says the
   last one isn't enough.
4. **Trust CV over the public LB** (ch. 15). Decide offline; submit to confirm.
5. **Freeze early; leave days for submission-environment bugs** (ch. 13).

## Week 1 — Pipeline + baseline + trustworthy CV

- Read the task and data the day they drop. Run a **fertility check** (ch. 3) to pick
  candidate models from your shortlist (ch. 4).
- **First**, build the metric and a leak-proof CV (ch. 14–15): dedup (exact + near),
  group by source, stratify. This is the foundation; everything else trusts it.
- Build the **cheapest baselines**: zero/few-shot prompting (ch. 11) and the
  frozen-embedding classifier (ch. 5). **Submit** one. Confirm format and offline loading
  (ch. 13) now, not at the deadline.
- Output: a submitted score, a CV you trust, a model shortlist.

## Week 2 — The right approach

- Decide prompt vs RAG vs fine-tune (ch. 8) from Week-1 evidence. If knowledge task →
  stand up **RAG** (ch. 12). If skill/format task and baseline is short → **QLoRA** a
  small model (ch. 9) on cleaned data (ch. 10).
- Get *one* good non-baseline model working end to end and beating the baseline on your
  CV beyond noise (F14). Resist tuning before you have a working pipeline.
- Output: a model meaningfully above baseline; data pipeline solid (ch. 10).

## Week 3 — Depth: data and tuning

- The highest-return work is usually **data** (ch. 10): augmentation (validated on CV),
  cleaning, English-pivot tricks (ch. 11), better instruction formatting.
- Tune what matters: LoRA rank/$\eta$ (ch. 9), decoding (ch. 13), thresholds to the metric
  (ch. 14). Use the CV, ignore public-LB noise.
- Build **diversity** for the ensemble: a second base model, a different approach, varied
  seeds/prompts (ch. 16). Save out-of-fold predictions for every model.
- Output: 2–4 diverse, strong single models with OOF predictions.

## Week 4 — Ensemble + harden + freeze

- **Blend** by CV (ch. 16): rank/prob average or hill-climbed weights; consider careful
  pseudo-labeling if it holds up on clean CV.
- **Inference hardening** (ch. 13): merge adapters, batch + length-sort, confirm the
  blend fits the time budget, run the offline pre-submission checklist.
- **Freeze ${\sim}3$ days out.** Use the buffer only for submission-environment bugs. **Select
  final submissions by CV** (one safe, one aggressive), not by public LB.
- Output: a counted, reproducible submission you trust.

## Risk management

| Risk | Mitigation |
|------|-----------|
| Never submitted / format wrong | Submit Week 1, re-check format Week 4 (ch. 13) |
| Leakage / CV doesn't match LB | Dedup + group + adversarial validation (ch. 15) |
| Overfit the public LB | Decide on CV; limit probing |
| OOM / timeout at submission | Measure throughput early; use fallback chain (ch. 9, 13) |
| Burned the month tuning before a working pipeline | Baseline first, escalate by evidence |

**Time allocation rule of thumb:** ${\sim}20\%$ pipeline+CV, ${\sim}15\%$ baseline,
${\sim}35\%$ data+approach, ${\sim}15\%$ tuning, ${\sim}15\%$ ensemble+hardening. Notice how little
is "tuning" — the leverage is in data and validation.

## Try it Yourself

```python
# Days-remaining triage: what to prioritize given time left and current state.
def triage(days_left, has_baseline, has_trusted_cv):
    if not has_trusted_cv:
        return "STOP. Build the metric + leak-proof CV first (ch. 14-15). Nothing else matters yet."
    if not has_baseline:
        return "Submit a prompt/embedding baseline TODAY (ch. 8, 11). Validate the pipeline."
    if days_left > 14: return "Escalate approach by evidence (ch. 8); improve data (ch. 10)."
    if days_left > 5:  return "Lock approach; build ensemble diversity (ch. 16); save OOF."
    return "FREEZE. Ensemble by CV, harden inference (ch. 13), select finals by CV. No new ideas."

for d, b, cv in [(25, False, False), (20, True, True), (3, True, True)]:
    print(f"{d:2d} days | baseline={b} cv={cv}: {triage(d, b, cv)}")
```

```output
25 days | baseline=False cv=False: STOP. Build the metric + leak-proof CV first (ch. 14-15). Nothing else matters yet.
20 days | baseline=True cv=True: Escalate approach by evidence (ch. 8); improve data (ch. 10).
 3 days | baseline=True cv=True: FREEZE. Ensemble by CV, harden inference (ch. 13), select finals by CV. No new ideas.
```
