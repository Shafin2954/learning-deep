## The project

Build a complete, competition-grade Bengali LLM solution end to end — the same artifact
you'll submit on July 25 — **from scratch, with no AI writing it for you.** The point is
to prove you own every stage this track taught, so that on competition day nothing is
new. Pick any public Bengali task to build against now (sentiment, QA, NER, or
translation — e.g., a BanglaBERT-benchmark dataset or a Bengali subset of an Indic
benchmark); when the real task drops, you swap the data and reuse the machinery.

This is the integrative test. If you can do this unaided, you're ready.

## Hard constraints (the "no vibe coding" rules)

- **No AI assistant writes your code or your decisions.** You may read docs and these
  chapters. You may not have an LLM generate the pipeline.
- **Build the core stages yourself**, not by calling one high-level "do everything"
  wrapper. Specifically: write your own normalization, your own CV split, your own metric,
  your own ensembling/blend, and your own inference loop. You may use `transformers`/
  `peft` for the model itself (you built the concepts in F10–F12; re-deriving a 7B model
  isn't the point) — but the *pipeline* around it is yours.
- **Every decision must trace to evidence on your CV**, not to vibes.

## What it must contain (and the chapter each tests)

1. **Data pipeline** (ch. 1, 2, 10): load → normalize (NFC + Bengali rules) → language-
   filter → exact + near-dedup → group-and-stratify split → augment train-only → format
   with the chat template → loss-mask the prompt (labels = -100).
2. **Trustworthy CV + metric** (ch. 14, 15): implement the exact metric from scratch;
   build a leak-proof CV; run adversarial validation; confirm CV tracks a held-out probe.
3. **Baselines** (ch. 5, 8, 11): a frozen-embedding classifier *and* a zero/few-shot
   prompt. Record both scores as the bar to beat.
4. **A primary model** (ch. 4, 6, 7, 9): pick the architecture for the task; QLoRA-fine-
   tune it on your 8GB card (or justify prompting/RAG instead) with correct LoRA config,
   lr, warmup, early stopping. Beat the baseline on CV beyond noise.
5. **RAG, if the task needs knowledge** (ch. 12): chunk → hybrid retrieve (dense + BM25 +
   RRF) → rerank → grounded generation; evaluate retrieval and generation separately.
6. **Ensemble** (ch. 16): ≥2 diverse members, combined by a CV-optimized blend, with
   strict out-of-fold discipline.
7. **Inference + submission** (ch. 13): merged/quantized model, batched length-sorted
   generation, offline-load tested, output matching the submission format, within a time
   budget.
8. **A short write-up**: your CV scheme, what each experiment scored on CV vs the bar,
   why you chose what you chose, and your final-submission selection rule.

## Acceptance criteria

- The ensemble beats your best single model on CV, which beats both baselines — all
  measured on the *same* leak-proof CV (F14, F15).
- Your CV correlates with a held-out/probe score (no leakage).
- The whole thing runs offline end to end within a realistic time budget and emits a
  correctly formatted submission.
- You can explain every design choice in terms of CV evidence and the constraints of an
  8GB card — no step is magic to you.

## How to use it

Do a dry run on a public Bengali task this week. The first time you build the pipeline is
when you discover what you don't actually understand — far better to discover it now than
on July 25. When the competition opens, you replace the dataset and the metric, and the
machinery you already built and trust carries you. That readiness — not any single trick —
is what this whole track was for.

## Skills checklist (tick before competition day)

- [ ] I can normalize and dedup Bengali text correctly and explain why each step matters.
- [ ] I can pick a base model by task + fertility + 8GB feasibility.
- [ ] I can QLoRA-fine-tune on 8GB and debug OOM/NaN from memory.
- [ ] I can build a leak-proof, stratified, grouped CV and prove it tracks the LB.
- [ ] I can implement the competition metric from scratch.
- [ ] I can stand up Bengali RAG (hybrid + rerank) and evaluate it in two halves.
- [ ] I can ensemble diverse models by CV without leaking.
- [ ] I can ship an offline, in-budget, correctly-formatted submission.

When every box is ticked, the competition is just execution.
