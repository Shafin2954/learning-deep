## The project

You will build a complete Transformer from scratch — multi-head self-attention, positional
encoding, a full encoder or decoder block — train it on a real sequence task (char-level
language model or toy translation), and implement a KV cache for fast generation. The goal is
a working, trained model whose internals you built and understand, not a wrapper around
`nn.Transformer`.

## Hard constraints (no vibe coding)

- You MAY NOT use `torch.nn.Transformer`, `torch.nn.TransformerEncoder`, or
  `torch.nn.TransformerDecoder` for the core. These are off-limits.
- You MAY NOT use `torch.nn.MultiheadAttention` for the core attention. Implement scaled
  dot-product attention and multi-head attention yourself.
- You MAY use `torch.nn.Linear`, `torch.nn.Embedding`, `torch.nn.LayerNorm`, and
  `torch.nn.Dropout`.
- You MAY use the HuggingFace `tokenizers` library for tokenization, but not for
  model architecture.
- You MAY use a standard training loop from any tutorial — the constraint is on the
  model architecture, not the training boilerplate.

## What it must contain (and the chapter each tests)

- **Scaled dot-product attention with masking** (ch. 6–7): implement
  $\text{Attention}(Q,K,V) = \text{softmax}(QK^T/\sqrt{d_k})V$ from scratch; apply
  causal mask for decoder; verify output shape matches expected $(B, T, d_v)$.
- **Multi-head attention** (ch. 7): split into $h$ heads, attend in parallel, project
  back; verify that using $h=1$ with $d_k = d_{model}$ gives the same result as
  single-head attention.
- **Positional encoding** (ch. 8): implement sinusoidal PE; add an option for learned
  embeddings; plot the encoding matrix and verify the pattern.
- **Full Transformer block** (ch. 9): pre-norm or post-norm LayerNorm, FFN with 4×
  expansion, residual connections; stack at least 2 blocks.
- **Training** (ch. 10): train on a char-level language model (Shakespeare, Python code,
  or similar) or a toy translation task; reach perplexity below 3.0 on char-LM or
  BLEU > 10 on translation; log loss per epoch.
- **Decoding** (ch. 11): implement temperature sampling and greedy decoding; generate
  at least 200 characters of coherent (not random) text.
- **KV cache** (ch. 11): implement a KV cache that stores past keys and values;
  verify it produces identical output to the uncached forward pass for the same inputs;
  measure the speedup at sequence length 512.
- **Architecture comparison** (ch. 12): train both an encoder-only and a decoder-only
  variant on the same task; note the differences in how masking is applied.

## Acceptance criteria

- Scaled dot-product attention output matches PyTorch's
  `torch.nn.functional.scaled_dot_product_attention` on the same inputs to 1e-5
  absolute tolerance.
- Causal mask correctly prevents attention to future positions: attention weight matrix
  is lower-triangular (verified by inspection).
- Sinusoidal PE is unique per position and smooth: visualized as a heatmap.
- Model trains to perplexity ≤ 3.0 on a char-level dataset within 20 epochs (or
  BLEU > 10 if translation task).
- Generated text is coherent: not a random character sequence and follows the training
  distribution visibly.
- KV cache produces byte-for-byte identical logits vs the uncached model on the same
  prefix (verified with `torch.allclose`).
- KV cache provides measurable speedup (≥ 1.5×) at sequence length ≥ 256 vs
  recomputing from scratch each step.
