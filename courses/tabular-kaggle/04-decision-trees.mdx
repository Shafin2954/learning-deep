## Why trees dominate tabular data

Decision trees are the atom of tabular ML — random forests and gradient boosting (the
Kaggle-winning models) are just clever ensembles of them. Build one by hand and the whole
tree-based world opens up.

A decision tree repeatedly splits the data on a single feature threshold, carving the input
space into rectangular regions; each leaf predicts a constant (the mean target for regression,
the majority/probability for classification). To predict, you drop a sample down the tree,
following the splits, until it lands in a leaf.

Trees (1) capture **nonlinearity** and **feature interactions** automatically (a split on
feature $B$ inside a branch of $A$ is an interaction); (2) need **no feature scaling**
(splits are scale-invariant — contrast with linear models, ch. 2); (3) handle **mixed types**
and missing values natively; (4) are **interpretable**. The price: a single deep tree
**overfits** badly.

## How a split is chosen

At each node, the tree tries every feature and every candidate threshold and picks the split
that most reduces **impurity**:

- **Regression:** impurity = SSE; a good split makes the two children's targets as
  homogeneous (low-variance) as possible.
- **Classification:** impurity = **Gini** $\bigl(1 - \sum_k p_k^2\bigr)$ or **entropy**
  $\bigl(-\sum_k p_k \log p_k\bigr)$; a good split makes each child as class-pure as possible.

Greedy and recursive: choose the best split, partition, then recurse on each child.

## Building a regression tree from scratch

```python
import numpy as np
np.random.seed(0)

# A regression decision tree from scratch (CART with SSE splits).
class Node:
    __slots__ = ("feat", "thr", "left", "right", "val")

def build(X, y, depth=0, max_depth=5, min_leaf=10):
    node = Node(); node.feat = None; node.val = y.mean()
    node.left = node.right = node.thr = None
    if depth >= max_depth or len(y) < 2*min_leaf or np.allclose(y, y[0]):
        return node                                            # make a leaf
    best = (np.inf, None, None)                                # (sse, feature, threshold)
    for f in range(X.shape[1]):
        vals = np.unique(X[:, f]); cand = (vals[:-1] + vals[1:]) / 2
        for t in cand:
            m = X[:, f] <= t
            if m.sum() < min_leaf or (~m).sum() < min_leaf:
                continue
            sse = ((y[m]-y[m].mean())**2).sum() + ((y[~m]-y[~m].mean())**2).sum()
            if sse < best[0]:
                best = (sse, f, t)
    if best[1] is None:                                        # no useful split -> leaf
        return node
    node.feat, node.thr = best[1], best[2]
    m = X[:, node.feat] <= node.thr
    node.left  = build(X[m],  y[m],  depth+1, max_depth, min_leaf)
    node.right = build(X[~m], y[~m], depth+1, max_depth, min_leaf)
    return node

def predict(node, X):
    def one(x):
        n = node
        while n.feat is not None:
            n = n.left if x[n.feat] <= n.thr else n.right
        return n.val
    return np.array([one(x) for x in X])

# fit and evaluate
N = 400
X = np.random.randn(N, 3)
y = X[:, 0]**2 + 2*X[:, 1] + 0.3*np.random.randn(N)            # nonlinear target
tr, va = slice(0, 300), slice(300, 400)
tree = build(X[tr], y[tr], max_depth=5, min_leaf=10)
rmse = np.sqrt(((predict(tree, X[va]) - y[va])**2).mean())
baseline = np.sqrt(((y[tr].mean() - y[va])**2).mean())
print(f"tree RMSE {rmse:.3f} vs predict-the-mean {baseline:.3f}")  # tree wins
```

```output
tree RMSE 1.078 vs predict-the-mean 2.211
```

## Overfitting and the depth tradeoff

A shallow tree underfits (high bias); a deep unconstrained tree overfits (high variance). This
is the bias–variance tradeoff (F6) made concrete. The overfitting parameters — `max_depth`,
`min_samples_leaf`, `min_split_gain` — are exactly the regularization knobs you'll see in
XGBoost and LightGBM (ch. 7).

## Try it Yourself

```python
import numpy as np
np.random.seed(0)
# (reuse build/predict from the Example above)

# Watch overfitting: deeper, less-constrained trees fit train better but val worse.
N = 400; X = np.random.randn(N, 3)
y = X[:, 0]**2 + 2*X[:, 1] + 0.3*np.random.randn(N)
tr, va = slice(0, 300), slice(300, 400)

for depth in [1, 3, 5, 10, 20]:
    t = build(X[tr], y[tr], max_depth=depth, min_leaf=1)
    tr_rmse = np.sqrt(((predict(t, X[tr]) - y[tr])**2).mean())
    va_rmse = np.sqrt(((predict(t, X[va]) - y[va])**2).mean())
    print(f"max_depth={depth:2d}: train RMSE {tr_rmse:.3f}  val RMSE {va_rmse:.3f}")
# Train RMSE keeps dropping; val RMSE bottoms out then rises -> the U-curve (F6).
# That gap is why a single tree isn't enough: ch. 5 (averaging) and ch. 6 (boosting) fix it.
```

```output
max_depth= 1: train RMSE 1.801  val RMSE 1.706
max_depth= 3: train RMSE 1.164  val RMSE 1.258
max_depth= 5: train RMSE 0.661  val RMSE 0.947
max_depth=10: train RMSE 0.180  val RMSE 0.684
max_depth=20: train RMSE 0.000  val RMSE 0.681
```
