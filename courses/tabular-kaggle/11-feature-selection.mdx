## Why selection matters

After generating many features (ch. 9–10), you'll have noise mixed with signal. **Feature
selection** keeps the useful columns and drops the rest. Too many noise features (1) increase
overfitting and training time, (2) dilute importance signals, (3) hurt linear/distance models
badly, and (4) make ensembles and inference slower.

## Three families of selection

### Filter methods

Score each feature against the target *independently* of any model, then keep the top ones.
Cheap and fast. Tools: **correlation** (Pearson/Spearman) for numeric, **mutual information**
(any dependence, including nonlinear), **chi-square** for categoricals. Good for a first cull
of obviously-useless columns; blind to feature interactions.

### Wrapper methods

Use a model's performance to choose. **RFE:** train, drop the weakest feature(s), repeat.
**Forward/backward selection:** add or remove features greedily by CV score. Accurate but
expensive. Use on moderate feature counts.

### Embedded methods

Selection happens *inside* model training. **L1/Lasso** zeros out weak features (ch. 3).
**Tree importance** (gain) ranks features by how much they reduced loss — you get it from the
model you're already training.

## Permutation importance — the gold standard

Train a model, then for each feature, **shuffle that column** and measure how much the score
drops. A big drop = the model relied on it; ~0 = useless. Model-agnostic and reflects the
feature's *real* contribution, unlike tree "gain" which is biased toward high-cardinality and
correlated features.

```python
import numpy as np
from numpy.random import default_rng
rng = default_rng(1)

# Permutation importance: shuffle a feature, measure the score drop.
X = rng.normal(size=(400, 4))
w = np.array([2.0, 0.0, 1.0, 0.0])               # only features 0 and 2 matter
y = (X @ w + rng.normal(0, 0.3, 400) > 0).astype(int)

def fit_logreg(X, y, steps=500, lr=0.1):
    Xb = np.c_[np.ones(len(X)), X]; w = np.zeros(Xb.shape[1])
    for _ in range(steps):
        p = 1 / (1 + np.exp(-Xb @ w)); w -= lr * Xb.T @ (p - y) / len(X)
    return w
def accuracy(X, w, y):
    Xb = np.c_[np.ones(len(X)), X]; return ((1/(1+np.exp(-Xb@w)) > 0.5) == y).mean()

w_fit = fit_logreg(X, y)
base = accuracy(X, w_fit, y)
print(f"baseline accuracy: {base:.3f}\npermutation importance per feature:")
for j in range(4):
    Xp = X.copy(); rng.shuffle(Xp[:, j])          # destroy feature j's signal
    print(f"  feature {j}: drop = {base - accuracy(Xp, w_fit, y):.3f}")
# Features 0 and 2 show real drops; 1 and 3 are ~0 -> drop them.
```

```output
baseline accuracy: 0.963
permutation importance per feature:
  feature 0: drop = 0.393
  feature 1: drop = 0.003
  feature 2: drop = 0.152
  feature 3: drop = 0.000
```

## Null importance — is it real or luck?

Train with the **real** target and record importances; then train many times with a **shuffled**
target and record the importances features get *by chance*. A feature is genuinely useful only
if its real importance clearly exceeds its null (shuffled-target) distribution. Invaluable when
you've generated hundreds of engineered features.

## Try it Yourself

```python
import numpy as np
from numpy.random import default_rng
rng = default_rng(0)
# Null importance: is a feature's importance real, or luck? Compare to shuffled-target runs.
X = rng.normal(size=(300, 3))
y = (X[:, 0] * 1.5 + rng.normal(0, 0.5, 300) > 0).astype(int)   # only feature 0 matters

def importance(X, y):
    # crude importance = |correlation of each feature with the target|
    return np.array([abs(np.corrcoef(X[:, j], y)[0, 1]) for j in range(X.shape[1])])

real = importance(X, y)
null = np.array([importance(X, rng.permutation(y)) for _ in range(200)])   # shuffled targets
print("feature | real imp | 95th pct of null | real > null?")
for j in range(3):
    thresh = np.percentile(null[:, j], 95)
    print(f"   {j}    |  {real[j]:.3f}  |     {thresh:.3f}        | {real[j] > thresh}")
# Feature 0's real importance clears its null distribution -> genuine.
# Features 1,2 sit inside their null -> importance is just chance -> drop.
```

```output
feature | real imp | 95th pct of null | real > null?
   0    |  0.757  |     0.124        | True
   1    |  0.056  |     0.106        | False
   2    |  0.043  |     0.114        | False
```
