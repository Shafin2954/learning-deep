## The priority order — internalize this ranking

This chapter ties the track into one execution plan for a tabular competition.

1. **A CV you can trust** (ch. 12–13) — the foundation; everything is measured against it.
2. **Features** (ch. 9–10) — the biggest accuracy lever on tabular data.
3. **Diverse models + ensembling** (ch. 7, 16) — the finisher.
4. **Tuning** (ch. 15) — the smallest lever; do it last and lightly.

Spend your time in that order. Most people invert it (tuning first) and lose.

## Phase 1 — Understand & set up (first day or two)

- Read the problem and the **exact metric** (ch. 14). Implement the metric yourself.
- **EDA** (ch. 8): target distribution, missingness, train-vs-test comparison, hunt leaks and
  structure. Produce a "findings → actions" list.
- **Build a trustworthy CV** (ch. 12): match the fold scheme to the test (stratified/grouped/
  time). Run **adversarial validation** (ch. 13) to confirm train ≈ test.
- **Baseline + submit**: a default LightGBM in your CV, submitted to validate the pipeline and
  format. Record the CV and LB.

## Phase 2 — Features (the bulk of your time)

- Engineer aggressively (ch. 9): groupby aggregations, ratios, datetime, missing indicators.
- Encode categoricals safely (ch. 10): frequency, leak-safe K-fold target encoding.
- Prune with feature selection (ch. 11): permutation + null importance, confirm on CV.
- Every change is kept only if it beats CV beyond fold-noise.

## Phase 3 — Models & light tuning

- Train **LightGBM, XGBoost, CatBoost** under the *same* CV (ch. 7), saving each one's OOF and
  test predictions. Add a linear model and maybe a neural net for diversity.
- **Tune lightly** with Optuna (ch. 15): LR + complexity + regularization, on CV, with early
  stopping. Don't over-tune.

## Phase 4 — Ensemble & finish

- Blend/stack the diverse models on OOF (ch. 16): simple average → rank/weighted → stack, keep
  the CV winner. Add **seed averaging** (ch. 17).
- Consider careful **pseudo-labeling** (ch. 17) if it holds up on a clean CV.
- **Post-process to the metric** (ch. 14): threshold/rounding/calibration on OOF.
- **Select final submissions by CV** (ch. 13), one safe and one aggressive — never by the public
  LB. Confirm reproducibility.

## Competition burndown tracker

```python
# The competition burndown — are you doing things in the right order?
phases = {
    "Phase 1 (setup)":   ["metric implemented", "EDA findings->actions", "trustworthy CV built",
                          "adversarial validation run", "baseline submitted"],
    "Phase 2 (features)":["groupby aggregations", "safe categorical encoding (ch.10)",
                          "feature selection confirmed on CV"],
    "Phase 3 (models)":  ["LGB+XGB+CatBoost on same CV", "OOF + test preds saved",
                          "light Optuna tuning"],
    "Phase 4 (finish)":  ["ensemble blended/stacked by CV", "seed averaging",
                          "post-processed to metric", "finals selected by CV"],
}
done = {"metric implemented", "EDA findings->actions", "trustworthy CV built"}
for phase, tasks in phases.items():
    for t in tasks:
        print(f"[{'x' if t in done else ' '}] {phase}: {t}")
```

## Try it Yourself

```python
# Priority-check: given limited time, what should you do next?
def next_move(has_cv, has_baseline, features_done, has_ensemble, days_left):
    if not has_cv:        return "Build a trustworthy CV + adversarial validation (ch.12-13). Nothing else counts yet."
    if not has_baseline:  return "Submit a baseline TODAY (ch.1). Validate pipeline + format."
    if not features_done and days_left > 5: return "Engineer features (ch.9-10) — the biggest lever."
    if not has_ensemble:  return "Train diverse models + ensemble on OOF (ch.7,16)."
    return "Post-process to metric (ch.14), seed-average, select finals by CV. Freeze."

for state in [
    dict(has_cv=False, has_baseline=False, features_done=False, has_ensemble=False, days_left=20),
    dict(has_cv=True,  has_baseline=True,  features_done=True,  has_ensemble=False, days_left=8),
    dict(has_cv=True,  has_baseline=True,  features_done=True,  has_ensemble=True,  days_left=2),
]:
    print(next_move(**state))
```

```output
Build a trustworthy CV + adversarial validation (ch.12-13). Nothing else counts yet.
Train diverse models + ensemble on OOF (ch.7,16).
Post-process to metric (ch.14), seed-average, select finals by CV. Freeze.
```
