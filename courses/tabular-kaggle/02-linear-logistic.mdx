## Fast, honest baselines

Linear and logistic regression are your fast, honest **baselines** — the models you fit in
minutes to get a number every fancier approach must beat. They're also the conceptual root of
everything: a neuron is a linear model + nonlinearity (F1–F2), and understanding them cold
makes regularization (ch. 3) and boosting (ch. 6) obvious.

## Linear regression

Predicts a continuous target as a weighted sum of features plus a bias:

$$
\hat{y} = w^\top x + b
$$

You fit $w$ by minimizing **mean squared error** (MSE). Two ways, and they agree:

- **Closed form (normal equation):** $w = (X^\top X)^{-1} X^\top y$. Exact, fine for few
  features, expensive/unstable for many.
- **Gradient descent (F2, F4):** iteratively step downhill on the MSE. Scales to many features.

```python
import numpy as np
np.random.seed(0)

N = 200
X = np.random.randn(N, 3)
true_w = np.array([2.0, -1.0, 0.5])
y = X @ true_w + 0.1 * np.random.randn(N)
Xb = np.c_[np.ones(N), X]                       # add bias column

# closed form: w = (X'X)^-1 X'y
w_closed = np.linalg.solve(Xb.T @ Xb, Xb.T @ y)

# gradient descent (F2/F4)
w = np.zeros(4); lr = 0.1
for _ in range(2000):
    grad = Xb.T @ (Xb @ w - y) / N              # gradient of MSE
    w -= lr * grad

print("closed form:", w_closed.round(3))
print("grad descent:", w.round(3), "-> match:", np.allclose(w, w_closed, atol=1e-2))
# Both recover the true [bias≈0, 2, -1, 0.5].
```

```output
closed form: [-0.015  2.003 -1.014  0.492]
grad descent: [-0.015  2.003 -1.014  0.492] -> match: True
```

## Logistic regression

The classification cousin: push the linear score through a **sigmoid** to get a probability in
$(0, 1)$, then minimize **log loss** (F3):

$$
\hat{p} = \sigma(w^\top x + b) = \frac{1}{1 + e^{-(w^\top x + b)}}
$$

Despite the name it's a *classifier*. It outputs calibrated-ish probabilities (good for
AUC/log-loss metrics), is extremely fast, and is a genuinely strong baseline on high-dimensional
sparse data. The gradient of log loss is simply $X^\top(p - y)/N$ (F1's clean result):

```python
import numpy as np
np.random.seed(0)
# Logistic regression from scratch — the classification baseline.
def sigmoid(z): return 1 / (1 + np.exp(-z))

X = np.random.randn(300, 2)
y = (X[:, 0] + 0.5 * X[:, 1] > 0).astype(int)
Xb = np.c_[np.ones(300), X]; w = np.zeros(3); lr = 0.5
for _ in range(2000):
    p = sigmoid(Xb @ w)
    w -= lr * Xb.T @ (p - y) / 300              # gradient of log loss = X'(p - y)/N
print("accuracy:", round(float(((sigmoid(Xb @ w) > 0.5) == y).mean()), 3))
print("probabilities, not just classes -> good for AUC/log-loss metrics")
```

```output
accuracy: 0.997
probabilities, not just classes -> good for AUC/log-loss metrics
```

## Why baselines matter on Kaggle

A linear/logistic baseline (a) validates your whole pipeline — data loading, CV, submission
format; (b) gives a score to beat; and (c) is a **diverse ensemble member** (ch. 16) — a
linear model's errors differ from a tree's, so blending the two often helps even when the linear
model is weaker alone.

## Practical notes

- **Scale features** (standardize) before linear/logistic models — unlike trees, they're
  sensitive to feature scale. Use a `Pipeline` so scaling is fit inside the CV fold (avoids
  preprocessing leakage, F15).
- **Encode categoricals** sensibly: one-hot for linear models (label-encoding implies a
  false order). High-cardinality categoricals need target/frequency encoding (ch. 10).
- **Regularize** (ch. 3): plain linear/logistic overfit when features are many or correlated.

## Try it Yourself

```python
import numpy as np
np.random.seed(1)
def sigmoid(z): return 1 / (1 + np.exp(-z))

# Why scaling matters for linear/logistic (but NOT for trees).
N = 400
X = np.random.randn(N, 2)
X[:, 1] *= 1000                                  # one feature on a huge scale
y = (X[:, 0] + X[:, 1] / 1000 > 0).astype(int)

def train(Xin, steps=300, lr=0.1):
    Xb = np.c_[np.ones(len(Xin)), Xin]; w = np.zeros(Xb.shape[1])
    for _ in range(steps):
        p = sigmoid(Xb @ w); w -= lr * Xb.T @ (p - y) / len(Xin)
    return ((sigmoid(Xb @ w) > 0.5) == y).mean()

raw = train(X)
Xs = (X - X.mean(0)) / X.std(0)                  # standardize
scaled = train(Xs)
print(f"accuracy raw features  : {raw:.3f}")
print(f"accuracy standardized  : {scaled:.3f}")
# Standardizing fixes the badly-scaled feature -> faster, better convergence.
# (A decision tree would be unaffected by scale — that's a tree advantage, ch. 4.)
```

```output
accuracy raw features  : 0.710
accuracy standardized  : 0.980
```
