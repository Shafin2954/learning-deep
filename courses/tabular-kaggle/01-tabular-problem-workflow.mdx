## Why tabular is its own world

Most Kaggle competitions are **tabular**: rows of samples, columns of features, one target
to predict. This track is about winning those, and the workflow here is the backbone every
later chapter plugs into.

Unlike images or text, tabular data is heterogeneous — mixed numeric and categorical columns,
different scales, missing values, nonlinear interactions, no spatial/sequential structure to
exploit. The models that dominate here are **gradient-boosted decision trees**
(XGBoost/LightGBM/CatBoost), not neural nets, because trees handle mixed types, missing values,
and feature interactions natively, need no scaling, and are extremely sample-efficient.

## Where tabular competitions are actually won

Not in model choice (everyone uses the same three libraries) but in three places:

1. A **trustworthy validation scheme** that mirrors the test set (ch. 12–13).
2. **Feature engineering** that exposes signal the model can't find on its own (ch. 9–10).
3. **Ensembling** diverse models (ch. 16).

Tuning hyperparameters is real but the smallest of these levers.

## The Kaggle workflow

1. **Understand the problem and metric.** Read the task, the data, and the exact **evaluation
   metric** (ch. 14). Everything you do optimizes *this*.
2. **EDA.** Explore distributions, missingness, correlations, the target, and the relation between
   train and test (ch. 8). Hunt for leaks and quirks now.
3. **Build a trustworthy CV.** Before modeling, decide the cross-validation scheme that imitates
   how the test set was made (ch. 12). This is your compass; everything is measured against it.
4. **Baseline.** Train a simple model (a default LightGBM, or even predict-the-mean), score it on
   CV, and **submit** it. A submitted baseline validates your whole pipeline and gives you a number
   to beat.
5. **Iterate by evidence.** Improve features, handle missing/categoricals, tune — keeping only
   changes that beat your CV beyond its noise (ch. 14). Track every experiment.
6. **Ensemble.** Combine diverse models by CV (ch. 16). Usually the final points.
7. **Select and submit finals by CV, not the public LB** (ch. 13). Guard against the shake-up.

## Try it Yourself

```python
# A complete (tiny) tabular pipeline you can run: synthetic data, CV, OOF, score.
import numpy as np
np.random.seed(0)

N, D = 1000, 5
X = np.random.randn(N, D)
y = (X @ np.array([1.5, -1, 0.5, 0, 2]) + np.random.randn(N) > 0).astype(int)

def sigmoid(z): return 1 / (1 + np.exp(-z))

def auc(scores, y):
    o = np.argsort(scores); r = np.empty_like(o, float); r[o] = np.arange(len(scores)) + 1
    p = y == 1; return (r[p].sum() - p.sum()*(p.sum()+1)/2) / (p.sum()*(~p).sum())

# 5-fold CV with a logistic-regression model, collecting OOF predictions
k = 5; folds = np.arange(N) % k; np.random.shuffle(folds)
oof = np.zeros(N)
for f in range(k):
    tr, va = folds != f, folds == f
    Xt = np.c_[np.ones(tr.sum()), X[tr]]; w = np.zeros(D+1)
    for _ in range(500):
        p = sigmoid(Xt @ w); w -= 0.1 * Xt.T @ (p - y[tr]) / tr.sum()
    oof[va] = sigmoid(np.c_[np.ones(va.sum()), X[va]] @ w)
print("CV AUC from OOF predictions:", round(auc(oof, y), 4))
```

```output
CV AUC from OOF predictions: 0.9632
```
