## From your hand-built GBM to the libraries

You built gradient boosting by hand (ch. 6). The three libraries that dominate tabular Kaggle
are that algorithm with better math and serious engineering. You rarely need to choose just one
тАФ top solutions ensemble all three (ch. 16) because their different growth strategies make
different errors.

## XGBoost тАФ the robust classic

Grows trees **level-wise (depth-wise)**: it expands all nodes at a depth before going deeper,
producing balanced trees. It uses a **second-order (Newton) objective** тАФ both the gradient
and the curvature (Hessian) of the loss to choose splits and leaf values, which is more precise
than the first-order residual fitting you coded. Built-in L1/L2 regularization on leaf values
(`reg_alpha`, `reg_lambda`) and a `gamma` minimum-split-gain. Reliable, well-documented, a
strong default.

## LightGBM тАФ the fast favorite

Grows trees **leaf-wise (best-first)**: it splits the single leaf that most reduces loss,
anywhere in the tree, producing deeper, asymmetric trees that fit faster and often more
accurately тАФ but overfit more on small data (control with `num_leaves` and `min_data_in_leaf`,
*not* just `max_depth`). Three speed tricks: **histogram-based splitting** (bin continuous
features into ~255 buckets), **GOSS** (keep the high-gradient rows, subsample the well-fit
ones), and **EFB** (bundle sparse mutually-exclusive features). Watch `num_leaves` тАФ set it
well below $2^{\text{max\_depth}}$ or it overfits.

## CatBoost тАФ the categorical specialist

Two signature ideas. **Ordered target statistics:** a leak-safe way to target-encode
categoricals (ch. 10) using only rows seen "before" each one in a random permutation тАФ so it
handles categoricals *natively*, no manual encoding, no target leak. **Ordered boosting:** a
permutation scheme that fixes the subtle **prediction-shift** bias of standard boosting. Often
the best out-of-the-box on categorical-heavy data with little tuning. Pass categorical columns
via `cat_features` and let it handle them.

## Parameters that actually matter

Tune roughly in this order (ch. 15):

1. **learning\_rate** + **n\_estimators** with **early stopping** тАФ set a smallish LR (0.01тАУ0.05),
   many rounds, and let early stopping pick the count. Do this first.
2. **Tree complexity** тАФ `num_leaves`/`max_depth`, `min_child_samples`/`min_data_in_leaf`.
3. **Randomness/regularization** тАФ `subsample`/`bagging_fraction`,
   `colsample_bytree`/`feature_fraction`, `reg_alpha`/`reg_lambda`.
4. **objective** and **eval\_metric** must match the competition metric.

## Library templates (run locally)

```python
# The three libraries, same shape (run locally). Note: all use early stopping on a val set.
import numpy as np
# import lightgbm as lgb, xgboost as xgb; from catboost import CatBoostClassifier

def fit_lightgbm(Xtr, ytr, Xva, yva):
    import lightgbm as lgb
    model = lgb.LGBMClassifier(
        learning_rate=0.03, n_estimators=5000,        # many rounds + early stopping
        num_leaves=31, min_child_samples=30,           # leaf-wise complexity control
        subsample=0.8, colsample_bytree=0.8,
        reg_alpha=0.0, reg_lambda=1.0, random_state=42)
    model.fit(Xtr, ytr, eval_set=[(Xva, yva)],
              callbacks=[lgb.early_stopping(150), lgb.log_evaluation(0)])
    return model

def fit_xgboost(Xtr, ytr, Xva, yva):
    import xgboost as xgb
    model = xgb.XGBClassifier(
        learning_rate=0.03, n_estimators=5000,
        max_depth=6, min_child_weight=5,               # level-wise complexity control
        subsample=0.8, colsample_bytree=0.8,
        reg_lambda=1.0, eval_metric="auc",
        early_stopping_rounds=150, random_state=42)
    model.fit(Xtr, ytr, eval_set=[(Xva, yva)], verbose=False)
    return model

def fit_catboost(Xtr, ytr, Xva, yva, cat_features):
    from catboost import CatBoostClassifier
    model = CatBoostClassifier(
        learning_rate=0.03, iterations=5000, depth=6,
        l2_leaf_reg=3.0, random_seed=42, verbose=0,
        cat_features=cat_features)                     # native categorical handling
    model.fit(Xtr, ytr, eval_set=(Xva, yva), early_stopping_rounds=150)
    return model
print("Same loop as ch.6, engineered three ways. Ensemble all three (ch.16).")
```
