## The project

You will implement the core of graph neural network learning from scratch — GCN message
passing, negative sampling, and TransE KG embeddings — and compare them against framework
implementations. Then you will do both node classification and link prediction on the Cora
citation network, visualize what the models learned, and demonstrate the over-smoothing
phenomenon empirically.

## Hard constraints (no vibe coding)

- You MAY NOT use `torch_geometric.nn.GCNConv` for the from-scratch GCN. Implement the
  GCN layer as sparse matrix multiplication: compute $\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}$
  yourself and apply it to $H W$.
- You MAY NOT use `torch_geometric.utils.negative_sampling` for the link prediction
  training loop. Implement your own random non-edge sampler.
- You MAY NOT use PyKEEN for the core TransE or DistMult implementation. Write the
  scoring function and margin loss yourself.
- For **comparison experiments** (not the from-scratch baseline), you MAY use
  `torch_geometric.nn.SAGEConv`, `GATConv`, and `GATv2Conv`.
- You MAY use PyTorch's `DataLoader`, `Planetoid`, and `TUDataset`.

## What it must contain (and the chapter each tests)

- **GCN from scratch** (ch. 4–5): implement symmetric-normalized adjacency multiplication
  and a 2-layer GCN with ReLU, dropout, and cross-entropy on the Cora train split. Report
  train/val/test accuracy after 200 epochs.
- **Negative sampling from scratch** (ch. 9): write a function that, given `edge_index` and
  `num_nodes`, returns random non-edges. Verify: zero overlap with the real edge set on 1000
  sampled pairs.
- **Link prediction** (ch. 9): train a GNN encoder + dot-product decoder on Cora edges (use
  `RandomLinkSplit` for the split). Report AUC-ROC and Average Precision on the test split.
- **Inductive experiment** (ch. 6): train GraphSAGE on 80% of Cora nodes; embed and classify
  the remaining 20% without re-training. Report test accuracy on the held-out nodes.
- **GAT attention visualization** (ch. 7): inspect the attention weights of head 0 in layer 1
  for 3 selected nodes. Print their top-5 attended neighbors and explain whether the pattern
  makes sense given node labels.
- **Over-smoothing demonstration** (ch. 5): train GCN with depth 1, 2, 4, 8, 16 on Cora.
  Plot test accuracy vs depth; show it peaks at depth 2 and degrades.
- **TransE or DistMult** (ch. 11): implement on a small subset of a real KG (or the toy KG
  from ch. 11). Train to convergence and show the top-3 predictions for 2 test queries;
  report MRR on held-out triples.
- **t-SNE visualization** (ch. 3): extract final node embeddings from your best GCN model
  and visualize them with t-SNE, colored by ground-truth class. Describe the clustering.

## Acceptance criteria

- From-scratch GCN reaches ≥ 78% test accuracy on Cora within 200 epochs.
- Negative sampler has zero overlap with real edges on a 1000-sample sanity check.
- Link prediction AUC ≥ 0.85 on Cora citation test split.
- Inductive GraphSAGE test accuracy on unseen nodes ≥ 72%.
- GAT attention visualization prints non-uniform weights (not all equal); the highest-
  attended neighbors are plausibly related to the node's class.
- Over-smoothing plot clearly shows accuracy peak at depth 2 then monotonic decrease.
- KG embedding ranks at least one correct tail in the top 3 for each test query.
- t-SNE plot shows at least 4 visually distinct clusters corresponding to node classes.
