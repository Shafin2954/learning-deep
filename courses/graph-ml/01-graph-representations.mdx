## What a graph is

A graph $G = (V, E)$ has a set of **nodes** $V$ and **edges** $E \subseteq V \times V$. Nodes represent entities (people, papers, atoms); edges represent relationships (friendship, citation, bond). Both can carry **attributes**: node features $x_v \in \mathbb{R}^d$, edge weights $w_{uv} \in \mathbb{R}$.

Graphs come in four flavors based on two binary choices:

| | Undirected | Directed |
|---|---|---|
| **Unweighted** | social network | web page links |
| **Weighted** | road distances | email frequency |

In an **undirected** graph $(u,v)$ and $(v,u)$ are the same edge. In a **directed** graph (digraph) they are distinct.

## Three ways to store a graph

### Adjacency matrix

The $N \times N$ matrix where $A_{uv} = 1$ if edge $(u,v)$ exists (0 otherwise). For weighted graphs, $A_{uv} = w_{uv}$.

```python
import numpy as np

# 4-node undirected graph
A = np.array([
    [0, 1, 1, 0],
    [1, 0, 1, 1],
    [1, 1, 0, 0],
    [0, 1, 0, 0],
])
print("Adjacency matrix:")
print(A)
print(f"Density: {A.sum() / (A.shape[0]**2 - A.shape[0]):.2f}")
```

```output
Adjacency matrix:
[[0 1 1 0]
 [1 0 1 1]
 [1 1 0 0]
 [0 1 0 0]]
Density: 0.67
```

The adjacency matrix costs $O(N^2)$ memory. For a social network with 1 million users that's 1 TB at float32 — completely unworkable. Real graphs are **sparse**: most node pairs aren't connected. Facebook's social graph has ~3 billion nodes but average degree ~200, making it 99.999993% zeros.

### Adjacency list

Each node stores only its neighbors. Cost: $O(N + E)$.

```python
# Same graph as adjacency list
adj_list = {
    0: [1, 2],
    1: [0, 2, 3],
    2: [0, 1],
    3: [1],
}
for node, neighbors in adj_list.items():
    print(f"Node {node}: neighbors {neighbors}")
```

```output
Node 0: neighbors [1, 2]
Node 1: neighbors [0, 2, 3]
Node 2: neighbors [0, 1]
Node 3: neighbors [1]
```

### Edge list

Just list all $(u, v)$ pairs. Compact for serialization; slow for neighbor lookup.

```python
edge_list = [(0,1),(0,2),(1,2),(1,3)]
print(f"Edges: {edge_list}")
print(f"Nodes: {sorted(set(u for e in edge_list for u in e))}")
```

```output
Edges: [(0, 1), (0, 2), (1, 2), (1, 3)]
Nodes: [0, 1, 2, 3]
```

## NetworkX: graphs in Python

NetworkX is the standard Python library for graph manipulation, analysis, and small-to-medium scale experiments.

```python
import networkx as nx

G = nx.karate_club_graph()   # Zachary's karate club: 34 nodes, 78 edges
print(f"Nodes: {G.number_of_nodes()}, Edges: {G.number_of_edges()}")
print(f"Avg degree: {sum(d for _, d in G.degree()) / G.number_of_nodes():.2f}")
print(f"Is connected: {nx.is_connected(G)}")

# Node attributes
G.nodes[0]['club']   # 'Mr. Hi' or 'Officer'
clubs = [G.nodes[n]['club'] for n in G.nodes()]
print(f"Unique clubs: {set(clubs)}")
```

```output
Nodes: 34, Edges: 78
Avg degree: 4.59
Is connected: True
Unique clubs: {'Mr. Hi', 'Officer'}
```

```python
# Adjacency via scipy sparse matrix
import scipy.sparse as sp

A_sparse = nx.to_scipy_sparse_array(G, format='csr')
print(f"Sparse matrix shape: {A_sparse.shape}")
print(f"Stored entries: {A_sparse.nnz}  (vs {34*34} for dense)")
print(f"Sparsity: {1 - A_sparse.nnz/(34*34):.4f}")
```

```output
Sparse matrix shape: (34, 34)
Stored entries: 156  (vs 1156 for dense)
Sparsity: 0.8650
```

## The three graph ML tasks

**Node classification** — label each node. Given a citation network (papers = nodes, citations = edges, bag-of-words features on nodes), classify each paper's research area. The key: a node's label often correlates with its neighborhood's labels (homophily).

**Link prediction** — predict which edges will appear. Given a social network at time $t$, predict friendships at time $t+1$. Used in recommendation (user–item bipartite graph).

**Graph classification** — label an entire graph. Given a molecule (atoms = nodes, bonds = edges), predict toxicity or solubility. Requires aggregating node information into a single vector.

```python
# Benchmark datasets you'll encounter
datasets = {
    'Cora':       {'nodes': 2708,  'edges': 10556,  'task': 'node classification', 'classes': 7},
    'Citeseer':   {'nodes': 3327,  'edges': 9228,   'task': 'node classification', 'classes': 6},
    'MUTAG':      {'nodes': 3371,  'edges': 7442,   'task': 'graph classification','classes': 2},
    'ogbn-arxiv': {'nodes': 169343,'edges': 1166243,'task': 'node classification', 'classes': 40},
}
for name, info in datasets.items():
    print(f"{name:12s} | {info['nodes']:7d} nodes | {info['edges']:8d} edges | {info['task']}")
```

```output
Cora         |    2708 nodes |    10556 edges | node classification
Citeseer     |    3327 nodes |     9228 edges | node classification
MUTAG        |    3371 nodes |     7442 edges | graph classification
ogbn-arxiv   |  169343 nodes |  1166243 edges | node classification
```

## Why graph ML is hard

Standard ML assumes i.i.d. samples. Graph data violates this: nodes are connected, so their labels are not independent. The challenge is to exploit the connectivity signal without leaking labels.

Three approaches historically: (1) hand-craft node features from graph topology (this chapter, ch. 2); (2) learn embeddings by "walking" the graph (ch. 3); (3) learn end-to-end with graph neural networks (ch. 4–10).

## Exercise

The Karate Club graph has 34 nodes. Compute: (1) the number of triangles in the graph, (2) the global clustering coefficient, and (3) which node has the highest degree.

<details>
<summary>Solution</summary>

```python
import networkx as nx

G = nx.karate_club_graph()

triangles = sum(nx.triangles(G).values()) // 3
clustering = nx.average_clustering(G)
max_degree_node = max(G.degree(), key=lambda x: x[1])

print(f"Triangles: {triangles}")
print(f"Global clustering coefficient: {clustering:.4f}")
print(f"Highest degree node: {max_degree_node[0]} with degree {max_degree_node[1]}")
```

Triangles count shared connections (friends of friends who are also friends). The clustering coefficient near 0.57 confirms strong local clustering typical of social networks — most of node 0's neighbors know each other.
</details>
