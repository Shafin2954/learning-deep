## Centrality: measuring node importance

Before learned embeddings existed, practitioners hand-crafted features from graph topology. **Centrality** measures quantify how "important" a node is — but importance means different things in different contexts.

## Degree centrality

The simplest measure: how many neighbors does a node have?

$$C_D(v) = \frac{\deg(v)}{N - 1}$$

Normalized by $N-1$ so values lie in $[0,1]$ regardless of graph size. High degree = locally influential. In social networks, high-degree nodes are hubs.

```python
import networkx as nx

G = nx.karate_club_graph()
deg_centrality = nx.degree_centrality(G)

# Top 5 by degree
top5 = sorted(deg_centrality.items(), key=lambda x: -x[1])[:5]
print("Top 5 by degree centrality:")
for node, score in top5:
    print(f"  Node {node:2d}: {score:.4f}  (degree {G.degree(node)})")
```

```output
Top 5 by degree centrality:
  Node  0: 0.4848  (degree 16)
  Node 33: 0.3636  (degree 12)
  Node  2: 0.3030  (degree 10)
  Node  1: 0.2727  (degree  9)
  Node  3: 0.1818  (degree  6)
```

## Betweenness centrality

How often does a node sit on the shortest path between two other nodes?

$$C_B(v) = \sum_{s \neq v \neq t} \frac{\sigma_{st}(v)}{\sigma_{st}}$$

where $\sigma_{st}$ is the number of shortest paths from $s$ to $t$, and $\sigma_{st}(v)$ is those passing through $v$. Betweenness identifies **brokers** — nodes that bridge communities. Removing them would disconnect the graph.

```python
betweenness = nx.betweenness_centrality(G, normalized=True)
top5_b = sorted(betweenness.items(), key=lambda x: -x[1])[:5]
print("Top 5 by betweenness centrality:")
for node, score in top5_b:
    print(f"  Node {node:2d}: {score:.4f}")
```

```output
Top 5 by betweenness centrality:
  Node  0: 0.4376
  Node 33: 0.3040
  Node 32: 0.1454
  Node  2: 0.1435
  Node  1: 0.1054
```

## Closeness centrality

How close is a node to all others (average inverse shortest path)?

$$C_C(v) = \frac{N-1}{\sum_{u \neq v} d(v, u)}$$

High closeness → information from this node reaches the rest of the network quickly.

```python
closeness = nx.closeness_centrality(G)
top5_c = sorted(closeness.items(), key=lambda x: -x[1])[:5]
print("Top 5 by closeness centrality:")
for node, score in top5_c:
    print(f"  Node {node:2d}: {score:.4f}")
```

```output
Top 5 by closeness centrality:
  Node  0: 0.5689
  Node  2: 0.5593
  Node 33: 0.5503
  Node 31: 0.5155
  Node  8: 0.5155
```

## PageRank

Google's original web-ranking algorithm. A random surfer follows edges uniformly; with probability $1-\alpha$ they teleport to a random node. The stationary distribution is PageRank:

$$r_v = \frac{1-\alpha}{N} + \alpha \sum_{u \in \mathcal{N}_{in}(v)} \frac{r_u}{\deg_{out}(u)}$$

The damping factor $\alpha = 0.85$ is standard. PageRank is iteratively computed: start with uniform $r$, apply the update until convergence.

```python
pr = nx.pagerank(G, alpha=0.85, max_iter=100)
top5_pr = sorted(pr.items(), key=lambda x: -x[1])[:5]
print("Top 5 by PageRank:")
for node, score in top5_pr:
    print(f"  Node {node:2d}: {score:.4f}")
```

```output
Top 5 by PageRank:
  Node  0: 0.0970
  Node 33: 0.0718
  Node  2: 0.0571
  Node  1: 0.0529
  Node  3: 0.0366
```

### Eigenvector centrality

A node is important if its neighbors are important:

$$C_E(v) = \frac{1}{\lambda} \sum_{u \in \mathcal{N}(v)} C_E(u)$$

This is just the principal eigenvector of the adjacency matrix. PageRank generalizes this to directed graphs with teleportation.

## Clustering coefficient

The fraction of a node's neighbors that are also connected to each other:

$$C_L(v) = \frac{2 \cdot |\text{triangles through } v|}{\deg(v)(\deg(v)-1)}$$

```python
clustering = nx.clustering(G)
avg_cc = nx.average_clustering(G)

# Distribution
import numpy as np
cc_values = list(clustering.values())
print(f"Average clustering coefficient: {avg_cc:.4f}")
print(f"Nodes with CC=1.0: {sum(1 for c in cc_values if c == 1.0)}")
print(f"Nodes with CC=0.0: {sum(1 for c in cc_values if c == 0.0)}")
print(f"CC percentiles: 25th={np.percentile(cc_values,25):.3f}, 75th={np.percentile(cc_values,75):.3f}")
```

```output
Average clustering coefficient: 0.5706
Nodes with CC=1.0: 7
Nodes with CC=0.0: 0
CC percentiles: 25th=0.333, 75th=0.750
```

## Comparing centralities

Different centralities identify different kinds of influence. Compare them side by side:

```python
import pandas as pd

data = {
    'degree': deg_centrality,
    'betweenness': betweenness,
    'closeness': closeness,
    'pagerank': pr,
    'clustering': clustering,
}
df = pd.DataFrame(data)
print(df.sort_values('degree', ascending=False).head(8).round(4))
```

## Why hand-crafted features fall short

These features work on a single, fixed graph but fail to generalize:
- They don't transfer across graphs (node 5's degree in graph A means nothing in graph B).
- They don't incorporate node **content** features (text, image, numeric attributes).
- Computing betweenness is $O(NE)$ — prohibitive for million-node graphs.
- They treat all nodes identically by structure; two nodes with the same degree are indistinguishable.

Graph neural networks (ch. 4–8) learn features that combine topology and content, generalize across graphs, and scale via message passing.

import networkx as nx

G = nx.karate_club_graph()

## Exercise

The Karate Club graph split into two communities (the "Mr. Hi" group and the "Officer" group) after a conflict. Using only graph topology (no club labels), identify the two key nodes most likely to be the community leaders — one per community.

<details>
<summary>Solution</summary>

```python
import networkx as nx

G = nx.karate_club_graph()
pr = nx.pagerank(G, alpha=0.85)
clubs = nx.get_node_attributes(G, 'club')

# Find highest PageRank node per club (using ground truth to verify, not to find)
by_club = {'Mr. Hi': [], 'Officer': []}
for node, score in pr.items():
    by_club[clubs[node]].append((node, score))

for club, nodes in by_club.items():
    leader = max(nodes, key=lambda x: x[1])
    print(f"{club}: Node {leader[0]} (PageRank {leader[1]:.4f})")
```

Node 0 leads "Mr. Hi" and node 33 leads "Officer" — exactly the instructors who split the club. PageRank identifies them purely from the link structure. This is the power of centrality as a feature even before any machine learning.
</details>
