## The MDP formalism

A **Markov Decision Process** (MDP) is the standard mathematical framework for sequential decision-making. It's defined by the tuple $(S, A, P, R, \gamma)$:

- $S$: state space
- $A$: action space
- $P(s' | s, a)$: transition probability (dynamics model)
- $R(s, a, s')$: reward function (often just $R(s, a)$)
- $\gamma \in [0,1)$: discount factor

The transition dynamics $P(s'|s,a)$ define how the world responds to actions. In model-free RL you don't know $P$ and must learn from samples. In model-based RL or dynamic programming, you have $P$ and can plan.

## Policies

A **policy** $\pi$ maps states to action probabilities: $\pi(a|s) = P(A_t = a | S_t = s)$. A **deterministic policy** maps each state to exactly one action: $\pi(s) = a$.

The goal: find the policy $\pi^*$ that maximizes expected return $G_t$ from every starting state.

## Value functions

The **state-value function** $v_\pi(s)$ is the expected return when starting in state $s$ and following policy $\pi$:

$$
v_\pi(s) = \mathbb{E}_\pi[G_t | S_t = s] = \mathbb{E}_\pi\left[\sum_{k=0}^\infty \gamma^k R_{t+k+1} \,\Big|\, S_t = s\right]
$$

The **action-value function** $q_\pi(s, a)$ is the expected return of taking action $a$ in state $s$, then following $\pi$:

$$
q_\pi(s, a) = \mathbb{E}_\pi[G_t | S_t = s, A_t = a]
$$

The relationship: $v_\pi(s) = \sum_a \pi(a|s)\, q_\pi(s, a)$.

## Bellman equations

The key insight: value functions satisfy recursive consistency conditions.

### Bellman expectation equation for $v_\pi$

$$
v_\pi(s) = \sum_a \pi(a|s) \sum_{s', r} p(s', r | s, a)\bigl[r + \gamma\, v_\pi(s')\bigr]
$$

In words: the value of a state equals the expected immediate reward plus discounted value of the next state, averaged over actions (under $\pi$) and transitions (under $P$).

### Bellman optimality equations

The **optimal value functions** $v_*(s) = \max_\pi v_\pi(s)$ and $q_*(s,a) = \max_\pi q_\pi(s,a)$ satisfy:

$$
v_*(s) = \max_a \sum_{s', r} p(s', r | s, a)\bigl[r + \gamma\, v_*(s')\bigr]
$$

$$
q_*(s, a) = \sum_{s', r} p(s', r | s, a)\bigl[r + \gamma \max_{a'} q_*(s', a')\bigr]
$$

Once you have $q_*$, the optimal policy is simply greedy: $\pi^*(s) = \arg\max_a q_*(s, a)$.

## Gridworld example

A 4×4 gridworld has 16 states, 4 actions (up/down/left/right), transitions that stay in place at edges, and a reward of −1 per step (goal: reach terminal states quickly).

```python
import numpy as np

# 4x4 gridworld: states 0-15, terminal = {0, 15}
N_STATES = 16
ACTIONS = ['U', 'D', 'L', 'R']
GAMMA = 1.0

def next_state(s, a):
    row, col = divmod(s, 4)
    if a == 0: row = max(row-1, 0)      # Up
    elif a == 1: row = min(row+1, 3)    # Down
    elif a == 2: col = max(col-1, 0)    # Left
    elif a == 3: col = min(col+1, 3)    # Right
    return row * 4 + col

# Policy evaluation for uniform random policy (each action prob 0.25)
V = np.zeros(N_STATES)
terminal = {0, 15}

for sweep in range(200):
    delta = 0
    for s in range(N_STATES):
        if s in terminal:
            continue
        v_new = 0
        for a in range(4):
            s_prime = next_state(s, a)
            reward = 0 if s_prime in terminal else -1
            v_new += 0.25 * (reward + GAMMA * V[s_prime])
        delta = max(delta, abs(v_new - V[s]))
        V[s] = v_new
    if delta < 1e-6:
        print(f"Converged after {sweep+1} sweeps")
        break

print("Value function (random policy):")
print(np.round(V.reshape(4, 4), 1))
```

```output
Converged after 173 sweeps
Value function (random policy):
[[ 0.  -14.  -20.  -22.]
 [-14.  -18.  -20.  -20.]
 [-20.  -20.  -18.  -14.]
 [-22.  -20.  -14.   0.]]
```

States near the terminals have values close to 0 (few steps to escape). Corner states far from terminals have values around −22 (many expected steps).

### Interpreting the value matrix

The values tell you: if you follow the random policy starting from state $s$, you expect to take $|v_\pi(s)|$ steps before reaching a terminal (since each step costs −1 and $\gamma=1$).

## Try it Yourself

```python
import numpy as np

# Build the full transition matrix P and reward R for the gridworld
# then verify that V satisfies the Bellman equation: V = R + γ P V

N_STATES = 16
terminal = {0, 15}
GAMMA = 1.0

def next_state(s, a):
    row, col = divmod(s, 4)
    if a == 0: row = max(row-1, 0)
    elif a == 1: row = min(row+1, 3)
    elif a == 2: col = max(col-1, 0)
    elif a == 3: col = min(col+1, 3)
    return row * 4 + col

# Build P (16x16) and R (16,) for uniform random policy
P = np.zeros((N_STATES, N_STATES))
R = np.zeros(N_STATES)
for s in range(N_STATES):
    if s in terminal:
        P[s, s] = 1.0
        continue
    for a in range(4):
        sp = next_state(s, a)
        P[s, sp] += 0.25
        if sp not in terminal:
            R[s] += 0.25 * (-1)

# Solve V = (I - γP)^{-1} R via matrix inversion
V_exact = np.linalg.solve(np.eye(N_STATES) - GAMMA * P, R)
print("Exact solution (matrix inverse):")
print(np.round(V_exact.reshape(4, 4), 1))
```

```output
Exact solution (matrix inverse):
[[ 0.  -14.  -20.  -22.]
 [-14.  -18.  -20.  -20.]
 [-20.  -20.  -18.  -14.]
 [-22.  -20.  -14.   0.]]
```
