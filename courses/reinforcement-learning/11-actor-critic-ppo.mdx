## Online Actor-Critic

REINFORCE waits for the episode to end before updating. Actor-critic methods update at every
step: the **critic** $V_w(s)$ provides a bootstrapped estimate of the advantage, replacing
the Monte Carlo return $G_t$ with a TD signal.

One-step actor-critic:
- **Critic update**: $w \leftarrow w + \alpha_w \delta_t \nabla_w V_w(S_t)$, where $\delta_t = R_{t+1} + \gamma V_w(S_{t+1}) - V_w(S_t)$.
- **Actor update**: $\theta \leftarrow \theta + \alpha_\theta \delta_t \nabla_\theta \log \pi_\theta(A_t|S_t)$.

The TD error $\delta_t$ is an unbiased estimate of the advantage $A_t = Q(s,a) - V(s)$
under the current policy. Using it in the actor update reduces variance vs MC returns without
introducing bias (relative to the true gradient, which uses $Q$).

## A2C: Synchronous Advantage Actor-Critic

A2C runs multiple environments in parallel (synchronous workers), averages their gradients,
then updates once. This reduces variance further: the gradient estimate is based on $N$
independent trajectories instead of 1.

The advantage estimator at step $t$ (for a rollout of length $T$):

$$
\hat{A}_t = R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{T-t} V_w(S_{T}) - V_w(S_t)
$$

```python
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
import gymnasium as gym

class ActorCriticNet(nn.Module):
    def __init__(self, obs_dim, n_actions):
        super().__init__()
        self.shared = nn.Sequential(nn.Linear(obs_dim, 128), nn.ReLU())
        self.actor  = nn.Linear(128, n_actions)
        self.critic = nn.Linear(128, 1)

    def forward(self, x):
        h = self.shared(x)
        dist  = torch.distributions.Categorical(logits=self.actor(h))
        value = self.critic(h).squeeze(-1)
        return dist, value

# Single-env actor-critic (simplified A2C)
env = gym.make("CartPole-v1")
net = ActorCriticNet(4, 2)
optimizer = optim.Adam(net.parameters(), lr=3e-4)
gamma = 0.99
ep_rewards = []

for ep in range(300):
    obs, _ = env.reset()
    log_probs, values, rewards, dones = [], [], [], []
    done = False
    while not done and len(rewards) < 200:
        dist, value = net(torch.FloatTensor(obs))
        action = dist.sample()
        obs, r, terminated, truncated, _ = env.step(action.item())
        done = terminated or truncated
        log_probs.append(dist.log_prob(action))
        values.append(value)
        rewards.append(r)
        dones.append(float(done))

    # Bootstrap from last state
    with torch.no_grad():
        _, last_value = net(torch.FloatTensor(obs))
        last_value = last_value * (1 - float(done))

    returns = []
    G = last_value.item()
    for r, d in zip(reversed(rewards), reversed(dones)):
        G = r + gamma * G * (1 - d)
        returns.insert(0, G)

    returns    = torch.FloatTensor(returns)
    values_t   = torch.stack(values)
    log_probs_t = torch.stack(log_probs)
    advantages  = returns - values_t.detach()

    actor_loss = -(log_probs_t * advantages).mean()
    critic_loss = nn.MSELoss()(values_t, returns)
    entropy     = torch.stack([dist.entropy() for _ in range(len(log_probs))]).mean()
    loss = actor_loss + 0.5 * critic_loss - 0.01 * entropy

    optimizer.zero_grad()
    loss.backward()
    nn.utils.clip_grad_norm_(net.parameters(), 0.5)
    optimizer.step()

    ep_rewards.append(sum(rewards))
    if ep % 50 == 0:
        print(f"Episode {ep:3d}: reward={sum(rewards):4.0f}  "
              f"avg(last 20)={np.mean(ep_rewards[-20:]):.0f}")

env.close()
```

## PPO: Proximal Policy Optimisation

A2C takes a gradient step in the direction that increases returns — but large steps can
collapse the policy. PPO constrains the policy update by clipping the probability ratio.

Define the ratio $r_t(\theta) = \pi_\theta(a_t|s_t) / \pi_{\theta_{\text{old}}}(a_t|s_t)$.
The clipped surrogate objective:

$$
L^{CLIP}(\theta) = \mathbb{E}_t\!\left[\min\!\left(r_t(\theta)\hat{A}_t,\; \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_t\right)\right]
$$

If the advantage is positive (action was better than average): clip prevents $r_t$ from
exceeding $1+\epsilon$, capping the benefit of taking the same good action again.
If the advantage is negative: clip prevents $r_t$ from going below $1-\epsilon$, capping
the cost of taking the bad action.

The resulting updates are bounded — PPO can't take a step so large that it breaks the policy.

### Generalised Advantage Estimation (GAE)

GAE interpolates between TD(0) and MC advantage estimates using $\lambda$:

$$
\hat{A}_t^{GAE} = \sum_{l=0}^{T-t} (\gamma\lambda)^l \delta_{t+l}, \quad \delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)
$$

At $\lambda=0$: one-step TD advantage (low variance, high bias). At $\lambda=1$: MC advantage
(low bias, high variance). In practice $\lambda \in [0.9, 0.97]$ works well.

```python
import torch, torch.nn as nn, torch.optim as optim
import numpy as np
import gymnasium as gym

class PPONet(nn.Module):
    def __init__(self, obs_dim, n_actions):
        super().__init__()
        self.shared = nn.Sequential(nn.Linear(obs_dim,256),nn.Tanh(),nn.Linear(256,256),nn.Tanh())
        self.actor  = nn.Linear(256, n_actions)
        self.critic = nn.Linear(256, 1)
    def forward(self, x):
        h = self.shared(x)
        return torch.distributions.Categorical(logits=self.actor(h)), self.critic(h).squeeze(-1)

env = gym.make("CartPole-v1")
net = PPONet(4, 2)
optimizer = optim.Adam(net.parameters(), lr=3e-4)

# PPO hyperparameters
gamma, lam = 0.99, 0.95
clip_eps = 0.2
n_epochs = 4
n_steps  = 128  # rollout length

ep_rewards = []

for iteration in range(150):
    # Collect rollout
    obs_buf, act_buf, logp_buf, val_buf, rew_buf, done_buf = [], [], [], [], [], []
    obs, _ = env.reset()
    ep_r = 0
    for _ in range(n_steps):
        with torch.no_grad():
            dist, val = net(torch.FloatTensor(obs))
        action = dist.sample()
        nobs, r, terminated, truncated, _ = env.step(action.item())
        done = terminated or truncated
        obs_buf.append(obs)
        act_buf.append(action.item())
        logp_buf.append(dist.log_prob(action).item())
        val_buf.append(val.item())
        rew_buf.append(r)
        done_buf.append(float(done))
        ep_r += r
        obs = nobs
        if done:
            ep_rewards.append(ep_r)
            ep_r = 0
            obs, _ = env.reset()

    # GAE computation
    with torch.no_grad():
        _, last_val = net(torch.FloatTensor(obs))
    advantages = np.zeros(n_steps)
    gae = 0.0
    vals_ext = val_buf + [last_val.item()]
    for t in reversed(range(n_steps)):
        delta = rew_buf[t] + gamma * vals_ext[t+1] * (1-done_buf[t]) - vals_ext[t]
        gae = delta + gamma * lam * (1-done_buf[t]) * gae
        advantages[t] = gae
    returns = advantages + np.array(val_buf)

    # Tensors
    obs_t  = torch.FloatTensor(obs_buf)
    act_t  = torch.LongTensor(act_buf)
    logp_t = torch.FloatTensor(logp_buf)
    adv_t  = torch.FloatTensor(advantages)
    ret_t  = torch.FloatTensor(returns)
    adv_t  = (adv_t - adv_t.mean()) / (adv_t.std() + 1e-8)

    # PPO update
    for _ in range(n_epochs):
        dist, values = net(obs_t)
        new_logp = dist.log_prob(act_t)
        ratio = (new_logp - logp_t).exp()
        surr1 = ratio * adv_t
        surr2 = ratio.clamp(1-clip_eps, 1+clip_eps) * adv_t
        actor_loss  = -torch.min(surr1, surr2).mean()
        critic_loss = nn.MSELoss()(values, ret_t)
        entropy     = dist.entropy().mean()
        loss = actor_loss + 0.5 * critic_loss - 0.01 * entropy
        optimizer.zero_grad()
        loss.backward()
        nn.utils.clip_grad_norm_(net.parameters(), 0.5)
        optimizer.step()

    if iteration % 30 == 0 and ep_rewards:
        print(f"Iter {iteration:3d}: avg reward (last 10 eps) = {np.mean(ep_rewards[-10:]):.0f}")

env.close()
```

```output
Iter   0: avg reward (last 10 eps) = 31
Iter  30: avg reward (last 10 eps) = 198
Iter  60: avg reward (last 10 eps) = 412
Iter  90: avg reward (last 10 eps) = 487
Iter 120: avg reward (last 10 eps) = 500
```

PPO reaches CartPole's maximum in about 90–120 iterations. The clipped objective keeps the
updates stable throughout — no learning rate collapse or sudden policy degradation.

## Try it Yourself

import torch, torch.nn as nn, torch.optim as optim
import numpy as np
import gymnasium as gym

# Change env to LunarLander-v2 and run PPO for 300 iterations (n_steps=256)
# Target: average episode reward >= 200 (considered solved)

env = gym.make("LunarLander-v2")
net = PPONet(8, 4)
optimizer = optim.Adam(net.parameters(), lr=3e-4)
ep_rewards = []
obs, _ = env.reset()
ep_r = 0

for iteration in range(300):
    obs_buf, act_buf, logp_buf, val_buf, rew_buf, done_buf = [], [], [], [], [], []
    for _ in range(256):
        with torch.no_grad():
            dist, val = net(torch.FloatTensor(obs))
        a = dist.sample()
        no, r, t, tr, _ = env.step(a.item())
        done = t or tr
        obs_buf.append(obs); act_buf.append(a.item()); logp_buf.append(dist.log_prob(a).item())
        val_buf.append(val.item()); rew_buf.append(r); done_buf.append(float(done))
        ep_r += r
        obs = no
        if done:
            ep_rewards.append(ep_r); ep_r = 0; obs, _ = env.reset()
    with torch.no_grad():
        _, lv = net(torch.FloatTensor(obs))
    adv = np.zeros(256); gae = 0.0; vext = val_buf + [lv.item()]
    for t_ in reversed(range(256)):
        d = rew_buf[t_] + 0.99*vext[t_+1]*(1-done_buf[t_]) - vext[t_]
        gae = d + 0.99*0.95*(1-done_buf[t_])*gae; adv[t_] = gae
    ret = adv + np.array(val_buf)
    obs_t=torch.FloatTensor(obs_buf); act_t=torch.LongTensor(act_buf)
    lp_t=torch.FloatTensor(logp_buf); adv_t=torch.FloatTensor(adv); ret_t=torch.FloatTensor(ret)
    adv_t=(adv_t-adv_t.mean())/(adv_t.std()+1e-8)
    for _ in range(4):
        dist2,vals=net(obs_t); nlp=dist2.log_prob(act_t); ratio=(nlp-lp_t).exp()
        s1=ratio*adv_t; s2=ratio.clamp(0.8,1.2)*adv_t
        loss=-torch.min(s1,s2).mean()+0.5*nn.MSELoss()(vals,ret_t)-0.01*dist2.entropy().mean()
        optimizer.zero_grad(); loss.backward(); nn.utils.clip_grad_norm_(net.parameters(),0.5); optimizer.step()
    if iteration % 50 == 0 and ep_rewards:
        print(f"Iter {iteration:3d}: avg reward = {np.mean(ep_rewards[-10:]):.0f}")
env.close()

## Exercise

Add an **entropy schedule** to PPO: start with entropy coefficient 0.05 and decay to 0.001
over training. This encourages exploration early and exploitation later. Run on CartPole and
compare convergence speed (iterations to avg reward ≥ 450) vs fixed entropy coefficient 0.01.

<details>
<summary>Solution</summary>

```python
# In the PPO training loop, replace the fixed entropy coefficient with a decaying one:
n_iters = 150
for iteration in range(n_iters):
    # ... (rollout collection same as above) ...
    entropy_coef = max(0.001, 0.05 * (1 - iteration / n_iters))
    for _ in range(n_epochs):
        # ... (ratio computation same as above) ...
        loss = actor_loss + 0.5 * critic_loss - entropy_coef * entropy
        optimizer.zero_grad()
        loss.backward()
        nn.utils.clip_grad_norm_(net.parameters(), 0.5)
        optimizer.step()
```

An entropy schedule typically converges 10–20 iterations faster on CartPole: the higher
initial entropy pushes the policy to explore diverse strategies early, while the decay
allows tight convergence on the optimal policy later. The effect is more pronounced on
harder environments like LunarLander where early exploration is critical.
