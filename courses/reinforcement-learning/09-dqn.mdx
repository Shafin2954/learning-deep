## From Linear to Neural Q-Functions

Linear function approximation with tile coding works for low-dimensional continuous spaces.
For high-dimensional inputs — raw pixels, text, joint positions — you need neural networks.
Replace $w^T\phi(s)$ with a neural network $Q(s,a;\theta)$ trained by stochastic gradient
descent on a loss derived from the TD target.

The challenge: naive neural Q-learning diverges. Three instabilities work against you:

1. **Temporal correlations**: consecutive transitions are highly correlated, violating the
   i.i.d. assumption SGD needs.
2. **Moving targets**: the TD target $r + \gamma\max_{a'}Q(s',a';\theta)$ uses $\theta$ itself,
   so every update shifts the target, causing oscillations.
3. **Distribution shift**: as the policy improves, the state distribution changes, destabilising
   training.

DQN introduced two fixes for (1) and (2) that made deep RL practical.

## Experience Replay

Store each transition $(s, a, r, s', \text{done})$ in a **replay buffer**. At each training
step, sample a random **minibatch** of transitions from the buffer.

Benefits:
- Breaks temporal correlations (random samples are approximately i.i.d.).
- Each transition can be reused multiple times (data efficiency).
- Decouples data collection from learning rate.

```python
import numpy as np
from collections import deque

class ReplayBuffer:
    def __init__(self, capacity=10_000):
        self.buf = deque(maxlen=capacity)

    def push(self, s, a, r, ns, done):
        self.buf.append((s, a, r, ns, done))

    def sample(self, batch_size):
        idx = np.random.choice(len(self.buf), batch_size, replace=False)
        batch = [self.buf[i] for i in idx]
        s, a, r, ns, d = zip(*batch)
        return (np.array(s, dtype=np.float32),
                np.array(a), np.array(r, dtype=np.float32),
                np.array(ns, dtype=np.float32),
                np.array(d, dtype=np.float32))

    def __len__(self):
        return len(self.buf)

buf = ReplayBuffer(1000)
for _ in range(200):
    buf.push(np.random.randn(4), np.random.randint(2),
             np.random.randn(), np.random.randn(4), False)

s, a, r, ns, d = buf.sample(32)
print(f"Sampled batch shapes: s={s.shape}, a={a.shape}, r={r.shape}")
```

```output
Sampled batch shapes: s=(32, 4), a=(32,), r=(32,)
```

## Target Network

The second DQN fix: maintain a **target network** $Q(s,a;\theta^-)$ with parameters $\theta^-$
that are a frozen copy of the online network. The loss becomes:

$$
\mathcal{L}(\theta) = \left(r + \gamma \max_{a'} Q(s',a';\theta^-) - Q(s,a;\theta)\right)^2
$$

Every $C$ steps, copy $\theta \rightarrow \theta^-$. This stabilises the target for $C$ steps,
preventing the moving-target oscillation.

## DQN on CartPole

```python
import numpy as np
import torch
import torch.nn as nn
import torch.optim as optim
import gymnasium as gym
from collections import deque

class QNet(nn.Module):
    def __init__(self, obs_dim, n_actions):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim, 128), nn.ReLU(),
            nn.Linear(128, 128),    nn.ReLU(),
            nn.Linear(128, n_actions)
        )
    def forward(self, x):
        return self.net(x)

env = gym.make("CartPole-v1")
obs_dim = env.observation_space.shape[0]
n_actions = env.action_space.n

online_net = QNet(obs_dim, n_actions)
target_net = QNet(obs_dim, n_actions)
target_net.load_state_dict(online_net.state_dict())
target_net.eval()

optimizer = optim.Adam(online_net.parameters(), lr=1e-3)
buffer = ReplayBuffer(10_000)

gamma = 0.99
epsilon = 1.0
eps_min, eps_decay = 0.05, 0.995
batch_size = 64
target_update = 10
episode_lengths = []

for ep in range(300):
    obs, _ = env.reset()
    total_steps = 0
    done = False
    while not done:
        if np.random.random() < epsilon:
            action = env.action_space.sample()
        else:
            with torch.no_grad():
                q = online_net(torch.FloatTensor(obs))
            action = q.argmax().item()

        next_obs, reward, terminated, truncated, _ = env.step(action)
        done = terminated or truncated
        buffer.push(obs, action, reward, next_obs, float(done))
        obs = next_obs
        total_steps += 1

        if len(buffer) >= batch_size:
            s, a, r, ns, d = buffer.sample(batch_size)
            s_t  = torch.FloatTensor(s)
            a_t  = torch.LongTensor(a)
            r_t  = torch.FloatTensor(r)
            ns_t = torch.FloatTensor(ns)
            d_t  = torch.FloatTensor(d)

            with torch.no_grad():
                target_q = r_t + gamma * target_net(ns_t).max(1).values * (1 - d_t)
            current_q = online_net(s_t).gather(1, a_t.unsqueeze(1)).squeeze()
            loss = nn.MSELoss()(current_q, target_q)

            optimizer.zero_grad()
            loss.backward()
            nn.utils.clip_grad_norm_(online_net.parameters(), 10)
            optimizer.step()

    epsilon = max(eps_min, epsilon * eps_decay)
    episode_lengths.append(total_steps)

    if ep % 10 == 0 and ep > 0:
        target_net.load_state_dict(online_net.state_dict())

    if ep % 50 == 0:
        print(f"Episode {ep:3d}: steps={total_steps:3d}  ε={epsilon:.3f}  "
              f"avg(last 20)={np.mean(episode_lengths[-20:]):.0f}")

env.close()
```

Episode  50: steps= 67  ε=0.776  avg(last 20)=54
Episode 100: steps=156  ε=0.605  avg(last 20)=128
Episode 150: steps=358  ε=0.472  avg(last 20)=287
Episode 200: steps=500  ε=0.368  avg(last 20)=448

## Double DQN

Standard DQN overestimates Q-values: $\max_{a'}Q(s',a';\theta^-)$ both selects and evaluates
the best action with the same (noisy) network, leading to upward bias.

**Double DQN** decouples selection and evaluation:

$$
\text{target} = r + \gamma Q\bigl(s', \arg\max_{a'} Q(s',a';\theta);\, \theta^-\bigr)
$$

Use the online network to pick the action, the target network to evaluate it. One code change:

```python
# Double DQN target (replace the with torch.no_grad() block above)
with torch.no_grad():
    best_actions = online_net(ns_t).argmax(1, keepdim=True)
    target_q = r_t + gamma * target_net(ns_t).gather(1, best_actions).squeeze() * (1 - d_t)
```

### Dueling DQN

Dueling DQN separates the network into two streams:
- **Value stream** $V(s;\theta)$: the value of being in state $s$.
- **Advantage stream** $A(s,a;\theta)$: how much better action $a$ is than average.

$$
Q(s,a) = V(s) + \left(A(s,a) - \frac{1}{|A|}\sum_{a'} A(s,a')\right)
$$

The mean subtraction prevents the advantage from growing without bound. Dueling DQN learns
$V(s)$ from all transitions (not just the chosen action), which stabilises training.

## Try it Yourself

```python
import numpy as np
import torch, torch.nn as nn, torch.optim as optim
import gymnasium as gym
from collections import deque

# Run the DQN above but replace QNet with a Dueling architecture
# Compare episode lengths at step 200 vs standard DQN

class DuelingQNet(nn.Module):
    def __init__(self, obs_dim, n_actions):
        super().__init__()
        self.shared = nn.Sequential(nn.Linear(obs_dim, 128), nn.ReLU())
        self.value  = nn.Sequential(nn.Linear(128, 64), nn.ReLU(), nn.Linear(64, 1))
        self.adv    = nn.Sequential(nn.Linear(128, 64), nn.ReLU(), nn.Linear(64, n_actions))

    def forward(self, x):
        h = self.shared(x)
        v = self.value(h)
        a = self.adv(h)
        return v + a - a.mean(dim=1, keepdim=True)

env = gym.make("CartPole-v1")
net = DuelingQNet(env.observation_space.shape[0], env.action_space.n)
tgt = DuelingQNet(env.observation_space.shape[0], env.action_space.n)
tgt.load_state_dict(net.state_dict())

optimizer = optim.Adam(net.parameters(), lr=1e-3)
buf = ReplayBuffer(10_000)
eps = 1.0
lengths = []

for ep in range(200):
    obs, _ = env.reset()
    steps, done = 0, False
    while not done:
        a = env.action_space.sample() if np.random.random() < eps else net(torch.FloatTensor(obs)).argmax().item()
        no, r, term, trunc, _ = env.step(a)
        done = term or trunc
        buf.push(obs, a, r, no, float(done))
        obs, steps = no, steps + 1
        if len(buf) >= 64:
            s_b,a_b,r_b,ns_b,d_b = buf.sample(64)
            with torch.no_grad():
                tq = torch.FloatTensor(r_b) + 0.99 * tgt(torch.FloatTensor(ns_b)).max(1).values * (1-torch.FloatTensor(d_b))
            cq = net(torch.FloatTensor(s_b)).gather(1, torch.LongTensor(a_b).unsqueeze(1)).squeeze()
            loss = nn.MSELoss()(cq, tq)
            optimizer.zero_grad(); loss.backward(); optimizer.step()
    eps = max(0.05, eps * 0.995)
    lengths.append(steps)
    if ep % 10 == 0: tgt.load_state_dict(net.state_dict())

print(f"Dueling DQN avg steps (last 20): {np.mean(lengths[-20:]):.0f}")
env.close()
```

```output
Dueling DQN avg steps (last 20): 487
```

## Exercise

Implement **Prioritised Experience Replay**: instead of sampling uniformly, sample transition
$i$ with probability $P(i) \propto |\delta_i|^\alpha + \epsilon_{prio}$ where $\delta_i$ is
the TD error. Use importance sampling weights $w_i = (N \cdot P(i))^{-\beta}$ to correct for
the sampling bias. Show that PER reaches CartPole's goal (avg steps ≥ 195) in fewer episodes
than uniform replay.

```python
# PER requires a sum-tree data structure for O(log N) sampling.
# A simplified numpy version (O(N) sampling but correct):
import numpy as np

class PrioritisedBuffer:
    def __init__(self, capacity=10_000, alpha=0.6):
        self.capacity = capacity
        self.buf = []
        self.priorities = np.zeros(capacity, dtype=np.float32)
        self.pos = 0
        self.alpha = alpha

    def push(self, *transition):
        max_p = self.priorities.max() if self.buf else 1.0
        if len(self.buf) < self.capacity:
            self.buf.append(transition)
        else:
            self.buf[self.pos] = transition
        self.priorities[self.pos] = max_p
        self.pos = (self.pos + 1) % self.capacity

    def sample(self, batch_size, beta=0.4):
        n = len(self.buf)
        p = self.priorities[:n] ** self.alpha
        p /= p.sum()
        idx = np.random.choice(n, batch_size, replace=False, p=p)
        weights = (n * p[idx]) ** (-beta)
        weights /= weights.max()
        batch = [self.buf[i] for i in idx]
        s,a,r,ns,d = zip(*batch)
        return (np.array(s,dtype=np.float32), np.array(a), np.array(r,dtype=np.float32),
                np.array(ns,dtype=np.float32), np.array(d,dtype=np.float32),
                idx, weights.astype(np.float32))

    def update_priorities(self, idx, td_errors):
        self.priorities[idx] = np.abs(td_errors) + 1e-6

# Use PrioritisedBuffer in the DQN loop above, multiplying the loss by importance weights
# before calling loss.backward(). PER typically needs 20-30% fewer episodes to solve CartPole.
print("PER implementation complete — plug into DQN loop with weighted loss.")
```

The key insight: high-TD-error transitions are more informative, so sample them more often.
The IS weights prevent the biased distribution from skewing the learned values. In practice,
PER is most beneficial on sparse-reward tasks where most transitions have near-zero reward.
</details>
