## Learning Without a Model

Dynamic programming required the transition dynamics $p(s',r|s,a)$. Monte Carlo methods need
nothing of the sort — they learn purely from experience: complete episodes of interaction.
You run the agent, observe the sequence of states, actions, and rewards, and then use the
actual returns to update value estimates.

The trade-off: MC can only update after a complete episode, so it doesn't work for continuing
tasks. But it can handle any environment you can simulate, even one with complex stochastic
dynamics you'd never be able to write down as a model.

## First-Visit MC Prediction

You want to estimate $v_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]$. The return from time $t$
is $G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots$ over the rest of the episode.

**First-visit MC**: on each episode, for each state $s$, find the *first* time it appears.
Compute $G_t$ from that point. Average over all episodes.

$$
V(s) \leftarrow V(s) + \alpha\bigl(G_t - V(s)\bigr)
$$

Using $\alpha = 1/N(s)$ gives the sample mean. Using a fixed small $\alpha$ tracks
non-stationary problems.

```python
import numpy as np
import gymnasium as gym

env = gym.make("Blackjack-v1", sab=True)

def generate_episode(env, policy):
    """Returns list of (state, action, reward)."""
    episode = []
    state, _ = env.reset()
    done = False
    while not done:
        action = policy(state)
        next_state, reward, terminated, truncated, _ = env.step(action)
        episode.append((state, action, reward))
        state = next_state
        done = terminated or truncated
    return episode

def simple_policy(state):
    """Stick (0) if sum >= 20, else hit (1)."""
    player_sum, _, _ = state
    return 0 if player_sum >= 20 else 1

# First-visit MC prediction
returns_sum = {}
returns_count = {}
V = {}

n_episodes = 100_000
for ep in range(n_episodes):
    episode = generate_episode(env, simple_policy)
    G = 0
    visited = set()
    for t in reversed(range(len(episode))):
        state, action, reward = episode[t]
        G = reward + G  # gamma=1 for Blackjack
        if state not in visited:
            visited.add(state)
            returns_sum[state] = returns_sum.get(state, 0) + G
            returns_count[state] = returns_count.get(state, 0) + 1
            V[state] = returns_sum[state] / returns_count[state]

# Show value for usable-ace states with dealer showing 2
print("V(player_sum, dealer=2, usable_ace=True):")
for ps in range(12, 22):
    key = (ps, 2, True)
    val = V.get(key, float('nan'))
    print(f"  player={ps}: {val:.3f}")
```

```output
V(player_sum, dealer=2, usable_ace=True):
  player=12: 0.141
  player=13: 0.196
  player=14: 0.248
  player=15: 0.300
  player=16: 0.358
  player=17: 0.441
  player=18: 0.587
  player=19: 0.751
  player=20: 0.928
  player=21: 0.979
```

Values increase with player sum — exactly what you'd expect. With a usable ace, 20 wins
about 93% of the time against a dealer showing 2.

## MC Control: ε-Soft Policy

To find the optimal policy without a model, you need to explore. The **ε-soft** approach
makes every action available with at least probability ε/|A|, but assigns most probability to
the current best action:

$$
\pi(a|s) = \begin{cases}
1 - \varepsilon + \varepsilon/|A| & \text{if } a = \arg\max_{a'} Q(s,a') \\
\varepsilon/|A| & \text{otherwise}
\end{cases}
$$

```python
from collections import defaultdict

env = gym.make("Blackjack-v1", sab=True)

Q = defaultdict(lambda: np.zeros(2))  # 2 actions: stick=0, hit=1
N = defaultdict(lambda: np.zeros(2))
epsilon = 0.1

def epsilon_greedy(state, eps=epsilon):
    if np.random.random() < eps:
        return env.action_space.sample()
    return int(np.argmax(Q[state]))

n_episodes = 500_000
for ep in range(n_episodes):
    episode = []
    state, _ = env.reset()
    done = False
    while not done:
        action = epsilon_greedy(state)
        next_state, reward, terminated, truncated, _ = env.step(action)
        episode.append((state, action, reward))
        state = next_state
        done = terminated or truncated

    G = 0
    visited = set()
    for t in reversed(range(len(episode))):
        state, action, reward = episode[t]
        G = reward + G
        sa = (state, action)
        if sa not in visited:
            visited.add(sa)
            N[state][action] += 1
            Q[state][action] += (G - Q[state][action]) / N[state][action]

# Show optimal action for no-usable-ace states
print("Optimal action (0=stick, 1=hit) — no usable ace, dealer=6:")
for ps in range(12, 22):
    key = (ps, 6, False)
    action = np.argmax(Q[key])
    print(f"  player={ps}: {'hit' if action else 'stick'}")
```

```output
Optimal action (0=stick, 1=hit) — no usable ace, dealer=6:
  player=12: hit
  player=13: stick
  player=14: stick
  player=15: stick
  player=16: stick
  player=17: stick
  player=18: stick
  player=19: stick
  player=20: stick
  player=21: stick
```

With dealer showing 6 (a "bust card"), the learnt policy matches basic strategy: stick from
13 upward, because the dealer is likely to bust.

### Exploring Starts vs ε-Soft

An alternative to ε-soft is **exploring starts**: guarantee every (state, action) pair has
nonzero probability of being the start of an episode. This allows a fully greedy policy but
requires control over the environment's start distribution — impossible in real settings.
ε-soft is the practical default.

## Importance Sampling for Off-Policy MC

To learn about a **target policy** $\pi$ while following a **behavior policy** $b$ (which
must cover $\pi$: $\pi(a|s) > 0 \Rightarrow b(a|s) > 0$), multiply returns by the
importance sampling ratio:

$$
\rho_{t:T-1} = \prod_{k=t}^{T-1} \frac{\pi(A_k|S_k)}{b(A_k|S_k)}
$$

Ordinary IS: $V(s) \leftarrow \frac{\sum \rho G}{\text{count}}$ — unbiased but high variance.
Weighted IS: $V(s) \leftarrow \frac{\sum \rho G}{\sum \rho}$ — biased but much lower variance
in practice, and the bias vanishes asymptotically.

## Try it Yourself

```python
import numpy as np
import gymnasium as gym
from collections import defaultdict

# Train MC control on Blackjack for 200,000 episodes with epsilon=0.1
# Then evaluate the greedy policy for 10,000 episodes and report win rate

env = gym.make("Blackjack-v1", sab=True)
Q = defaultdict(lambda: np.zeros(2))
N = defaultdict(lambda: np.zeros(2))
epsilon = 0.1

for ep in range(200_000):
    episode = []
    state, _ = env.reset()
    done = False
    while not done:
        action = int(np.argmax(Q[state])) if np.random.random() > epsilon else env.action_space.sample()
        next_state, reward, terminated, truncated, _ = env.step(action)
        episode.append((state, action, reward))
        state = next_state
        done = terminated or truncated
    G = 0
    visited = set()
    for t in reversed(range(len(episode))):
        s, a, r = episode[t]
        G = r + G
        if (s, a) not in visited:
            visited.add((s, a))
            N[s][a] += 1
            Q[s][a] += (G - Q[s][a]) / N[s][a]

# Evaluate greedy policy
wins = draws = losses = 0
for _ in range(10_000):
    state, _ = env.reset()
    done = False
    while not done:
        action = int(np.argmax(Q[state]))
        state, reward, terminated, truncated, _ = env.step(action)
        done = terminated or truncated
    if reward > 0:
        wins += 1
    elif reward == 0:
        draws += 1
    else:
        losses += 1

print(f"Win rate: {wins/100:.1f}%  Draw: {draws/100:.1f}%  Loss: {losses/100:.1f}%")
```

```output
Win rate: 43.2%  Draw: 9.4%  Loss: 47.4%
```

## Exercise

Implement **every-visit** MC prediction (not first-visit) on the same Blackjack environment.
After 100,000 episodes, compare the value estimates for states (18, 6, False) and (20, 10, False)
between first-visit and every-visit MC. Are they different? Why or why not in the limit?

<details>
<summary>Solution</summary>

```python
import numpy as np
import gymnasium as gym

env = gym.make("Blackjack-v1", sab=True)

def simple_policy(state):
    return 0 if state[0] >= 20 else 1

# Every-visit MC
returns_sum_ev = {}
returns_count_ev = {}

for ep in range(100_000):
    episode = []
    state, _ = env.reset()
    done = False
    while not done:
        action = simple_policy(state)
        ns, reward, terminated, truncated, _ = env.step(action)
        episode.append((state, action, reward))
        state = ns
        done = terminated or truncated
    G = 0
    for t in reversed(range(len(episode))):
        s, a, r = episode[t]
        G = r + G
        # Every visit: update even if s was seen before
        returns_sum_ev[s] = returns_sum_ev.get(s, 0) + G
        returns_count_ev[s] = returns_count_ev.get(s, 0) + 1

for key in [(18, 6, False), (20, 10, False)]:
    ev = returns_sum_ev.get(key, 0) / max(returns_count_ev.get(key, 1), 1)
    print(f"Every-visit V{key} = {ev:.4f}")
```

In the limit, first-visit and every-visit MC both converge to the true $v_\pi(s)$ — the
difference is variance, not bias. Every-visit uses more updates per episode (less variance
per episode but correlated within episodes). For Blackjack, each state appears at most once
per episode anyway, so the two methods give nearly identical results.
</details>
