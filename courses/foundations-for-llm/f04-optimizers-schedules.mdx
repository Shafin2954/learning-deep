## Why the optimizer matters

Backprop (F2) gives you gradients. The *optimizer* decides how to turn gradients
into weight updates — and the **learning-rate schedule** decides how big those steps
are over time. When you fine-tune a Bengali LLM (Bengali ch. 9), almost every knob
you touch lives here: learning rate, warmup, the schedule, weight decay. Getting
these wrong is the #1 reason a fine-tune diverges or learns nothing. So understand
them, don't copy them blindly.

## SGD and the ill-conditioning problem

Plain SGD takes a step opposite the gradient:

$$
w \leftarrow w - \eta \cdot g
$$

The catch: the largest safe $\eta$ is bounded by the *steepest* direction of the loss.
But deep networks are wildly **ill-conditioned** — some directions are steep, others
nearly flat. A single $\eta$ that's safe for the steep directions is painfully slow in
the flat ones. (Verified below: on a problem with curvatures 1 and 200, SGD crawls
while adaptive methods finish.)

## Momentum: accumulate velocity

Momentum keeps a velocity vector $v$ and updates:

$$
v \leftarrow \beta v - \eta g, \qquad w \leftarrow w + v \qquad (\beta \approx 0.9)
$$

Momentum averages recent gradients, so directions where the gradient consistently
points the same way build up speed, while oscillating directions cancel out. It's a
big, cheap win over SGD on ill-conditioned losses — exactly the regime nets live in.

## Adam: a per-parameter learning rate

Adam keeps a running mean of gradients ($m$, like momentum) *and* a running mean of
squared gradients ($v$), then scales each parameter's step by $1/\sqrt{v}$. Every
parameter gets its own effective learning rate — large where gradients have been
small, small where they've been large.

### Adam update rule

$$
m \leftarrow \beta_1 m + (1 - \beta_1) g \qquad (\beta_1 \approx 0.9)
$$

$$
v \leftarrow \beta_2 v + (1 - \beta_2) g^2 \qquad (\beta_2 \approx 0.999)
$$

$$
\hat{m} = \frac{m}{1 - \beta_1^t}, \qquad \hat{v} = \frac{v}{1 - \beta_2^t} \qquad \text{(bias correction)}
$$

$$
w \leftarrow w - \eta \cdot \frac{\hat{m}}{\sqrt{\hat{v}} + \varepsilon}
$$

This adaptivity is why Adam (and its variant **AdamW**, which decouples weight decay)
is the **default optimizer for transformers**: it's robust to the per-parameter scale
differences that are everywhere in attention blocks, and it needs far less
learning-rate babysitting than SGD. Use **AdamW** in practice. For LLM fine-tuning,
typical $\eta$ is small: `1e-4` to `2e-5` for full/LoRA fine-tuning (Bengali ch. 9
gives concrete numbers).

```python
import numpy as np

# Ill-conditioned quadratic: curvatures 1 (flat) and 200 (steep). Condition # = 200.
a, b = 1.0, 200.0
def grad(w): return np.array([a*w[0], b*w[1]])
def loss(w): return 0.5*(a*w[0]**2 + b*w[1]**2)

def optimize(kind, steps=300, lr=0.009):
    w = np.array([5.0, 5.0]); v = np.zeros(2); m = np.zeros(2); s = np.zeros(2)
    for t in range(1, steps + 1):
        g = grad(w)
        if kind == "sgd":
            w = w - lr * g
        elif kind == "momentum":
            v = 0.9 * v - lr * g; w = w + v
        elif kind == "adam":
            m = 0.9*m + 0.1*g; s = 0.999*s + 0.001*g*g
            mh = m/(1 - 0.9**t); sh = s/(1 - 0.999**t)
            w = w - 0.1 * mh / (np.sqrt(sh) + 1e-8)
    return loss(w)

for k in ["sgd", "momentum", "adam"]:
    print(f"{k:9s} final loss: {optimize(k):.3e}")
# sgd ~5.5e-2 (stuck, lr capped by the steep direction);
# momentum ~5.6e-11, adam ~3.4e-10  -> both crush SGD.
```

```output
sgd       final loss: 5.509e-02
momentum  final loss: 5.604e-11
adam      final loss: 3.370e-10
```

SGD stays stuck at $5 \times 10^{-2}$; both momentum and Adam reach near-zero. On a
real transformer, where condition numbers are far worse, this gap is what separates
"trains" from "crawls."

## Warmup: don't take big steps before the optimizer knows the landscape

At step 0, Adam's running averages ($m$, $v$) are basically empty, so its step
directions are unreliable — and transformer activations are fragile early in
training. Taking a full-size step now can spike the loss to NaN. **Warmup** ramps
$\eta$ linearly from 0 to its target over the first few hundred/thousand steps. For
transformers this is not optional folklore; skipping it is a classic way to make
fine-tuning diverge.

## Linear warmup → cosine decay schedule

After warmup, you *decay* the learning rate so the model can settle into a good
minimum instead of bouncing around it. The standard recipe is **linear warmup →
cosine decay**: ramp up, then ease down following a half-cosine to $\approx 0$.

$$
\eta(t) = \begin{cases}
\eta_{\max} \cdot \dfrac{t}{T_{\text{warmup}}} & t < T_{\text{warmup}} \\[6pt]
\dfrac{\eta_{\max}}{2}\!\left(1 + \cos\!\left(\pi \cdot \dfrac{t - T_{\text{warmup}}}{T_{\text{total}} - T_{\text{warmup}}}\right)\right) & t \ge T_{\text{warmup}}
\end{cases}
$$

```python
import numpy as np
# Linear warmup -> cosine decay schedule (what you'll actually use).
def lr_at(step, base_lr=2e-4, warmup=100, total=1000):
    if step < warmup:
        return base_lr * step / warmup                     # ramp up
    p = (step - warmup) / (total - warmup)                 # 0 -> 1
    return 0.5 * base_lr * (1 + np.cos(np.pi * p))         # cosine to ~0

for s in [0, 50, 100, 300, 600, 1000]:
    print(f"step {s:4d}: lr = {lr_at(s):.2e}")
```

```output
step    0: lr = 0.00e+00
step   50: lr = 1.00e-04
step  100: lr = 2.00e-04
step  300: lr = 1.77e-04
step  600: lr = 8.26e-05
step 1000: lr = 0.00e+00
```

**Mental model:** optimizer = *how* to step (direction/scale per parameter);
schedule = *how big* to step over time (warmup up, then decay down). Both matter;
neither rescues a broken loss or bad data.

## Exercise

Implement an `AdamW` step in NumPy: standard Adam, but apply weight decay *directly
to the weights* (`w ← w − lr·wd·w`) rather than adding it to the gradient. Use it to
fit the logistic-regression weights for a small dataset. (Decoupled weight decay is
the "W" in AdamW and the correct way to regularize transformers.)

<details>
<summary>Solution sketch</summary>

```python
import numpy as np
class AdamW:
    def __init__(self, shape, lr=1e-3, b1=0.9, b2=0.999, eps=1e-8, wd=0.01):
        self.lr, self.b1, self.b2, self.eps, self.wd = lr, b1, b2, eps, wd
        self.m = np.zeros(shape); self.v = np.zeros(shape); self.t = 0
    def step(self, w, g):
        self.t += 1
        self.m = self.b1*self.m + (1-self.b1)*g
        self.v = self.b2*self.v + (1-self.b2)*g*g
        mh = self.m/(1-self.b1**self.t); vh = self.v/(1-self.b2**self.t)
        w = w - self.lr*(mh/(np.sqrt(vh)+self.eps) + self.wd*w)   # decoupled decay
        return w
```

Key line: weight decay multiplies `w`, not `g`. Coupling it into the gradient (plain
"L2") interacts badly with Adam's per-parameter scaling — decoupling fixes it.
</details>
