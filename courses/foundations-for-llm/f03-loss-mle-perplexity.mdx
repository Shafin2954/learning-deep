## Why cross-entropy is the right loss

You've been minimizing cross-entropy. This chapter explains *why* that's correct and connects it
to the two numbers you'll stare at all month: **training loss** and **perplexity**. The punchline:
minimizing cross-entropy equals **maximizing the likelihood of the data**, and an LLM is just this
applied to "predict the next token," over and over.

### From likelihood to cross-entropy

A loss function scores how wrong the model is. For predicting a category, the principled choice
comes from **maximum likelihood estimation (MLE)**: pick parameters that make the observed data
most probable. The probability the model assigns to all the true labels is

$$
\mathcal{L}(\theta) = \prod_i p_\theta(y_i \mid x_i)
$$

Products of many small probabilities underflow and are awkward to differentiate, so take the log
(monotonic — same maximizer) and negate (so we *minimize*):

$$
-\log \mathcal{L}(\theta) = -\sum_i \log p_\theta(y_i \mid x_i)
$$

That sum, averaged, *is* cross-entropy. So "minimize cross-entropy" and "maximize likelihood" are
the same instruction — and why F1's gradient $\mathrm{softmax} - \mathrm{onehot}$ keeps appearing:
it's the gradient of negative log-likelihood.

## A language model is next-token classification

The whole idea of an LLM in one line: at each position, predict the next token as a classification
over the vocabulary. The classes are the ~30k–250k tokens; the label is the actual next token. The
loss is cross-entropy averaged over every position:

$$
L = -\frac{1}{T} \sum_{t=1}^{T} \log p_\theta\big(\text{token}_t \mid \text{token}_1 \ldots \text{token}_{t-1}\big)
$$

Pretraining minimizes exactly this over a huge corpus; fine-tuning (Bengali ch. 8–9) minimizes the
same loss on your task. Nothing exotic — F1's cross-entropy with the vocabulary as the class set
and a causal mask so position $t$ can't peek at the future (you build that mask in F10/F12).

## Perplexity is exp(loss)

$$
\mathrm{perplexity} = e^{\,L}
$$

Interpretation: the model's **effective branching factor** — on average, how many tokens it's
choosing between. A model uniformly unsure among 5 tokens has cross-entropy $\ln 5$ and perplexity
5. A perfect model has perplexity 1. Lower is better.

A caveat that bites in Bengali: perplexity depends on the **tokenizer** — different tokenizations
of the same text give different perplexities, so never compare perplexities across models with
different vocabularies. This ties straight to fertility (Bengali ch. 3).

### Computing loss and perplexity in code

```python
import numpy as np
def softmax(z):
    z = z - z.max(-1, keepdims=True); e = np.exp(z); return e / e.sum(-1, keepdims=True)

# A tiny "language model" scoring a 4-token sequence over a vocab of 6.
np.random.seed(0); vocab = 6
logits  = np.random.randn(4, vocab)        # one logit row per position
targets = np.array([2, 5, 0, 3])           # the actual next tokens

p   = softmax(logits)
nll = -np.log(p[np.arange(4), targets])    # per-position negative log-likelihood
loss = nll.mean(); ppl = np.exp(loss)
print("per-token NLL :", nll.round(3))
print("cross-entropy :", round(loss, 4))
print("perplexity    :", round(ppl, 3))

# Sanity: a perfectly uniform model has perplexity == vocab size.
uniform = np.zeros((4, vocab))
print("uniform ppl   :",
      round(np.exp(-np.log(softmax(uniform)[np.arange(4), targets]).mean()), 3),
      "(should equal vocab = 6)")
```

```output
per-token NLL : [2.289 0.97  1.681 1.24 ]
cross-entropy : 1.5448
perplexity    : 4.687
uniform ppl   : 6.0 (should equal vocab = 6)
```

## Try it Yourself

```python
import numpy as np
def softmax(z):
    z = z - z.max(-1, keepdims=True); e = np.exp(z); return e / e.sum(-1, keepdims=True)

# Watch perplexity drop as the model gets more confident in the RIGHT token.
vocab = 10; target = 3
for boost in [0, 1, 2, 4, 8]:
    logits = np.zeros(vocab); logits[target] = boost     # make the true token more likely
    p = softmax(logits); ce = -np.log(p[target])
    print(f"boost={boost}: p(true)={p[target]:.3f}  CE={ce:.3f}  ppl={np.exp(ce):.3f}")
```

```output
boost=0: p(true)=0.100  CE=2.303  ppl=10.000
boost=1: p(true)=0.232  CE=1.461  ppl=4.311
boost=2: p(true)=0.451  CE=0.797  ppl=2.218
boost=4: p(true)=0.858  CE=0.153  ppl=1.165
boost=8: p(true)=0.997  CE=0.003  ppl=1.003
```

At boost 0 the model is uniform (ppl 10); as confidence in the true token rises, perplexity falls
toward 1.

## Why this matters for the competition

Whether your task is classification, QA, or generation, the model under the hood is minimizing
token-level cross-entropy. Understanding that (1) demystifies the training curves, (2) tells you
"good loss" and "good task metric" can diverge (Bengali ch. 14 — you optimize loss but are *scored*
on F1/BLEU/EM), and (3) explains why label quality matters so much: MLE faithfully fits whatever
labels you give it, errors included.

## Exercise

Given a model's predicted probability for the correct token at each of $T$ positions (a 1-D array
`p_true`), write `perplexity(p_true)` directly — no logits needed. Then explain in one line why you
must average the *log* probabilities, not the probabilities themselves.

<details>
<summary>Solution</summary>

```python
import numpy as np
def perplexity(p_true):
    return np.exp(-np.log(p_true).mean())

print(perplexity(np.array([0.5, 0.25, 0.1, 0.9])))
```

Perplexity is the **geometric** mean of the per-token branching factor. Averaging in log-space
gives the geometric mean of probabilities; one near-zero probability (a token the model thought
impossible but happened) correctly blows perplexity up — averaging raw probabilities would hide it.
</details>
