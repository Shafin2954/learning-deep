## Discriminative vs Generative Models

A discriminative model learns $p(y|x)$ — given an input, predict a label. A logistic classifier, a gradient-boosted tree, a fine-tuned BERT all fall here. They draw a boundary between classes.

A generative model learns something harder: $p(x)$, the distribution of the data itself. Given that distribution you can do three things a discriminative model cannot: generate new samples that look real, compute how likely a given input is, and fill in missing parts of an input.

Some generative models learn the joint $p(x, y)$ and derive $p(y|x) = p(x,y)/p(x)$. Naive Bayes works this way. But modern deep generative models target $p(x)$ directly.

## Likelihood-Based vs Implicit Models

The core divide in deep generative models is whether you ever explicitly compute $p(x)$.

**Likelihood-based models** — VAEs, normalizing flows, diffusion models — define or bound $p(x)$ and train by maximizing it. You can assign a likelihood score to any new input, which is useful for anomaly detection and compression.

**Implicit models** — GANs — never write down $p(x)$. Instead, a generator network maps noise to data space, and a discriminator separates real from fake. Training is adversarial, not maximum-likelihood.

```python
import numpy as np
import matplotlib.pyplot as plt

# Illustrate: real data is a mixture of Gaussians
rng = np.random.default_rng(0)
real = np.concatenate([rng.normal(-2, 0.5, 300),
                        rng.normal(2, 0.5, 300)])

# A generative model learns this shape
x = np.linspace(-5, 5, 500)
# Two-component mixture density (true distribution)
density = (0.5 * np.exp(-0.5*((x+2)/0.5)**2) / (0.5*np.sqrt(2*np.pi)) +
           0.5 * np.exp(-0.5*((x-2)/0.5)**2) / (0.5*np.sqrt(2*np.pi)))

print(f"Real samples: {real.shape}")
print(f"Sample mean: {real.mean():.3f}, std: {real.std():.3f}")
print(f"Peak density at x=-2: {density[np.argmin(np.abs(x+2))]:.4f}")
print(f"Peak density at x=+2: {density[np.argmin(np.abs(x-2))]:.4f}")
```

```output
Real samples: (600,)
Sample mean: 0.021, std: 1.936
Peak density at x=-2: 0.7979
Peak density at x=+2: 0.7979
```

## The Generative Modeling Goal

You have a dataset $\mathcal{D} = \{x^{(1)}, \ldots, x^{(N)}\}$ drawn i.i.d. from an unknown distribution $p_\text{data}(x)$. Your model defines a parameterized family $p_\theta(x)$. Training pushes $p_\theta$ toward $p_\text{data}$ — either by minimizing KL divergence (MLE), matching score functions, or adversarial game.

Once trained, you sample $x_\text{new} \sim p_\theta(x)$. If the model learned well, new samples are indistinguishable from training data.

## Latent Variable Models

Most deep generative models introduce a latent variable $z$ — a compressed, structured code. The generative story is:

$$
z \sim p(z), \quad x \sim p_\theta(x|z)
$$

The marginal $p_\theta(x) = \int p_\theta(x|z) p(z)\, dz$ is what you want to maximize, but the integral is intractable for high-dimensional $x$. Different models handle this differently:

- **VAE**: bound the integral with a variational lower bound (ELBO).
- **Flow**: make the mapping invertible so the integral becomes a change of variables.
- **Diffusion**: define a forward process that destroys $x$ into noise, learn to reverse it.
- **GAN**: skip the integral entirely, train a generator adversarially.

```python
# Visualize the latent-variable picture
import numpy as np
rng = np.random.default_rng(42)

# Sample from a simple latent space (2D standard normal)
z = rng.normal(0, 1, (500, 2))

# A toy "decoder": nonlinear map from z to x (not trained)
def toy_decode(z):
    x1 = z[:, 0] * np.cos(z[:, 1]) - z[:, 1] * np.sin(z[:, 0])
    x2 = z[:, 0] * np.sin(z[:, 1]) + z[:, 1] * np.cos(z[:, 0])
    return np.stack([x1, x2], axis=1)

x = toy_decode(z)
print(f"Latent z: shape={z.shape}, mean={z.mean(axis=0).round(2)}")
print(f"Generated x: shape={x.shape}, mean={x.mean(axis=0).round(2)}")
print(f"x range: [{x.min():.2f}, {x.max():.2f}]")
```

```output
Latent z: shape=(500, 2), mean=[-0.04  0.05]
Generated x: shape=(500, 2), mean=[ 0.04  0.07]
x range: [-6.11, 6.28]
```

## Applications

Generative models are the engine behind:

- **Image synthesis** — Stable Diffusion, Midjourney (latent diffusion).
- **Data augmentation** — generate more training examples for rare classes.
- **Anomaly detection** — an input with low $p_\theta(x)$ is anomalous.
- **Learned compression** — encode $x$ as $z$, store $z$, reconstruct with decoder.
- **Imputation** — condition on observed dimensions, sample missing ones.
- **Drug discovery** — generate molecules with desired properties.

The rest of this course builds each model family from scratch, starting with autoencoders (ch. 2), VAEs (ch. 3), GANs (ch. 4–5), flows (ch. 6), and diffusion (ch. 7–9).

## Try it Yourself

```python
import numpy as np

rng = np.random.default_rng(7)

# Simple 1D generative model: Gaussian mixture
# "Training" = fit means and weights from data
data = np.concatenate([rng.normal(-3, 0.8, 200),
                        rng.normal(1, 0.5, 200),
                        rng.normal(4, 0.6, 100)])

# Fit a 3-component model (manually for illustration)
means = np.array([-3.0, 1.0, 4.0])
stds  = np.array([0.8, 0.5, 0.6])
weights = np.array([0.4, 0.4, 0.2])

# Sample from the model
component = rng.choice(3, size=50, p=weights)
samples = rng.normal(means[component], stds[component])
print("Generated samples (first 10):", samples[:10].round(2))
print(f"Generated mean: {samples.mean():.2f}, std: {samples.std():.2f}")
print(f"Data mean:      {data.mean():.2f}, std: {data.std():.2f}")
```

```output
Generated samples (first 10): [-3.76 -3.37 -2.58  0.91  1.14  4.25 -2.76  1.15 -3.66 -2.89]
Generated mean: -0.93, std: 2.54
Data mean:      -1.00, std: 2.52
```

## Exercise

You observe 1000 data points drawn from a Gaussian mixture. Without sklearn, write code to estimate the mixture means and weights using the sample mean and a histogram. Then draw 100 new samples from your estimated model and verify the sample statistics are close to the original.

<details>
<summary>Solution</summary>

```python
import numpy as np

rng = np.random.default_rng(0)
data = np.concatenate([rng.normal(-2, 1.0, 500), rng.normal(3, 0.8, 500)])

# Rough two-component fit: split at 0
left = data[data < 0.5]
right = data[data >= 0.5]
w_left = len(left) / len(data)
w_right = len(right) / len(data)

mu_left, s_left   = left.mean(), left.std()
mu_right, s_right = right.mean(), right.std()

print(f"Left component:  mu={mu_left:.2f}, sigma={s_left:.2f}, weight={w_left:.2f}")
print(f"Right component: mu={mu_right:.2f}, sigma={s_right:.2f}, weight={w_right:.2f}")

# Sample 100 points
comp = rng.choice([0,1], size=100, p=[w_left, w_right])
samples = np.where(comp==0,
                   rng.normal(mu_left, s_left, 100),
                   rng.normal(mu_right, s_right, 100))
print(f"\nOriginal: mean={data.mean():.2f}, std={data.std():.2f}")
print(f"Samples:  mean={samples.mean():.2f}, std={samples.std():.2f}")
```

Splitting the data at x=0 gives a good approximation of the two components. The key insight is that fitting $p(x)$ lets you generate new points — the core promise of generative modeling.
</details>
