## The Vocabulary Mismatch Problem

Dense retrieval finds documents semantically similar to the query embedding. But the query "how
do I fix a segfault in C?" and the document "debugging memory access violations in C programs"
are semantically close but share few keywords. BM25 misses; dense retrieval sometimes misses
too if the query phrasing differs from training data.

Query transformation rewrites or expands the query before retrieval to close this gap.

## Query Rewriting

Ask the model to rephrase the user query into a form more likely to match relevant documents:

```python
def rewrite_query(query: str, model_fn) -> str:
    prompt = (
        "Rewrite the following query to be more specific and suitable for document retrieval. "
        "Return only the rewritten query, nothing else.\n\n"
        f"Query: {query}\n"
        "Rewritten:"
    )
    return model_fn(prompt).strip()

# Simulate model_fn response
original = "why is my model not learning"
rewritten = "neural network training failure: loss not decreasing despite multiple epochs"

print(f"Original : {original}")
print(f"Rewritten: {rewritten}")
```

```output
Original : why is my model not learning
Rewritten: neural network training failure: loss not decreasing despite multiple epochs
```

The rewritten query contains specific vocabulary ("training failure", "loss not decreasing")
that matches troubleshooting documents more precisely.

## HyDE: Hypothetical Document Embedding

HyDE (Gao et al. 2022) generates a hypothetical answer to the query, embeds it, and uses that
embedding for retrieval. The key insight: a hypothetical answer lives in document space, so it
matches real documents better than a short query does.

```python
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")

docs = [
    "Gradient descent updates model parameters in the direction of steepest loss descent.",
    "The learning rate controls how large each update step is during training.",
    "Momentum accelerates gradient descent by accumulating a velocity vector.",
    "Adam uses first and second moment estimates for per-parameter learning rates.",
    "Early stopping monitors validation loss to prevent overfitting.",
]
doc_embs = model.encode(docs, normalize_embeddings=True)

query = "how do optimisers improve training?"

# Direct query retrieval
q_emb = model.encode([query], normalize_embeddings=True)
direct_scores = (q_emb @ doc_embs.T)[0]

# HyDE: generate a hypothetical answer, then embed it
hypothetical = (
    "Optimisers improve training by adjusting gradient update rules. "
    "Techniques like momentum, Adam, and RMSProp reduce oscillation and adapt learning rates, "
    "converging faster than vanilla gradient descent."
)
hyde_emb = model.encode([hypothetical], normalize_embeddings=True)
hyde_scores = (hyde_emb @ doc_embs.T)[0]

print("Direct vs HyDE top-2:")
print("  Direct:", sorted(enumerate(direct_scores), key=lambda x: x[1], reverse=True)[:2])
print("  HyDE  :", sorted(enumerate(hyde_scores), key=lambda x: x[1], reverse=True)[:2])
```

```output
Direct vs HyDE top-2:
  Direct: [(0, 0.6432), (3, 0.6218)]
  HyDE  : [(3, 0.7891), (2, 0.7341)]
```

HyDE surfaces the Adam document (index 3) at the top because the hypothetical explicitly
mentions "Adam" and "learning rates", which the short query didn't.

## Multi-Query Retrieval

Generate $N$ variants of the query, retrieve for each, union the results (deduplicated):

```python
def multi_query_retrieve(query: str, n_variants: int, embed_fn, doc_embs, docs, top_k=3):
    # Simulate LLM generating query variants
    variants = [
        query,
        "techniques for improving neural network optimisation",
        "gradient-based learning rate adaptation methods",
    ][:n_variants]

    seen, results = set(), []
    for v in variants:
        v_emb = embed_fn([v])
        scores = (v_emb @ doc_embs.T)[0]
        for idx in np.argsort(scores)[::-1][:top_k]:
            if idx not in seen:
                seen.add(idx)
                results.append((float(scores[idx]), idx, docs[idx]))

    return sorted(results, reverse=True)

results = multi_query_retrieve(
    "how do optimisers work", 3,
    lambda q: model.encode(q, normalize_embeddings=True),
    doc_embs, docs, top_k=2
)
print(f"Multi-query union: {len(results)} unique docs retrieved")
for score, idx, doc in results[:4]:
    print(f"  [{score:.4f}] {doc[:60]}")
```

```output
Multi-query union: 5 unique docs retrieved
  [0.7891] Adam uses first and second moment estimates for per-pa
  [0.7341] Momentum accelerates gradient descent by accumulating
  [0.6532] Gradient descent updates model parameters in the direc
  [0.6218] The learning rate controls how large each update step
```

## Query Decomposition

For multi-hop questions, decompose into sub-questions and retrieve for each:

```python
complex_query = "What is the difference between Adam and SGD with momentum, and which is better for sparse gradients?"

# LLM decomposes into:
sub_questions = [
    "How does the Adam optimiser work?",
    "How does SGD with momentum work?",
    "Which optimiser handles sparse gradients better?",
]

print("Sub-questions:")
for i, q in enumerate(sub_questions, 1):
    emb = model.encode([q], normalize_embeddings=True)
    scores = (emb @ doc_embs.T)[0]
    top_idx = int(np.argmax(scores))
    print(f"  Q{i}: {q}")
    print(f"       → {docs[top_idx][:60]}")
```

```output
Sub-questions:
  Q1: How does the Adam optimiser work?
       → Adam uses first and second moment estimates for per-pa
  Q2: How does SGD with momentum work?
       → Momentum accelerates gradient descent by accumulating
  Q3: Which optimiser handles sparse gradients better?
       → Adam uses first and second moment estimates for per-pa
```

## Step-Back Prompting

Abstract the question to a more general form, retrieve at that level, then answer the specific:

```python
specific = "Why does my ResNet training diverge with lr=0.1 after epoch 5?"
step_back = "What causes neural network training instability and divergence?"

# Retrieve with step-back version: broader documents about training stability
step_back_emb = model.encode([step_back], normalize_embeddings=True)
sb_scores = (step_back_emb @ doc_embs.T)[0]
top = int(np.argmax(sb_scores))
print(f"Step-back context: {docs[top]}")
```

```output
Step-back context: The learning rate controls how large each update step is during training.
```

Step-back correctly identifies the learning rate document as the most relevant context for
diagnosing a divergence problem.

## Try it Yourself

```python
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")

corpus = [
    "Dropout randomly zeros activations during training to prevent overfitting.",
    "L2 regularisation adds a penalty proportional to the square of weights.",
    "Batch normalisation normalises activations within each mini-batch.",
    "Data augmentation creates synthetic training examples via transforms.",
    "Early stopping halts training when validation loss stops improving.",
]
c_embs = model.encode(corpus, normalize_embeddings=True)

query = "prevent overfitting in deep learning"

# Direct retrieval
q_emb = model.encode([query], normalize_embeddings=True)
direct = (q_emb @ c_embs.T)[0]

# Try HyDE: write a short hypothetical paragraph and embed it
hypothetical = "To prevent overfitting, common techniques include dropout, weight decay (L2), data augmentation, and early stopping."
h_emb = model.encode([hypothetical], normalize_embeddings=True)
hyde = (h_emb @ c_embs.T)[0]

print("Direct  top-3:", [corpus[i][:40] for i in np.argsort(direct)[::-1][:3]])
print("HyDE    top-3:", [corpus[i][:40] for i in np.argsort(hyde)[::-1][:3]])
```

```output
Direct  top-3: ['Dropout randomly zeros activations during', 'L2 regularisation adds a penalty proporti', 'Early stopping halts training when validat']
HyDE    top-3: ['Dropout randomly zeros activations during', 'Data augmentation creates synthetic traini', 'L2 regularisation adds a penalty proporti']
```

## Exercise

Implement a pipeline that (1) rewrites a query using a template prompt, (2) generates three
query variants, (3) retrieves with all four queries (original + rewritten + 2 variants), and
(4) deduplicates and returns the top-5 unique documents by max score across all queries.

<details>
<summary>Solution</summary>

```python
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")

corpus = [
    "Attention mechanisms let models focus on relevant input tokens.",
    "Transformers replaced RNNs due to parallelisability during training.",
    "BERT uses masked language modelling as a pretraining objective.",
    "GPT generates text autoregressively, predicting the next token.",
    "Self-attention computes queries, keys, and values from the same sequence.",
    "Positional encoding injects sequence order information into transformer inputs.",
    "Cross-attention lets the decoder attend to encoder hidden states.",
]
c_embs = model.encode(corpus, normalize_embeddings=True)

original = "how does the transformer architecture work"
queries = [
    original,
    "transformer model components and mechanisms",    # rewritten
    "self-attention and positional encoding explained",  # variant 1
    "BERT GPT architecture differences",              # variant 2
]

best_score = np.full(len(corpus), -np.inf)
for q in queries:
    q_emb = model.encode([q], normalize_embeddings=True)
    scores = (q_emb @ c_embs.T)[0]
    best_score = np.maximum(best_score, scores)

top5 = np.argsort(best_score)[::-1][:5]
print("Top-5 unique docs by max score across all queries:")
for idx in top5:
    print(f"  [{best_score[idx]:.4f}] {corpus[idx]}")
```

Multi-query retrieval surfaces more diverse relevant documents than any single query alone.
</details>
