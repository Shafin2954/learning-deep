## The project

You will build a production-style RAG pipeline entirely from scratch — no LangChain, no
LlamaIndex, no Haystack — that ingests a document corpus, indexes it for hybrid retrieval,
transforms queries to improve recall, and generates grounded answers. Then you will evaluate
the pipeline on a golden QA set you build yourself, measuring both retrieval quality and
answer faithfulness. The deliverable is a working pipeline with measured metrics and a written
analysis of what improved retrieval most.

## Hard constraints (no vibe coding)

- You MAY NOT use LangChain, LlamaIndex, Haystack, or any other RAG framework for the
  pipeline wiring. Wire every component yourself.
- You MAY NOT use `rank_bm25`, `whoosh`, or any BM25 library for the core scoring. Implement
  the BM25 Robertson formula yourself. You may use `rank_bm25` only to validate your
  implementation gives the same scores.
- Your RRF fusion function must be your own code (no library call).
- Your agentic loop must be custom (no LangGraph, Autogen, or CrewAI).
- You MAY use `sentence-transformers` for encoding, `faiss` for ANN indexing, and
  `sentence-transformers.CrossEncoder` for the reranker.
- You MAY use any LLM API (Anthropic, OpenAI, Ollama) for generation and the judge.

## What it must contain (and the chapter each tests)

- **Chunking pipeline** (ch. 2, 11): implement fixed-size chunking (with overlap) and
  at least one semantic or sentence-aware variant; compare chunk sizes on retrieval quality.
- **BM25 from scratch** (ch. 3, 11): implement the Robertson BM25 formula; build an inverted
  index; score a query against the full corpus in $O(\text{unique terms in query} \times df)$.
- **Dense retrieval with FAISS** (ch. 7, 11): encode all chunks with a sentence encoder;
  build a FAISS flat index; retrieve top-k by cosine similarity.
- **Hybrid RRF fusion** (ch. 11): combine BM25 and dense ranked lists with Reciprocal Rank
  Fusion; show it beats either system alone on at least one query.
- **Cross-encoder reranker** (ch. 11): rerank the top-20 from hybrid retrieval with a
  cross-encoder; measure recall improvement at k=5 before and after reranking.
- **Query rewriting and HyDE** (ch. 12): implement query rewriting (LLM rephrases the query)
  and HyDE (LLM generates a hypothetical answer, embed it); show HyDE improves recall on
  at least 3 of your 10 test queries.
- **Agentic reflect-and-retrieve loop** (ch. 13): after first retrieval, ask the LLM whether
  context is sufficient; if not, generate a follow-up query and retrieve again; the loop
  must terminate (max 3 steps) and demonstrate at least one self-correction on a multi-hop
  question.
- **Retrieval evaluation** (ch. 14): build a golden QA set of at least 10 question–answer–
  relevant\_doc\_ids tuples; compute Recall@5, Precision@5, and MRR for BM25-only,
  dense-only, and hybrid+rerank.
- **Faithfulness evaluation** (ch. 14): for each generated answer, compute an automated
  faithfulness score (LLM-as-judge or BERTScore); report mean faithfulness across 10 questions.
- **Multilingual query support** (ch. 15): accept a Bengali query, normalise Unicode (NFC,
  strip ZWJ), and retrieve from the corpus using a multilingual encoder (LaBSE or XLM-R).

## Acceptance criteria

- Recall@10 ≥ 0.70 on your golden QA set using the hybrid + reranker pipeline.
- Hybrid retrieval outperforms BM25-only on Recall@5 by at least 5 percentage points.
- Hybrid retrieval outperforms dense-only on at least one cluster of terminology-heavy queries.
- HyDE improves Recall@5 on at least 3 of 10 test queries compared to raw query retrieval.
- Agentic loop demonstrates at least one multi-step retrieval where the second query is
  meaningfully different from the first and retrieves a complementary document.
- Mean faithfulness score ≥ 0.80 (LLM-as-judge 1–5 scale, normalised to 0–1).
- Bengali query pipeline returns at least one relevant result for a query about a topic
  covered in the corpus, with NFC normalisation applied before encoding.
- A written retrieval quality report (markdown) that: describes the corpus and golden QA set,
  shows the metrics table, explains which improvement had the biggest impact, and notes at
  least one failure mode observed.
