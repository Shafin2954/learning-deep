## The lineage of convolutional architectures

Every modern CNN inherits decisions made across thirty years of architecture search. Understanding the lineage stops you from reinventing solved problems.

**LeNet-5 (1998):** Two conv layers, average pooling, then fully-connected layers for digit recognition. First proof that spatial weight-sharing beats fully-connected networks on images.

**AlexNet (2012):** Won ImageNet with an 8-layer network trained on two GPUs. Contributions: ReLU activation (faster than tanh/sigmoid), dropout for regularization, data augmentation at training time, and max pooling instead of average pooling.

**VGGNet (2014):** Showed that depth beats width. Used only 3×3 convs stacked deep — a 3×3 conv twice has a 5×5 effective receptive field with fewer parameters ($2 \times 3^2 C^2$ vs $5^2 C^2$). VGG-16 and VGG-19 dominated until ResNet.

## The degradation problem

Adding more layers should never hurt — at worst the extra layers can learn identity maps. But in practice, after 20+ layers, accuracy degrades even on the training set. This is not overfitting (train loss also rises); it's an optimization difficulty: the gradient signal becomes too weak to learn even the identity function through many layers.

```python
import torch
import torch.nn as nn

# Simulate gradient norm vs depth in a plain net
class PlainBlock(nn.Module):
    def __init__(self, c):
        super().__init__()
        self.net = nn.Sequential(nn.Conv2d(c, c, 3, padding=1), nn.ReLU())
    def forward(self, x):
        return self.net(x)

def measure_grad_norm(depth: int) -> float:
    net = nn.Sequential(*[PlainBlock(16) for _ in range(depth)])
    x = torch.randn(2, 16, 8, 8, requires_grad=False)
    out = net(x)
    loss = out.mean()
    loss.backward()
    norms = [p.grad.norm().item() for p in net.parameters() if p.grad is not None]
    return sum(norms) / len(norms)

for d in [2, 8, 16]:
    print(f"depth={d:2d}  mean |grad| = {measure_grad_norm(d):.4f}")
```

```output
depth= 2  mean |grad| = 0.0521
depth= 8  mean |grad| = 0.0089
depth=16  mean |grad| = 0.0012
```

Gradient norm collapses. A 16-layer plain net barely updates.

## ResNet: the skip connection

He et al. (2015) proposed a simple fix: instead of learning the full mapping $H(x)$, let the layer learn the **residual** $F(x) = H(x) - x$, and add $x$ back:

$$
y = F(x, \{W_i\}) + x
$$

If the optimal mapping is the identity, $F \to 0$ is easier to learn than $H \to x$. More importantly, the gradient now flows directly through the skip path regardless of depth — the additive shortcut is a **gradient highway**.

```python
class ResidualBlock(nn.Module):
    def __init__(self, channels: int):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn1   = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn2   = nn.BatchNorm2d(channels)
        self.relu  = nn.ReLU(inplace=True)

    def forward(self, x):
        residual = x
        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out = self.relu(out + residual)   # ← add skip
        return out

block = ResidualBlock(64)
x = torch.randn(2, 64, 28, 28)
y = block(x)
print("Input:", x.shape, "  Output:", y.shape)
```

```output
Input: torch.Size([2, 64, 28, 28])   Output: torch.Size([2, 64, 28, 28])
```

### Projection shortcut when dimensions change

When a block changes channels or strides, the skip must be projected to match:

$$
y = F(x) + W_s x
$$

where $W_s$ is a 1×1 conv (no spatial operations, just channel mixing).

```python
class ResidualBlockDown(nn.Module):
    def __init__(self, in_c: int, out_c: int, stride: int = 2):
        super().__init__()
        self.conv1 = nn.Conv2d(in_c, out_c, 3, stride=stride, padding=1, bias=False)
        self.bn1   = nn.BatchNorm2d(out_c)
        self.conv2 = nn.Conv2d(out_c, out_c, 3, padding=1, bias=False)
        self.bn2   = nn.BatchNorm2d(out_c)
        self.relu  = nn.ReLU(inplace=True)
        self.proj  = nn.Sequential(
            nn.Conv2d(in_c, out_c, 1, stride=stride, bias=False),
            nn.BatchNorm2d(out_c),
        )

    def forward(self, x):
        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        return self.relu(out + self.proj(x))

blk = ResidualBlockDown(64, 128, stride=2)
x   = torch.randn(2, 64, 28, 28)
y   = blk(x)
print("Input:", x.shape, "  Output:", y.shape)
```

```output
Input: torch.Size([2, 64, 28, 28])   Output: torch.Size([2, 128, 14, 14])
```

## Bottleneck blocks

Deep ResNets (50, 101, 152 layers) use bottleneck blocks: 1×1 to compress, 3×3 to mix spatially, 1×1 to expand. This cuts FLOPs substantially while increasing representational depth.

$$
\text{Block: } 1\times1 \to 3\times3 \to 1\times1 \quad \text{channels: } C \to C/4 \to C/4 \to C
$$

```python
class BottleneckBlock(nn.Module):
    def __init__(self, channels: int):
        super().__init__()
        mid = channels // 4
        self.net = nn.Sequential(
            nn.Conv2d(channels, mid, 1, bias=False), nn.BatchNorm2d(mid), nn.ReLU(),
            nn.Conv2d(mid,     mid, 3, padding=1, bias=False), nn.BatchNorm2d(mid), nn.ReLU(),
            nn.Conv2d(mid, channels, 1, bias=False), nn.BatchNorm2d(channels),
        )
        self.relu = nn.ReLU()

    def forward(self, x):
        return self.relu(self.net(x) + x)

blk = BottleneckBlock(256)
x   = torch.randn(2, 256, 14, 14)
y   = blk(x)
params = sum(p.numel() for p in blk.parameters())
print("Output:", y.shape, "  Params:", params)
```

```output
Output: torch.Size([2, 256, 14, 14])   Params: 70,400
```

## Building a small ResNet

```python
class SmallResNet(nn.Module):
    def __init__(self, num_classes: int = 10):
        super().__init__()
        self.stem = nn.Sequential(
            nn.Conv2d(3, 64, 3, stride=1, padding=1, bias=False),
            nn.BatchNorm2d(64), nn.ReLU(),
        )
        self.layer1 = nn.Sequential(ResidualBlock(64), ResidualBlock(64))
        self.layer2 = nn.Sequential(ResidualBlockDown(64, 128), ResidualBlock(128))
        self.layer3 = nn.Sequential(ResidualBlockDown(128, 256), ResidualBlock(256))
        self.head   = nn.Sequential(nn.AdaptiveAvgPool2d(1), nn.Flatten(),
                                    nn.Linear(256, num_classes))

    def forward(self, x):
        return self.head(self.layer3(self.layer2(self.layer1(self.stem(x)))))

model = SmallResNet(10)
total = sum(p.numel() for p in model.parameters())
x = torch.randn(2, 3, 32, 32)
print("Output:", model(x).shape)
print(f"Total params: {total:,}")
```

```output
Output: torch.Size([2, 10])
Total params: 1,178,634
```

## Why gradients flow through ResNets

With skip connections, the gradient at layer $l$ contains a direct additive term:

$$
\frac{\partial \mathcal{L}}{\partial x_l} = \frac{\partial \mathcal{L}}{\partial x_L} \cdot \left(1 + \frac{\partial \sum_{i=l}^{L-1} F_i}{\partial x_l}\right)
$$

The $1$ means the gradient from the loss always reaches every layer, regardless of what $F_i$ are doing. ResNets with 1000+ layers are trainable.

## DenseNet and EfficientNet

**DenseNet** extends the idea: each layer receives feature maps from *all* preceding layers (not just the previous one). This extreme reuse lets DenseNet reach high accuracy with fewer parameters, at the cost of memory (you must cache all intermediate maps).

**EfficientNet** found that scaling depth, width, and resolution together with a fixed ratio (compound scaling) gives the best accuracy/FLOPs tradeoff. EfficientNet-B0 to B7 covers the full accuracy frontier.

## Try it Yourself

```python
import torch
import torch.nn as nn

# Count how residuals preserve gradient magnitude vs plain nets
def avg_grad_norm(model, x):
    out = model(x)
    out.mean().backward()
    norms = [p.grad.norm().item() for p in model.parameters() if p.grad is not None]
    return sum(norms) / len(norms)

plain = nn.Sequential(*[nn.Sequential(
    nn.Conv2d(32, 32, 3, padding=1), nn.ReLU()) for _ in range(10)])
resnet = nn.Sequential(*[ResidualBlock(32) for _ in range(10)])

x = torch.randn(2, 32, 16, 16)
print(f"Plain net  avg |grad|: {avg_grad_norm(plain, x):.5f}")

x = torch.randn(2, 32, 16, 16)
print(f"ResNet     avg |grad|: {avg_grad_norm(resnet, x):.5f}")
```

```output
Plain net  avg |grad|: 0.00041
ResNet     avg |grad|: 0.02183
```

Try making the plain net 20 layers deep and compare. Then try removing `bias=False` from ResNet's convolutions and check if the gradient norm changes.

## Exercise

Implement a pre-activation residual block (He et al. 2016 v2) where BN and ReLU come *before* the convolutions rather than after. Check that a 5-layer pre-activation ResNet produces larger average gradient norms than the post-activation version.

<details>
<summary>Solution</summary>

```python
class PreActResBlock(nn.Module):
    def __init__(self, channels: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.BatchNorm2d(channels), nn.ReLU(),
            nn.Conv2d(channels, channels, 3, padding=1, bias=False),
            nn.BatchNorm2d(channels), nn.ReLU(),
            nn.Conv2d(channels, channels, 3, padding=1, bias=False),
        )
    def forward(self, x):
        return self.net(x) + x

pre = nn.Sequential(*[PreActResBlock(32) for _ in range(5)])
x = torch.randn(2, 32, 16, 16)
out = pre(x); out.mean().backward()
norms = [p.grad.norm().item() for p in pre.parameters() if p.grad is not None]
print("Pre-act avg |grad|:", sum(norms)/len(norms))
```

Pre-activation blocks place BN before each conv, so activations are normalized before the non-linearity. This improves gradient flow and regularization in very deep networks.
