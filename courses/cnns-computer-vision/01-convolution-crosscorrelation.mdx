## Why convolution for images?

Fully-connected layers treat every input pixel as independent — a 256×256 image has 65,536 inputs,
and a single FC layer with 1,024 hidden units needs 67 million parameters just to process it once.
Two structural priors make images tractable:

- **Local structure:** nearby pixels are correlated; distant pixels rarely interact directly.
- **Translation invariance:** the edge detector that fires on the left should fire the same way on the right.

Convolution exploits both. One kernel slides over the entire image; each output is the dot product
of the kernel with the local patch. The same weights are reused everywhere — **weight sharing**.

## Cross-correlation vs convolution

Mathematical convolution flips the kernel: $(f * g)(t) = \int f(\tau)g(t-\tau)\,d\tau$.
Cross-correlation doesn't: $(f \star g)(t) = \int f(\tau)g(t+\tau)\,d\tau$.

PyTorch's `nn.Conv2d` computes **cross-correlation** but calls it "convolution". That's fine —
the network learns the kernel values, so the flip doesn't matter; it just learns the flipped version.

## 2D cross-correlation from scratch

For input $X$ (shape $H \times W$) and kernel $K$ (shape $k \times k$):

$$
Y[i, j] = \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} X[i+m,\; j+n] \cdot K[m, n]
$$

```python
import numpy as np

def conv2d_naive(X: np.ndarray, K: np.ndarray) -> np.ndarray:
    """Cross-correlation. X: (H,W), K: (kh,kw). No padding."""
    H, W = X.shape
    kh, kw = K.shape
    out_h = H - kh + 1
    out_w = W - kw + 1
    Y = np.zeros((out_h, out_w), dtype=np.float64)
    for i in range(out_h):
        for j in range(out_w):
            Y[i, j] = (X[i:i+kh, j:j+kw] * K).sum()
    return Y

rng = np.random.default_rng(0)
X = rng.standard_normal((6, 6))
K = np.array([[1, 0, -1],
              [2, 0, -2],
              [1, 0, -1]], dtype=np.float64)   # vertical-edge (Sobel) kernel

Y = conv2d_naive(X, K)
print("Input shape:", X.shape)
print("Kernel shape:", K.shape)
print("Output shape:", Y.shape)
print("Output:\n", Y.round(3))
```

```output
Input shape: (6, 6)
Kernel shape: (3, 3)
Output shape: (4, 4)
Output:
 [[ 0.635 -1.802  1.453 -0.364]
 [-0.211 -1.031  3.021 -1.156]
 [-1.045  0.752  0.893 -0.826]
 [-0.218  2.476 -0.817 -1.274]]
```

The output is $(H - k + 1) \times (W - k + 1) = 4 \times 4$ when the kernel is $3 \times 3$ and
there is no padding.

## Verify against PyTorch

```python
import torch
import torch.nn.functional as F

X_t = torch.tensor(X, dtype=torch.float32).unsqueeze(0).unsqueeze(0)  # (1,1,6,6)
K_t = torch.tensor(K, dtype=torch.float32).unsqueeze(0).unsqueeze(0)  # (1,1,3,3)
```

## Weight sharing and sparse connectivity

A 6×6 image with a 3×3 kernel produces 16 output values. Each output uses only 9 of the 36 input
values — **sparse connectivity**. The same 9 kernel weights are shared across all 16 outputs —
**weight sharing**. Compare to a fully-connected layer on the same input: $36 \times 16 = 576$ weights.
Here: 9. And the 9 weights generalise across the image.

### Multiple kernels → multiple channels

In practice, a conv layer applies $C_{out}$ kernels simultaneously, producing $C_{out}$ feature maps.
Each kernel detects a different pattern. The full conv operation for an input of shape
$(C_{in}, H, W)$ and weight tensor $(C_{out}, C_{in}, k, k)$:

$$
Y[c_{out}, i, j] = \sum_{c_{in}} \sum_m \sum_n W[c_{out}, c_{in}, m, n] \cdot X[c_{in}, i+m, j+n] + b[c_{out}]
$$

```python
import torch
import torch.nn as nn

torch.manual_seed(0)
layer = nn.Conv2d(in_channels=3, out_channels=16, kernel_size=3, padding=0)
x = torch.randn(1, 3, 32, 32)   # batch=1, 3 channels, 32×32 image
y = layer(x)
print("Input:", x.shape)
print("Output:", y.shape)
print("Kernel tensor shape:", layer.weight.shape)
print("Parameters:", sum(p.numel() for p in layer.parameters()))
```

```output
Input: torch.Size([1, 3, 32, 32])
Output: torch.Size([1, 16, 30, 30])
Kernel tensor shape: torch.Size([16, 3, 3, 3])
Parameters: 448
```

$16 \times (3 \times 3 \times 3) + 16 = 432 + 16 = 448$ parameters.
A fully-connected equivalent would need $3072 \times 14400 = 44{,}236{,}800$.

## Exercise

Implement a faster version of `conv2d_naive` using `np.lib.stride_tricks.as_strided` to
extract all patches at once (no Python loops), then verify it gives identical results.

<details>
<summary>Solution</summary>

```python
import numpy as np

def conv2d_strided(X: np.ndarray, K: np.ndarray) -> np.ndarray:
    H, W = X.shape
    kh, kw = K.shape
    out_h, out_w = H - kh + 1, W - kw + 1
    # Build a view of shape (out_h, out_w, kh, kw) with strides
    s = X.strides
    patches = np.lib.stride_tricks.as_strided(
        X,
        shape=(out_h, out_w, kh, kw),
        strides=(s[0], s[1], s[0], s[1])
    )
    return np.einsum('ijmn,mn->ij', patches, K)

rng = np.random.default_rng(0)
X = rng.standard_normal((6, 6))
K = np.array([[1, 0, -1], [2, 0, -2], [1, 0, -1]], dtype=float)

Y_naive   = np.array([[((X[i:i+3, j:j+3]) * K).sum() for j in range(4)] for i in range(4)])
Y_strided = conv2d_strided(X, K)
print("Max diff:", np.abs(Y_naive - Y_strided).max())
```

The stride trick creates a zero-copy view of all patches simultaneously.
`einsum('ijmn,mn->ij', ...)` contracts the patch and kernel dimensions, giving the same result
without any Python loops over spatial positions.
</details>
