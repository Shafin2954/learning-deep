## The detection problem

Classification assigns one label to one image. Detection must find **multiple objects of possibly different classes at unknown locations**. Each prediction is a 5-tuple: $(x_1, y_1, x_2, y_2, \text{class})$ plus a confidence score.

Two sub-problems:
1. **Where** — predict bounding box coordinates (regression).
2. **What** — classify each detected region (classification).

## Intersection over Union (IoU)

IoU measures how much two boxes overlap:

$$
\text{IoU}(A, B) = \frac{|A \cap B|}{|A \cup B|}
$$

Range: 0 (no overlap) to 1 (perfect match). IoU $\geq$ 0.5 is typically called a "match"; IoU $\geq$ 0.75 is strict.

```python
import numpy as np

def iou(box_a: np.ndarray, box_b: np.ndarray) -> float:
    """boxes: [x1, y1, x2, y2]"""
    xi1 = max(box_a[0], box_b[0])
    yi1 = max(box_a[1], box_b[1])
    xi2 = min(box_a[2], box_b[2])
    yi2 = min(box_a[3], box_b[3])

    inter_w = max(0, xi2 - xi1)
    inter_h = max(0, yi2 - yi1)
    inter   = inter_w * inter_h

    area_a  = (box_a[2] - box_a[0]) * (box_a[3] - box_a[1])
    area_b  = (box_b[2] - box_b[0]) * (box_b[3] - box_b[1])
    union   = area_a + area_b - inter
    return inter / (union + 1e-7)

box1 = np.array([0, 0, 10, 10])
box2 = np.array([5, 5, 15, 15])
box3 = np.array([20, 20, 30, 30])

print(f"IoU(box1, box2) = {iou(box1, box2):.3f}")   # partial overlap
print(f"IoU(box1, box1) = {iou(box1, box1):.3f}")   # perfect
print(f"IoU(box1, box3) = {iou(box1, box3):.3f}")   # no overlap
```

```output
IoU(box1, box2) = 0.143
IoU(box1, box1) = 1.000
IoU(box1, box3) = 0.000
```

## Anchor boxes

Instead of predicting absolute box coordinates (unbounded, hard to regress), detectors place a grid of **anchor boxes** — predefined rectangles at multiple scales and aspect ratios — and predict **offsets** relative to each anchor.

Common anchor setup: at each grid cell, 3 scales × 3 aspect ratios = 9 anchors.

```python
def generate_anchors(
    feature_hw: tuple, stride: int,
    scales: list = [32, 64, 128],
    ratios: list = [0.5, 1.0, 2.0]
) -> np.ndarray:
    """Returns anchors in [x1, y1, x2, y2] for each (cell, scale, ratio)."""
    H, W = feature_hw
    anchors = []
    for row in range(H):
        for col in range(W):
            cx = (col + 0.5) * stride
            cy = (row + 0.5) * stride
            for s in scales:
                for r in ratios:
                    w = s * np.sqrt(r)
                    h = s / np.sqrt(r)
                    anchors.append([cx - w/2, cy - h/2, cx + w/2, cy + h/2])
    return np.array(anchors)

anchors = generate_anchors((7, 7), stride=32)
print(f"Feature map 7×7 → {len(anchors)} anchors")
print("First anchor (cx=16, cy=16):", anchors[0].round(1))
```

```output
Feature map 7×7 → 441 anchors
First anchor (cx=16, cy=16): [  -6.7  -27.3   38.7   59.3]
```

## Box parameterization

Rather than predicting $(x_1, y_1, x_2, y_2)$ directly, detectors predict offsets relative to an anchor $(a_x, a_y, a_w, a_h)$:

$$
t_x = (g_x - a_x) / a_w, \quad t_y = (g_y - a_y) / a_h
$$
$$
t_w = \log(g_w / a_w), \quad t_h = \log(g_h / a_h)
$$

where $(g_x, g_y, g_w, g_h)$ is the ground-truth box centre and size. The log prevents negative width/height predictions.

```python
def encode_box(gt: np.ndarray, anchor: np.ndarray) -> np.ndarray:
    """gt, anchor: [x1,y1,x2,y2] → encoded [tx,ty,tw,th]"""
    gt_cx, gt_cy = (gt[0]+gt[2])/2, (gt[1]+gt[3])/2
    gt_w,  gt_h  = gt[2]-gt[0], gt[3]-gt[1]
    a_cx,  a_cy  = (anchor[0]+anchor[2])/2, (anchor[1]+anchor[3])/2
    a_w,   a_h   = anchor[2]-anchor[0], anchor[3]-anchor[1]
    return np.array([
        (gt_cx - a_cx) / a_w,
        (gt_cy - a_cy) / a_h,
        np.log(gt_w / a_w),
        np.log(gt_h / a_h),
    ])

gt     = np.array([50., 50., 150., 200.])
anchor = np.array([40., 40., 140., 190.])
enc    = encode_box(gt, anchor)
print("Encoded:", enc.round(3))
```

```output
Encoded: [0.    0.    0.    0.048]
```

Near-zero offsets mean the anchor already closely matches the ground-truth.

## Non-Maximum Suppression (NMS)

A detector may fire many overlapping boxes for the same object. NMS keeps only the best:

1. Sort boxes by score (descending).
2. Take the highest-score box. Remove all others with IoU ≥ threshold against it.
3. Repeat on remaining boxes.

```python
def nms(boxes: np.ndarray, scores: np.ndarray, iou_threshold: float = 0.5) -> list:
    """
    boxes: (N, 4) [x1,y1,x2,y2]
    scores: (N,)
    returns indices of kept boxes
    """
    order   = scores.argsort()[::-1]
    keep    = []
    while len(order) > 0:
        i = order[0]
        keep.append(i)
        ious = np.array([iou(boxes[i], boxes[j]) for j in order[1:]])
        order = order[1:][ious < iou_threshold]
    return keep

boxes  = np.array([
    [10, 10, 80, 80],
    [15, 12, 85, 82],   # overlaps box 0 heavily
    [12, 11, 83, 83],   # overlaps box 0 heavily
    [200, 200, 280, 280],  # separate box
])
scores = np.array([0.95, 0.88, 0.76, 0.91])
kept   = nms(boxes, scores, iou_threshold=0.5)
print("Kept box indices:", kept)
print("Kept boxes:\n", boxes[kept])
```

```output
Kept box indices: [0, 3]
Kept boxes:
 [[ 10  10  80  80]
  [200 200 280 280]]
```

Boxes 1 and 2 were suppressed because they heavily overlapped box 0 (the highest-score box).

## Soft-NMS

Hard NMS discards boxes above the IoU threshold. Soft-NMS decays scores instead:

$$
s_i = s_i \cdot e^{-\text{IoU}(M, b_i)^2 / \sigma}
$$

then remove only boxes with score below a minimum. This helps when two objects genuinely overlap.

## Mean Average Precision (mAP)

For each class:
1. Rank predictions by confidence.
2. Mark each prediction as TP (IoU ≥ 0.5 with a GT box) or FP.
3. Compute a precision–recall curve.
4. Interpolate: $\text{AP} = \sum_{r} \text{precision}(r) \cdot \Delta r$ over 11 or 101 points.

mAP@0.5 = average AP across all classes at IoU threshold 0.5. COCO mAP averages over IoU 0.5:0.05:0.95.

```python
def compute_ap(precisions: np.ndarray, recalls: np.ndarray) -> float:
    """11-point interpolated AP."""
    ap = 0.0
    for thr in np.linspace(0, 1, 11):
        p = precisions[recalls >= thr]
        ap += (p.max() if len(p) > 0 else 0.0)
    return ap / 11

# Synthetic PR curve
recalls    = np.array([0.0, 0.1, 0.3, 0.5, 0.7, 0.9, 1.0])
precisions = np.array([1.0, 0.9, 0.8, 0.7, 0.5, 0.3, 0.2])
print(f"AP = {compute_ap(precisions, recalls):.3f}")
```

```output
AP = 0.627
```

## Exercise

Implement `batch_iou(boxes_a, boxes_b)` that computes the IoU of every pair in an $(N, 4)$ array with every box in an $(M, 4)$ array, returning an $(N, M)$ matrix — all in NumPy without explicit loops. This is the vectorized IoU needed inside fast NMS and anchor assignment.

<summary>Solution</summary>

```python
import numpy as np

def batch_iou(boxes_a: np.ndarray, boxes_b: np.ndarray) -> np.ndarray:
    """
    boxes_a: (N, 4)  boxes_b: (M, 4)  returns (N, M)
    """
    # Expand dims for broadcasting: (N,1,4) vs (1,M,4)
    a = boxes_a[:, None, :]  # (N, 1, 4)
    b = boxes_b[None, :, :]  # (1, M, 4)

    xi1 = np.maximum(a[..., 0], b[..., 0])
    yi1 = np.maximum(a[..., 1], b[..., 1])
    xi2 = np.minimum(a[..., 2], b[..., 2])
    yi2 = np.minimum(a[..., 3], b[..., 3])

    inter = np.maximum(0, xi2 - xi1) * np.maximum(0, yi2 - yi1)
    area_a = (boxes_a[:, 2] - boxes_a[:, 0]) * (boxes_a[:, 3] - boxes_a[:, 1])
    area_b = (boxes_b[:, 2] - boxes_b[:, 0]) * (boxes_b[:, 3] - boxes_b[:, 1])
    union  = area_a[:, None] + area_b[None, :] - inter
    return inter / (union + 1e-7)

A = np.array([[0,0,10,10],[5,5,15,15]], dtype=float)
B = np.array([[0,0,10,10],[20,20,30,30]], dtype=float)
print(batch_iou(A, B))
```

The key is broadcasting `(N,1,4)` against `(1,M,4)` — all pairwise operations vectorize without loops, giving $O(NM)$ time with minimal Python overhead.
</details>
