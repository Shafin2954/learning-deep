## Combining DataFrames — four operations

| Operation | pandas function | SQL analogy |
|-----------|----------------|-------------|
| Stack rows | `pd.concat(axis=0)` | `UNION ALL` |
| Stack cols | `pd.concat(axis=1)` | column join on index |
| Key-based join | `pd.merge(on=key)` | `JOIN` |
| Reshape long↔wide | `pivot_table` / `melt` | `PIVOT` / `UNPIVOT` |

## `pd.concat` — stacking DataFrames

```python
import pandas as pd

df1 = pd.DataFrame({'a': [1, 2], 'b': [3, 4]})
df2 = pd.DataFrame({'a': [5, 6], 'b': [7, 8]})

# Stack rows (axis=0) — columns must match
print(pd.concat([df1, df2], ignore_index=True))
```

```output
   a  b
0  1  3
1  2  4
2  5  7
3  6  8
```

```python
import pandas as pd

df1 = pd.DataFrame({'a': [1, 2]})
df2 = pd.DataFrame({'b': [3, 4]})

# Stack columns (axis=1) — rows aligned on index
print(pd.concat([df1, df2], axis=1))
```

```output
   a  b
0  1  3
1  2  4
```

Use `ignore_index=True` when row indices are meaningless after stacking.
Without it, the original indices are preserved (including duplicates from multiple files).

## `pd.merge` — SQL-style joins

The default is an inner join on the common column name. Always specify `how` explicitly.

```python
import pandas as pd

orders = pd.DataFrame({
    'order_id':   [1, 2, 3, 4],
    'customer_id': [101, 102, 101, 103],
    'amount':      [500, 300, 750, 200],
})
customers = pd.DataFrame({
    'customer_id': [101, 102, 104],
    'name':        ['alice', 'bob', 'carol'],
    'city':        ['Dhaka', 'Chittagong', 'Sylhet'],
})

# Inner join: only matching rows
inner = pd.merge(orders, customers, on='customer_id', how='inner')
print("inner:\n", inner)
```

```output
inner:
    order_id  customer_id  amount   name        city
0         1          101     500  alice       Dhaka
1         3          101     750  alice       Dhaka
2         2          102     300    bob  Chittagong
```

```python
import pandas as pd

orders = pd.DataFrame({
    'order_id':   [1, 2, 3, 4],
    'customer_id': [101, 102, 101, 103],
    'amount':      [500, 300, 750, 200],
})
customers = pd.DataFrame({
    'customer_id': [101, 102, 104],
    'name':        ['alice', 'bob', 'carol'],
    'city':        ['Dhaka', 'Chittagong', 'Sylhet'],
})

# Left join: keep all orders, NaN for unmatched customers
left = pd.merge(orders, customers, on='customer_id', how='left')
print("left:\n", left)
```

```output
left:
    order_id  customer_id  amount   name        city
0         1          101     500  alice       Dhaka
1         2          102     300    bob  Chittagong
2         3          101     750  alice       Dhaka
3         4          103     200    NaN         NaN
```

Order 4 has `customer_id=103` which doesn't appear in the customers table → NaN.
Order 3 appears twice because customer 101 placed two orders, and the join expands it.

### Left vs inner: when it matters

If your downstream analysis drops NaN rows, a left join looks like an inner join.
Make the join type explicit based on your *intent*, not what "happens to work."

## `melt` — wide to long

A "wide" table has one column per time period or category. Analysis prefers "long" (tidy)
format: one observation per row.

```python
import pandas as pd

wide = pd.DataFrame({
    'product': ['alpha', 'beta'],
    'jan':     [100, 200],
    'feb':     [110, 190],
    'mar':     [120, 210],
})
print("wide:\n", wide)

long = wide.melt(id_vars='product', var_name='month', value_name='sales')
print("\nlong:\n", long.sort_values(['product', 'month']).reset_index(drop=True))
```

```output
wide:
   product  jan  feb  mar
0   alpha  100  110  120
1    beta  200  190  210

long:
   product month  sales
0   alpha   feb    110
1   alpha   jan    100
2   alpha   mar    120
3    beta   feb    190
4    beta   jan    200
5    beta   mar    210
```

## `pivot_table` — long to wide with aggregation

`pivot_table` is the inverse: it takes a long table and spreads values into columns.
It also aggregates — so if there are multiple values for a `(row, col)` combination,
it applies a function (default: mean).

```python
import pandas as pd

long = pd.DataFrame({
    'product': ['alpha','alpha','alpha','beta','beta','beta'],
    'month':   ['jan','feb','mar','jan','feb','mar'],
    'sales':   [100, 110, 120, 200, 190, 210],
})

wide = long.pivot_table(
    index='product', columns='month', values='sales', aggfunc='sum'
)
print(wide)
```

```output
month   feb  jan  mar
product
alpha   110  100  120
beta    190  200  210
```

```python
import pandas as pd

long = pd.DataFrame({
    'product': ['alpha','alpha','alpha','beta','beta','beta'],
    'month':   ['jan','feb','mar','jan','feb','mar'],
    'sales':   [100, 110, 120, 200, 190, 210],
})

# Flatten multi-level column index if needed
wide = long.pivot_table(index='product', columns='month', values='sales', aggfunc='sum')
wide.columns.name = None  # remove 'month' label from column axis
wide = wide.reset_index()
print(wide)
```

```output
  product  feb  jan  mar
0   alpha  110  100  120
1    beta  190  200  210
```

## `stack` and `unstack` — index-level reshaping

`stack` moves a column level into the row index. `unstack` does the reverse. Use these
when you already have a MultiIndex and want to reshape it — they are faster than
`pivot_table`/`melt` for this case.

```python
import pandas as pd

df = pd.DataFrame({
    'jan': [100, 200],
    'feb': [110, 190],
}, index=['alpha', 'beta'])

stacked = df.stack()    # columns → innermost row level
print(stacked)
```

```output
alpha  jan    100
       feb    110
beta   jan    200
       feb    190
dtype: int64
```

## Try it Yourself

```python
import pandas as pd
import numpy as np

# Build a small sales dataset with missing data
rng = np.random.default_rng(11)
N = 20
df = pd.DataFrame({
    'store':    rng.choice(['S1', 'S2', 'S3'], N),
    'product':  rng.choice(['p', 'q'], N),
    'month':    rng.choice(['jan', 'feb', 'mar'], N),
    'revenue':  rng.integers(100, 500, N),
})

# 1) Pivot: average revenue by store (rows) and month (columns)
piv = df.pivot_table(index='store', columns='month', values='revenue',
                     aggfunc='mean', fill_value=0).round(0)
print("Pivot:\n", piv)

# 2) Melt it back and verify shape
long = piv.reset_index().melt(id_vars='store', var_name='month', value_name='avg_rev')
print("\nMelted shape:", long.shape)

# 3) Merge with a store metadata table
meta = pd.DataFrame({
    'store': ['S1', 'S2', 'S3'],
    'region': ['north', 'south', 'east'],
})
merged = pd.merge(long, meta, on='store', how='left')
print(merged.head(4))
```

```output
Pivot:

store
S1      211.0  241.0  339.0
S2      354.0  302.0  283.0
S3      247.0  176.0  371.0

Melted shape: (9, 3)
  store month  avg_rev region
0    S1   feb    211.0  north
1    S2   feb    354.0  south
2    S3   feb    247.0   east
3    S1   jan    241.0  north
```

## Exercise

Start with a wide DataFrame of student scores (rows = students, columns = subjects). Then:

1. Melt it to long format.
2. Compute the mean and max score per subject using groupby on the long frame.
3. Pivot back to wide format (one row per student, one col per subject) — verify you recover
   the original shape.

<details>
<summary>Solution</summary>

```python
import pandas as pd

wide = pd.DataFrame({
    'student': ['alice', 'bob', 'carol'],
    'math':    [85, 70, 90],
    'english': [78, 88, 82],
    'science': [92, 65, 88],
})

# 1) Melt
long = wide.melt(id_vars='student', var_name='subject', value_name='score')

# 2) Stats per subject
stats = long.groupby('subject')['score'].agg(['mean', 'max'])
print(stats)

# 3) Pivot back — should give the same shape as wide (minus subject order)
back = long.pivot_table(index='student', columns='subject', values='score')
back.columns.name = None
back = back.reset_index()
print("\nRecovered shape:", back.shape, "  original:", wide.shape)
```
