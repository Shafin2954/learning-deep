## The selection landscape

pandas has more selection syntax than any other library in the ecosystem. Knowing exactly
which accessor to reach for — and why — prevents a whole class of subtle bugs.

| Accessor | Basis | Returns |
|----------|-------|---------|
| `df[col]` | column name | Series |
| `df[[c1, c2]]` | list of column names | DataFrame |
| `df[bool_arr]` | boolean mask on rows | DataFrame |
| `.loc[row_label, col_label]` | label | scalar/Series/DataFrame |
| `.iloc[row_int, col_int]` | integer position | scalar/Series/DataFrame |
| `.query(expr)` | expression string | DataFrame |

The root confusion: `df[...]` means different things depending on what you pass. Use `.loc`
and `.iloc` for explicit, unambiguous selection.

## `loc` — label-based

`.loc[row_selector, col_selector]`. Both selectors use labels. Slices are **inclusive** at
both ends (unlike Python slices).

```python
import pandas as pd

df = pd.DataFrame({
    'score': [88, 75, 92, 60],
    'grade': ['B', 'C', 'A', 'D'],
}, index=['alice', 'bob', 'carol', 'dave'])

print(df.loc['alice'])                  # single row → Series
print()
print(df.loc['bob':'carol', 'score'])   # inclusive slice + column
print()
print(df.loc[['alice', 'dave'], :])     # fancy selection
```

```output
score    88
grade     B
Name: alice, dtype: object

bob      75
carol    92
Name: score, dtype: int64

       score grade
alice     88     B
dave      60     D
```

## `iloc` — position-based

`.iloc[row_int, col_int]`. Uses integer positions. Slices are **exclusive** at the end
(standard Python). Use this when you care about order, not labels — e.g. "the first 10 rows".

```python
import pandas as pd

df = pd.DataFrame({
    'a': [10, 20, 30, 40],
    'b': [1.1, 2.2, 3.3, 4.4],
    'c': ['w', 'x', 'y', 'z'],
})

print(df.iloc[1:3])                   # rows 1,2 (exclusive 3)
print()
print(df.iloc[[0, 3], 1:])            # rows 0 and 3, columns from position 1 onward
print()
print(df.iloc[-1, -1])                # last row, last col
```

```output
    a    b  c
1  20  2.2  x
2  30  3.3  y

      b  c
0   1.1  w
3   4.4  z

z
```

### Mixing loc and iloc — don't

Never mix `.loc` and `.iloc` in one call. `df.loc[0, 'a']` is label-based (row label `0`,
column label `'a'`). If the index happens to be a RangeIndex, labels and positions coincide,
which hides bugs until you reset the index.

## Boolean selection — filter rows with a mask

The most common selection in data work: keep rows satisfying a condition.

```python
import pandas as pd

df = pd.DataFrame({
    'city':  ['Dhaka', 'Dhaka', 'Chittagong', 'Dhaka', 'Sylhet'],
    'sales': [120, 90, 150, 200, 60],
    'year':  [2022, 2023, 2022, 2023, 2023],
})

# Single condition
print(df[df['sales'] > 100])
```

```output
         city  sales  year
0       Dhaka    120  2022
2  Chittagong    150  2022
3       Dhaka    200  2023
```

```python
import pandas as pd

df = pd.DataFrame({
    'city':  ['Dhaka', 'Dhaka', 'Chittagong', 'Dhaka', 'Sylhet'],
    'sales': [120, 90, 150, 200, 60],
    'year':  [2022, 2023, 2022, 2023, 2023],
})

# Compound condition — wrap each part in parens!
high_dhaka_2023 = df[(df['city'] == 'Dhaka') & (df['year'] == 2023) & (df['sales'] > 100)]
print(high_dhaka_2023)
```

```output
    city  sales  year
3  Dhaka    200  2023
```

Use `.isin()` to replace a chain of `|` conditions:

```python
import pandas as pd

df = pd.DataFrame({
    'city':  ['Dhaka', 'Dhaka', 'Chittagong', 'Dhaka', 'Sylhet'],
    'sales': [120, 90, 150, 200, 60],
    'year':  [2022, 2023, 2022, 2023, 2023],
})

big_cities = df[df['city'].isin(['Dhaka', 'Chittagong'])]
print(big_cities['city'].value_counts())
```

```output
city
Dhaka         3
Chittagong    1
Name: count, dtype: int64
```

## `.query()` — readable string expressions

`.query(expr)` evaluates a string expression against column names. Good for long chains of
conditions; bad for computed values or column names with special characters.

```python
import pandas as pd

df = pd.DataFrame({
    'city':  ['Dhaka', 'Dhaka', 'Chittagong', 'Dhaka', 'Sylhet'],
    'sales': [120, 90, 150, 200, 60],
    'year':  [2022, 2023, 2022, 2023, 2023],
})

# Equivalent to the compound condition above
result = df.query("city == 'Dhaka' and year == 2023 and sales > 100")
print(result)

# Reference a Python variable with @
threshold = 100
result2 = df.query("sales > @threshold and year == 2023")
print(result2)
```

```output
    city  sales  year
3  Dhaka    200  2023
    city  sales  year
3  Dhaka    200  2023
```

`.query()` is cleaner to read but slower for small DataFrames (it parses the string). For
DataFrames with millions of rows it can be faster because it uses numexpr internally.

## Assignment to selections — `loc` only

Always use `.loc` for assignment. Never use chained indexing (`df[...][...] = value`) — it
may or may not modify the original DataFrame depending on whether the intermediate is a copy.

```python
import pandas as pd

df = pd.DataFrame({
    'city':  ['Dhaka', 'Dhaka', 'Chittagong'],
    'sales': [120, 90, 150],
})

# Correct: loc-based assignment
df.loc[df['sales'] < 100, 'sales'] = 0
print(df)

# WRONG (may raise SettingWithCopyWarning or silently do nothing):
# df[df['sales'] < 100]['sales'] = 0   ← don't do this
```

```output
         city  sales
0       Dhaka    120
1       Dhaka      0
2  Chittagong    150
```

## Try it Yourself

```python
import pandas as pd
import numpy as np

rng = np.random.default_rng(9)
N = 200
df = pd.DataFrame({
    'category': rng.choice(['A', 'B', 'C'], N),
    'value':    rng.normal(50, 10, N).round(1),
    'flag':     rng.integers(0, 2, N).astype(bool),
})

# 1) What fraction of rows are category 'A' AND flag is True?
frac = ((df['category'] == 'A') & df['flag']).mean()
print(f"A-and-flag fraction: {frac:.3f}")

# 2) Use .loc to cap values above 70 to exactly 70
df.loc[df['value'] > 70, 'value'] = 70.0
print("max after cap:", df['value'].max())

# 3) Select the last 5 rows using iloc and show only 'value'
print(df.iloc[-5:][['value']])
```

```output
A-and-flag fraction: 0.175
max after cap: 70.0
   value
195   49.4
196   42.2
197   50.7
198   63.1
199   44.8
```

## Exercise

Create a DataFrame with columns `product`, `region`, `revenue`, `units` (8+ rows, multiple
products and regions). Then answer three questions using `.loc`, boolean masks, and `.query`:

1. Which rows have revenue per unit above 50?
2. Which products appear in all regions?
3. Set revenues below 100 to 0 using `.loc` assignment.

<details>
<summary>Solution</summary>

```python
import pandas as pd

df = pd.DataFrame({
    'product': ['X','X','X','Y','Y','Y','Z','Z'],
    'region':  ['N','S','E','N','S','E','N','S'],
    'revenue': [300, 80, 200, 150, 50, 400, 90, 600],
    'units':   [3, 2, 4, 5, 1, 6, 2, 8],
})

# 1) revenue per unit > 50
df['rev_per_unit'] = df['revenue'] / df['units']
print(df.loc[df['rev_per_unit'] > 50, ['product', 'region', 'rev_per_unit']])

# 2) Products in all regions
all_regions = set(df['region'])
in_all = df.groupby('product')['region'].apply(set).apply(lambda s: s >= all_regions)
print("in all regions:", in_all[in_all].index.tolist())

# 3) Set revenues below 100 to 0
df.loc[df['revenue'] < 100, 'revenue'] = 0
print(df)
```

Always use `.loc` for assignment — chained indexing (`df[mask]['col'] = v`) silently
fails with newer pandas, raising `SettingWithCopyWarning` or doing nothing.
</details>
