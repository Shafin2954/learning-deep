## What pandas adds to NumPy

NumPy's ndarray is great for homogeneous numerical arrays. Real data is heterogeneous:
one column of integers, one of strings, one of dates, with named columns and meaningful
row labels. pandas wraps NumPy arrays in a layer of **labeled axes** called the **Index**.

Three objects do essentially everything in pandas:

- **Index** — an immutable array of labels for rows or columns.
- **Series** — a 1-D labeled array (`index` + `values`).
- **DataFrame** — a 2-D table (`row index` + `column index` + one Series per column).

## Series — the labeled 1-D array

```python
import pandas as pd

# Integer values with a default RangeIndex
s = pd.Series([10, 20, 30], name='score')
print(s)
```

```output
0    10
1    20
2    30
Name: score, dtype: int64
```

```python
import pandas as pd

# Custom index — can be any hashable type
s = pd.Series([10, 20, 30], index=['alice', 'bob', 'carol'], name='score')
print(s)
print("dtype  :", s.dtype)
print("values :", s.values)    # plain NumPy array
print("index  :", s.index.tolist())
```

```output
alice    10
bob      20
carol    30
Name: score, dtype: int64
dtype  : int64
values : [10 20 30]
index  : ['alice', 'bob', 'carol']
```

You can access values by label (`s['bob']`) or by integer position (`s.iloc[1]`). The
former uses the index; the latter ignores it. That distinction becomes critical in ch. 5.

### Index alignment — the killer feature

When you add two Series, pandas aligns on the **index**, not on position.

```python
import pandas as pd

a = pd.Series([1, 2, 3], index=['x', 'y', 'z'])
b = pd.Series([10, 20, 30], index=['y', 'z', 'w'])

print(a + b)
```

```output
w     NaN
x     NaN
y    22.0
z    33.0
dtype: float64
```

Labels that exist in both: summed. Labels that appear in only one: `NaN`. This is what
makes pandas safe for misaligned data — you can't accidentally add a value for 'alice' to
a value for 'bob'. The dtype widened to float64 to accommodate NaN (NumPy int64 has no NaN).

## DataFrame — the labeled 2-D table

```python
import pandas as pd

df = pd.DataFrame({
    'name':  ['alice', 'bob', 'carol'],
    'score': [88, 75, 92],
    'grade': ['B', 'C', 'A'],
})
print(df)
print()
print("dtypes:\n", df.dtypes)
```

```output
    name  score grade
0  alice     88     B
1    bob     75     C
2  carol     92     A

dtypes:
 name     object
score     int64
grade    object
dtype: object
```

Each column is a **Series** sharing the same row index. `df['score']` returns the Series for
that column. `df[['score', 'grade']]` returns a new DataFrame with two columns.

### Accessing columns and the column index

```python
import pandas as pd

df = pd.DataFrame({'a': [1, 2, 3], 'b': [4, 5, 6]})

print(df.columns.tolist())         # ['a', 'b']
print(df['a'])                     # Series
print(type(df[['a']]))             # DataFrame (double brackets)
```

```output
['a', 'b']
0    1
1    2
2    3
Name: a, dtype: int64
<class 'pandas.core.frame.DataFrame'>
```

`df['a']` (single string) → Series. `df[['a']]` (list) → DataFrame. This inconsistency is
one of the oldest pandas gotchas; keep it in mind.

## The Index object

An Index is **immutable** (you can't assign to a single label) and **hashable** (supports
O(1) label look-up). It's not just column names — it's the row identifier that makes
alignment work.

```python
import pandas as pd

df = pd.DataFrame({'x': [10, 20, 30]}, index=['a', 'b', 'c'])
print("index  :", df.index.tolist())
print("columns:", df.columns.tolist())

# Reset index — moves the index into a column
df2 = df.reset_index()
print(df2)

# Set a column as the index
df3 = df2.set_index('index')
print(df3)
```

```output
index  : ['a', 'b', 'c']
columns: ['x']
  index   x
0     a  10
1     b  20
2     c  30
  x
index
a  10
b  20
c  30
```

`reset_index()` and `set_index()` are the primary tools for moving data between the index
and the columns. Keeping the right things in the index — especially after a groupby — is a
common source of confusion in ch. 6.

## Memory layout: pandas holds NumPy arrays

Internally, pandas groups columns of the same dtype into **blocks** — each block is a single
NumPy 2-D array. This is why a DataFrame with 5 float64 columns costs one allocation, not
five. You can inspect it with `df._data` (private) or `df.dtypes`.

```python
import pandas as pd
import numpy as np

df = pd.DataFrame({
    'a': np.ones(3, dtype=np.float64),
    'b': np.zeros(3, dtype=np.float64),
    'c': np.full(3, 2, dtype=np.int32),
})

print("Memory (deep):", df.memory_usage(deep=True).sum(), "bytes")
# float64 block: 3*8*2 = 48; int32 block: 3*4 = 12; index: 24 → ~84 + overhead
```

```output
Memory (deep): 152 bytes
```

Understanding the block layout matters in ch. 11 (performance): converting an `object`
column to `category` or int reduces memory by 10–100×, and also un-fragments the blocks.

## Try it Yourself

```python
import pandas as pd

# 1) Alignment in action: join two misaligned Series
gdp = pd.Series({'US': 25.5, 'EU': 17.1, 'CN': 17.9})    # 2023 GDP ~trillions
pop = pd.Series({'US': 335, 'CN': 1412, 'IN': 1417})      # ~millions

gdp_per_capita = (gdp * 1e12) / (pop * 1e6)
print("GDP per capita (rough):")
print(gdp_per_capita.dropna().apply(lambda x: f"${x:,.0f}"))

# 2) DataFrame from a dict of lists — verify column dtypes
import numpy as np
df = pd.DataFrame({
    'age':  [25, 32, 28],
    'wage': [55000.0, 72000.0, 61000.0],
    'city': ['Dhaka', 'Dhaka', 'Chittagong'],
})
print("\ndtypes:\n", df.dtypes)
print("memory:", df.memory_usage(deep=True).sum(), "bytes")
```

```output
GDP per capita (rough):
US    $76,119
CN    $12,676
dtype: object
EU      NaN
IN      NaN
dtype: object

dtypes:
 age       int64
wage    float64
city     object
dtype: object
memory: 430 bytes
```

## Exercise

Create a DataFrame with columns `product`, `units_sold`, `price` (at least 4 rows). Then:

1. Add a `revenue` column (units_sold × price) without a loop.
2. Set `product` as the index.
3. Reset it back and verify the index is a `RangeIndex` again.

<details>
<summary>Solution</summary>

```python
import pandas as pd

df = pd.DataFrame({
    'product':    ['alpha', 'beta', 'gamma', 'delta'],
    'units_sold': [120, 45, 200, 30],
    'price':      [9.99, 24.99, 4.99, 49.99],
})

# 1) Revenue — vectorized multiplication of two Series (aligned on RangeIndex)
df['revenue'] = df['units_sold'] * df['price']
print(df)

# 2) Set index
df = df.set_index('product')
print("\nindex type:", type(df.index).__name__)

# 3) Reset
df = df.reset_index()
print("after reset:", type(df.index).__name__)
```

The multiplication aligns on the shared RangeIndex — same position, same label — so it
behaves identically to NumPy element-wise multiply here. When the indices differ, pandas
aligns first (ch. 5 and 7 cover this in detail).
</details>
