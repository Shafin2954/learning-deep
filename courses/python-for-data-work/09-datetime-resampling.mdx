## Datetime in pandas: three objects

| Object | What it is | Created from |
|--------|------------|-------------|
| `pd.Timestamp` | a single point in time | string, Python datetime |
| `pd.DatetimeIndex` | an ordered array of timestamps | `pd.date_range`, `pd.to_datetime` |
| `pd.Timedelta` | a duration | subtraction, `pd.Timedelta('3 days')` |

Arithmetic between Timestamps gives Timedeltas. Indexing a Series on a DatetimeIndex
gives you partial-string selection ("2023" selects all of 2023).

## Parsing datetimes

```python
import pandas as pd

# From strings — pandas auto-detects many formats
dates = ['2023-01-15', '2023-02-28', '2023-03-31']
idx = pd.to_datetime(dates)
print(idx)
print("dtype:", idx.dtype)
```

```output
DatetimeIndex(['2023-01-15', '2023-02-28', '2023-03-31'], dtype='datetime64[ns]', freq=None)
dtype: datetime64[ns]
```

```python
import pandas as pd

# Non-standard formats: pass format= explicitly (faster and unambiguous)
s = pd.Series(['15/01/2023', '28/02/2023', '31/03/2023'])
parsed = pd.to_datetime(s, format='%d/%m/%Y')
print(parsed)
```

```output
0   2023-01-15
1   2023-02-28
2   2023-03-31
dtype: datetime64[ns]
```

Always pass `format=` when the format is known — auto-detection is slow on large Series and
can silently mis-parse ambiguous dates like `01/02/03`.

## The `.dt` accessor

Once a column is `datetime64`, the `.dt` accessor exposes all components without a loop:

```python
import pandas as pd

df = pd.DataFrame({
    'date': pd.to_datetime(['2023-01-15', '2023-06-28', '2023-12-31']),
    'value': [100, 200, 300],
})

df['year']       = df['date'].dt.year
df['month']      = df['date'].dt.month
df['day_of_week'] = df['date'].dt.dayofweek   # 0=Monday, 6=Sunday
df['quarter']    = df['date'].dt.quarter
df['is_weekend'] = df['date'].dt.dayofweek >= 5
print(df)
```

```output
        date  value  year  month  day_of_week  quarter  is_weekend
0 2023-01-15    100  2023      1            6        1        True
1 2023-06-28    200  2023      6            2        2       False
2 2023-12-31    300  2023     12            0        4       False
```

These are vectorized operations — no `.apply()` needed.

## `date_range` — generating regular time series

```python
import pandas as pd

# Daily index for January 2023
idx = pd.date_range('2023-01-01', '2023-01-07', freq='D')
print(idx)

# Monthly business-day start
idx2 = pd.date_range('2023-01-01', periods=4, freq='MS')  # MS = month start
print(idx2)
```

```output
DatetimeIndex(['2023-01-01', '2023-01-02', '2023-01-03', '2023-01-04',
               '2023-01-05', '2023-01-06', '2023-01-07'],
              dtype='datetime64[ns]', freq='D')
DatetimeIndex(['2023-01-01', '2023-02-01', '2023-03-01', '2023-04-01'], dtype='datetime64[ns]', freq='MS')
```

## Resampling — aggregating by time bucket

`resample` is `groupby` for time series. It takes a frequency string and aggregates values
within each time bucket. The Series/DataFrame must have a `DatetimeIndex`.

```python
import pandas as pd
import numpy as np

rng = np.random.default_rng(20)
idx = pd.date_range('2023-01-01', periods=90, freq='D')   # 90 days
s = pd.Series(rng.integers(50, 150, 90), index=idx, name='sales')

# Weekly totals (W-MON = week ending Monday)
weekly = s.resample('W').sum()
print(weekly.head(4))
```

```output
2023-01-02     97
2023-01-09    620
2023-01-16    671
2023-01-23    728
Freq: W-MON, Name: sales, dtype: int64
```

```python
import pandas as pd
import numpy as np

rng = np.random.default_rng(20)
idx = pd.date_range('2023-01-01', periods=90, freq='D')
s = pd.Series(rng.integers(50, 150, 90), index=idx, name='sales')

# Monthly mean
monthly = s.resample('ME').mean().round(1)   # ME = month end
print(monthly)
```

```output
2023-01-31    102.0
2023-02-28    103.6
2023-03-31     96.3
Freq: ME, Name: sales, dtype: float64
```

Common frequency strings: `'D'` (day), `'W'` (week), `'ME'` (month end), `'MS'` (month
start), `'QE'` (quarter end), `'YE'` (year end), `'h'` (hour), `'min'` (minute).

## Rolling windows

`rolling(n)` computes a statistic over the last `n` periods at each point. It's not a
resample (it doesn't change the index frequency) — every row gets a value.

```python
import pandas as pd
import numpy as np

rng = np.random.default_rng(20)
idx = pd.date_range('2023-01-01', periods=10, freq='D')
s = pd.Series(rng.integers(50, 150, 10), index=idx, name='sales')

df = pd.DataFrame({
    'sales': s,
    'roll7': s.rolling(7).mean().round(1),     # 7-day moving average
    'roll3_std': s.rolling(3).std().round(1),  # 3-day rolling std
})
print(df)
```

```output
            sales  roll7  roll3_std
2023-01-01     97    NaN        NaN
2023-01-02     86    NaN        NaN
2023-01-03    120    NaN        9.0  ← 3 values (window fills at position 3)
2023-01-04    134    NaN       24.7
2023-01-05    100    NaN       17.6
2023-01-06    116    NaN       17.6
2023-01-07     78   104.4      19.1  ← 7-day MA fills at position 7
2023-01-08    143   111.0      34.1
2023-01-09     63   107.7      41.0
2023-01-10    109   106.1      40.3
```

The first `n-1` values are NaN because the window isn't full yet. Use `min_periods=1` to
start computing with partial windows instead.

## Time-based indexing and slicing

With a DatetimeIndex, `.loc` accepts partial date strings for natural-language slicing:

```python
import pandas as pd
import numpy as np

rng = np.random.default_rng(20)
idx = pd.date_range('2023-01-01', periods=90, freq='D')
s = pd.Series(rng.integers(50, 150, 90), index=idx, name='sales')

# All of February 2023
feb = s.loc['2023-02']
print(f"February: {len(feb)} days, total={feb.sum()}")

# Jan 10 to Feb 5 inclusive
span = s.loc['2023-01-10':'2023-02-05']
print(f"Jan-10 to Feb-05: {len(span)} days")
```

```output
February: 28 days, total=2901
Jan-10 to Feb-05: 27 days
```

## Try it Yourself

```python
import pandas as pd
import numpy as np

# Simulate hourly electricity demand for 30 days
rng = np.random.default_rng(22)
idx = pd.date_range('2023-03-01', periods=30*24, freq='h')
demand = pd.Series(
    200 + 80 * np.sin(np.linspace(0, 30*2*np.pi, 30*24))   # daily cycle
    + rng.normal(0, 10, 30*24),
    index=idx, name='demand_kw'
)

# 1) Daily peak demand
daily_peak = demand.resample('D').max().round(1)
print("Daily peak (first 5):")
print(daily_peak.head(5))

# 2) 24-hour rolling mean (smoothed demand curve)
smooth = demand.rolling(24).mean()
print("\nHourly smooth (2023-03-03 noon):", round(smooth['2023-03-03 12:00'], 2))

# 3) Extract hour-of-day and find the highest-demand hour on average
df = demand.to_frame()
df['hour'] = demand.index.hour
avg_by_hour = df.groupby('hour')['demand_kw'].mean().round(1)
print("\nPeak hour:", avg_by_hour.idxmax(), "  avg demand:", avg_by_hour.max())
```

```output
Daily peak (first 5):
2023-03-01    279.9
2023-03-02    280.2
2023-03-03    280.1
2023-03-04    280.3
2023-03-05    280.1
Freq: D, Name: demand_kw, dtype: float64

Hourly smooth (2023-03-03 12:00): 199.74

Peak hour: 18   avg demand: 269.8
```

## Exercise

Simulate daily temperatures for a full year (use `pd.date_range` + a sine wave + noise).
Then compute:

1. Monthly average temperature.
2. A 30-day rolling max (to track warm spells).
3. The day of the year with the highest temperature.

<details>
<summary>Solution</summary>

```python
import pandas as pd
import numpy as np

rng = np.random.default_rng(99)
idx = pd.date_range('2023-01-01', periods=365, freq='D')
# Sine wave: peak in summer (day ~182), trough in winter
temps = 20 + 15 * np.sin((np.arange(365) - 80) * 2 * np.pi / 365) + rng.normal(0, 2, 365)
s = pd.Series(temps.round(1), index=idx, name='temp_c')

# 1) Monthly average
monthly = s.resample('ME').mean().round(2)
print("Monthly avg:\n", monthly.to_string())

# 2) 30-day rolling max
roll30 = s.rolling(30).max()
print("\nPeak rolling 30-day max:", roll30.max().round(1), "on", roll30.idxmax().date())

# 3) Hottest single day
print("Hottest day:", s.idxmax().date(), "  temp:", s.max())
```
