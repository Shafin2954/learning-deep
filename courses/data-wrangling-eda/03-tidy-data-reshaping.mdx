## What tidy data is

Hadley Wickham's 2014 paper defined three rules for "tidy" data:

1. **Each variable has its own column.**
2. **Each observation has its own row.**
3. **Each type of observational unit forms its own table.**

Violations of these rules cause almost every reshape bug in data work. When data is tidy,
groupby, pivot, and join all work without manual index gymnastics.

## Common untidy patterns and their fixes

### Pattern 1 — Column headers are values, not variable names

The months are values of a "month" variable, but they appear as column names.

```python
import pandas as pd

wide = pd.DataFrame({
    'product': ['alpha', 'beta', 'gamma'],
    'jan':     [100, 200, 150],
    'feb':     [110, 190, 160],
    'mar':     [120, 210, 140],
})
print("untidy (wide):")
print(wide)

# Fix: melt — move column names into a 'month' variable
long = wide.melt(id_vars='product', var_name='month', value_name='sales')
print("\ntidy (long):")
print(long.sort_values(['product', 'month']).reset_index(drop=True))
```

```output
untidy (wide):
  product  jan  feb  mar
0   alpha  100  110  120
1    beta  200  190  210
2   gamma  150  160  140

tidy (long):
   product month  sales
0    alpha   feb    110
1    alpha   jan    100
2    alpha   mar    120
3     beta   feb    190
4     beta   jan    200
5     beta   mar    210
6    gamma   feb    160
7    gamma   jan    150
8    gamma   mar    140
```

### Pattern 2 — Multiple variables in one column

A single `"age_sex"` column encodes two variables: age group and sex.

```python
import pandas as pd

df = pd.DataFrame({
    'region':  ['north', 'north', 'south'],
    'age_sex': ['25_M', '30_F', '25_M'],
    'count':   [120, 95, 200],
})

# Fix: split the compound column
df[['age', 'sex']] = df['age_sex'].str.split('_', expand=True)
df['age'] = df['age'].astype(int)
df = df.drop(columns='age_sex')
print(df)
```

```output
  region  count  age sex
0  north    120   25   M
1  north     95   30   F
2  south    200   25   M
```

### Pattern 3 — One variable across multiple columns

Systolic and diastolic blood pressure are a single "blood pressure" measurement stored in
two columns — they should become two rows (keyed by `measurement_type`).

```python
import pandas as pd

df = pd.DataFrame({
    'patient': [1, 2, 3],
    'systolic': [120, 130, 115],
    'diastolic': [80, 85, 75],
})

# Both pressure readings are the SAME variable (blood pressure) with different types
tidy = df.melt(id_vars='patient',
               value_vars=['systolic', 'diastolic'],
               var_name='measurement', value_name='mmhg')
print(tidy.sort_values(['patient', 'measurement']).reset_index(drop=True))
```

```output
   patient  measurement  mmhg
0        1    diastolic    80
1        1     systolic   120
2        2    diastolic    85
3        2     systolic   130
4        3    diastolic    75
5        3     systolic   115
```

### Pattern 4 — Observations split across rows

When a single "observation" is stored in two or more rows, you need to pivot from long
to wide to combine them.

```python
import pandas as pd

long = pd.DataFrame({
    'id':    [1, 1, 2, 2],
    'key':   ['name', 'age', 'name', 'age'],
    'value': ['alice', '25', 'bob', '32'],
})

wide = long.pivot(index='id', columns='key', values='value').reset_index()
wide.columns.name = None
wide['age'] = wide['age'].astype(int)
print(wide)
```

```output
   id  age   name
0   1   25  alice
1   2   32    bob
```

## Reshaping with `stack` and `unstack`

When you already have a MultiIndex, `stack`/`unstack` are faster than pivot/melt:

```python
import pandas as pd

df = pd.DataFrame({
    'jan': [10, 20], 'feb': [11, 21], 'mar': [12, 22]
}, index=['A', 'B'])
df.index.name = 'product'

# stack: columns → inner row level
long = df.stack()
long.index.names = ['product', 'month']
print(long)
```

```output
product  month
A        jan      10
         feb      11
         mar      12
B        jan      20
         feb      21
         mar      22
dtype: int64
```

## Try it Yourself

```python
import pandas as pd

# A real-world-style untidy table: one row per student, multiple test scores
untidy = pd.DataFrame({
    'student': ['alice', 'bob', 'carol'],
    'math_q1': [85, 70, 90],
    'math_q2': [88, 72, 87],
    'english_q1': [78, 85, 80],
    'english_q2': [80, 83, 82],
})

# Tidy it: separate subject and quarter
tidy = untidy.melt(id_vars='student', var_name='subject_quarter', value_name='score')
tidy[['subject', 'quarter']] = tidy['subject_quarter'].str.rsplit('_', n=1, expand=True)
tidy = tidy.drop(columns='subject_quarter').sort_values(['student','subject','quarter']).reset_index(drop=True)
print(tidy.head(8))

# Verify: per-student mean score per subject
print(tidy.groupby(['student', 'subject'])['score'].mean())
```

```output
  student subject quarter  score
0   alice english      q1     78
1   alice english      q2     80
2   alice    math      q1     85
3   alice    math      q2     88
4     bob english      q1     85
5     bob english      q2     83
6     bob    math      q1     70
7     bob    math      q2     72

student  subject
alice    english    79.0
         math       86.5
bob      english    84.0
         math       71.0
carol    english    81.0
         math       88.5
Name: score, dtype: float64
```

## Exercise

You have a DataFrame where each row records one transaction with columns
`user_id`, `item_bought`, `item_price`. The same user may appear multiple times.
Reshape it into a wide "user-item matrix" where rows are users, columns are items,
and values are the prices (or 0 if the user never bought that item).

<details>
<summary>Solution</summary>

```python
import pandas as pd

df = pd.DataFrame({
    'user_id':   [1, 1, 2, 3, 2, 1],
    'item':      ['A', 'B', 'A', 'C', 'B', 'C'],
    'price':     [10, 20, 10, 30, 20, 30],
})

# pivot_table handles duplicate (user, item) pairs (e.g. if a user buys the same item twice)
matrix = df.pivot_table(
    index='user_id', columns='item', values='price',
    aggfunc='sum', fill_value=0,
)
matrix.columns.name = None
print(matrix)
```

This is the "user-item matrix" used in collaborative filtering (ch. 15 — Recommender
Systems). Tidy format is the input; the wide pivot is the model's input format.
</details>
