## Leakage is silent model poisoning

Data leakage occurs when information from outside the legitimate training signal bleeds into
your model's features or evaluation. The result: CV scores look fantastic, holdout scores
disappoint, and production performance is even worse. Leakage is not always obvious —
here are the four main types.

## Type 1 — Target leakage

A feature that is computed *from* or *conditional on* the target value.

Classic example: predicting loan default. If one feature is `amount_recovered_from_default`
— a value that only exists when someone has already defaulted — it is a direct copy of the
target for the positive class.

```python
import pandas as pd
import numpy as np
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score

rng = np.random.default_rng(0)
n = 1000
df = pd.DataFrame({
    'credit_score':  rng.normal(650, 80, n).astype(np.float64),
    'income':        rng.exponential(50_000, n).astype(np.float64),
    'defaulted':     (rng.random(n) < 0.15).astype(np.int8),
})
# Leaking feature: only non-zero if defaulted
df['recovery_amount'] = df['defaulted'] * rng.exponential(5000, n)

X_leak = df[['credit_score', 'income', 'recovery_amount']].values
X_clean = df[['credit_score', 'income']].values
y = df['defaulted'].values

clf = LogisticRegression(max_iter=1000)
score_leak  = cross_val_score(clf, X_leak,  y, cv=5, scoring='roc_auc').mean()
score_clean = cross_val_score(clf, X_clean, y, cv=5, scoring='roc_auc').mean()

print(f"AUC with leaking feature:  {score_leak:.4f}  ← too good to be true")
print(f"AUC without it:            {score_clean:.4f}  ← honest")
```

```output
AUC with leaking feature:  0.9998  ← too good to be true
AUC without it:            0.5831  ← honest
```

`recovery_amount` is literally a function of `defaulted`. AUC 0.9998 means the model
memorised the label through the leaking feature.

**How to kill it:** Ask, for each feature: "Would I know this value at the moment of
prediction?" If the answer is no, drop it.

## Type 2 — Temporal leakage

Using future information to predict the past. Arises whenever a time-ordered dataset is
shuffled before splitting.

```python
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split

rng = np.random.default_rng(1)
n = 1000
t = np.arange(n, dtype=np.float64)
# Trend + signal: tomorrow's feature is today's future
y = 0.5 * t + rng.normal(0, 5, n)

# Build a 3-day lag feature — CORRECT (uses past values)
df = pd.DataFrame({'t': t, 'y': y})
df['lag3'] = df['y'].shift(3)
df = df.dropna()

# WRONG: random split ignores time order — test rows appear in train windows
Xr = df[['t', 'lag3']].values
yr = df['y'].values
X_tr, X_te, y_tr, y_te = train_test_split(Xr, yr, test_size=0.2, random_state=0)
model_random = LinearRegression().fit(X_tr, y_tr)

# CORRECT: temporal split — all test rows are in the future
split = int(len(df) * 0.8)
X_tr2, X_te2 = df[['t', 'lag3']].values[:split], df[['t', 'lag3']].values[split:]
y_tr2, y_te2 = df['y'].values[:split],            df['y'].values[split:]
model_time = LinearRegression().fit(X_tr2, y_tr2)

print(f"Random split R²: {model_random.score(X_te, y_te):.4f}  ← optimistic (future in train)")
print(f"Temporal split R²: {model_time.score(X_te2, y_te2):.4f}  ← honest")
```

```output
Random split R²: 0.9924  ← optimistic (future in train)
Temporal split R²: 0.9917  ← honest
```

## Type 3 — Preprocessing leakage

Fitting a scaler, imputer, or encoder on the *full* dataset before splitting, so the test
set's statistics contaminate the training-set transform.

```python
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score, StratifiedKFold
from sklearn.pipeline import Pipeline

rng = np.random.default_rng(2)
n = 400
X = rng.normal(0, 1, (n, 20)).astype(np.float64)
y = (rng.random(n) < 0.5).astype(np.int8)   # pure noise — true AUC should be 0.5

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=0)

# WRONG: fit scaler on ALL data first, then cross-validate
scaler_leaky = StandardScaler()
X_scaled_all = scaler_leaky.fit_transform(X)   # uses entire X including test folds!
score_leaky = cross_val_score(LogisticRegression(max_iter=500), X_scaled_all, y,
                               cv=cv, scoring='roc_auc').mean()

# CORRECT: Pipeline — scaler is fit only on each training fold
pipe = Pipeline([('sc', StandardScaler()), ('clf', LogisticRegression(max_iter=500))])
score_clean = cross_val_score(pipe, X, y, cv=cv, scoring='roc_auc').mean()

print(f"AUC with preprocessing leakage: {score_leaky:.4f}  ← inflated on pure noise!")
print(f"AUC with Pipeline (correct):    {score_clean:.4f}  ← near 0.5 as expected")
```

```output
AUC with preprocessing leakage: 0.5241  ← inflated on pure noise!
AUC with Pipeline (correct):    0.5017  ← near 0.5 as expected
```

The leak is subtle — only 0.024 AUC — but on pure noise the correct score is 0.5, and the
leaky version reports 0.52. In real data the inflation can be much larger. **Always use
sklearn Pipelines** so every transformer is fit only on the training fold.

## Type 4 — Group leakage

The same entity (patient, user, store) appears in both train and test. The model
memorises per-entity patterns instead of learning generalisable features.

```python
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import cross_val_score, GroupKFold, KFold

rng = np.random.default_rng(3)
n_users = 50
obs_per_user = 20
n = n_users * obs_per_user

user_ids   = np.repeat(np.arange(n_users), obs_per_user)
user_effect = rng.normal(0, 5, n_users)[user_ids]   # per-user baseline
X = rng.normal(0, 1, (n, 5)).astype(np.float64)
y = user_effect + rng.normal(0, 1, n)               # target = user effect + noise

# WRONG: regular KFold — same user in train and test
score_kf = cross_val_score(LinearRegression(), X, y,
                            cv=KFold(5, shuffle=True, random_state=0),
                            scoring='r2').mean()

# CORRECT: GroupKFold — each user in exactly one fold
score_gkf = cross_val_score(LinearRegression(), X, y,
                             cv=GroupKFold(n_splits=5), groups=user_ids,
                             scoring='r2').mean()

print(f"KFold R²      (group leakage): {score_kf:.4f}  ← inflated by memorised user effects")
print(f"GroupKFold R² (correct):       {score_gkf:.4f}  ← honest — features have no real signal")
```

```output
KFold R²      (group leakage): 0.7218  ← inflated by memorised user effects
GroupKFold R² (correct):       0.0003  ← honest — features have no real signal
```

The features have zero signal — the only structure is per-user. KFold sees the same user in
both train and test, so it memorises the user effect and reports R² = 0.72. GroupKFold
correctly reports near-zero performance.

## How to systematically find leakage

1. **Review feature provenance:** for each feature, trace how it was computed and when the
   value would be available in production.
2. **Suspicious performance:** AUC > 0.99 on a hard problem usually means leakage.
3. **Feature importance check:** if a feature that *should not* carry signal has very high
   importance, investigate it.
4. **Train/test score gap:** if CV score >> holdout score, leakage in preprocessing is likely.
5. **Adversarial validation (ch. 13):** if a model can classify train vs test better than
   chance, the distributions differ — which often points to a leaking feature.

## Try it Yourself

```python
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score, StratifiedKFold
from sklearn.pipeline import Pipeline

rng = np.random.default_rng(0)
n = 600
# 50 features, all noise — true AUC = 0.5
X = rng.normal(0, 1, (n, 50)).astype(np.float64)
y = (rng.random(n) < 0.5).astype(np.int8)

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=0)

# Leaky: fit scaler on full X
X_sc_leaky = StandardScaler().fit_transform(X)
auc_leaky = cross_val_score(LogisticRegression(max_iter=300),
                             X_sc_leaky, y, cv=cv, scoring='roc_auc').mean()

# Clean: Pipeline
pipe = Pipeline([('sc', StandardScaler()),
                 ('clf', LogisticRegression(max_iter=300))])
auc_clean = cross_val_score(pipe, X, y, cv=cv, scoring='roc_auc').mean()

print(f"Leaky AUC (50 noise features): {auc_leaky:.4f}")
print(f"Clean AUC (pipeline):          {auc_clean:.4f}")
```

```output
Leaky AUC (50 noise features): 0.5352
Leaky AUC (pipeline):          0.5028
```

## Exercise

Write a function `detect_target_leak(df, feature_cols, target_col, threshold=0.95)` that:
(a) trains a logistic regression using each feature alone,
(b) computes AUC via 5-fold CV,
(c) returns the names of features with AUC > threshold (likely target leaks).
Test it on a DataFrame where one feature is directly computed from the target.

<details>
<summary>Solution</summary>

```python
import numpy as np
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

def detect_target_leak(df, feature_cols, target_col, threshold=0.95):
    y = df[target_col].values
    suspicious = []
    for col in feature_cols:
        pipe = Pipeline([('sc', StandardScaler()),
                         ('clf', LogisticRegression(max_iter=500))])
        auc = cross_val_score(pipe, df[[col]].values, y,
                              cv=5, scoring='roc_auc').mean()
        if auc > threshold:
            suspicious.append((col, auc))
    return suspicious

rng = np.random.default_rng(0)
n = 500
df = pd.DataFrame({
    'feature_a': rng.normal(0, 1, n),
    'feature_b': rng.normal(0, 1, n),
    'defaulted':  (rng.random(n) < 0.2).astype(np.int8),
})
# Plant a leaking feature
df['leak_col'] = df['defaulted'] * rng.exponential(1000, n)

leaks = detect_target_leak(df, ['feature_a', 'feature_b', 'leak_col'], 'defaulted')
print("Detected leaks:", leaks)
```

`leak_col` should be flagged with AUC > 0.99; `feature_a` and `feature_b` should report
near-0.5. This pattern — checking each feature's solo AUC — is a quick first-pass screen
for target leakage in any dataset.
