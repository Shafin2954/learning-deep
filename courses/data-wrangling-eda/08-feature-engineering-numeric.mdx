## Why transform numeric features?

Most linear models assume a linear relationship between features and target — and that error
variance is constant. Right-skewed features, multiplicative relationships, and bounded counts
violate these assumptions. Numeric feature engineering turns awkward distributions into
model-friendly ones and creates new signals your model can't find from raw values alone.

## Binning

Binning discretises a continuous feature into categorical bins. Use it when you believe the
relationship with the target is step-like (e.g., age brackets, credit score tiers).

```python
import pandas as pd
import numpy as np

rng = np.random.default_rng(0)
n = 500
age = rng.integers(18, 80, n).astype(np.int32)
df = pd.DataFrame({'age': age})

# Uniform-width bins
df['age_bin_uniform'] = pd.cut(df['age'], bins=5,
                                labels=['18-30', '31-43', '44-56', '57-69', '70-80'])

# Quantile bins: each bin holds ~equal number of rows
df['age_bin_quantile'] = pd.qcut(df['age'], q=4,
                                  labels=['Q1', 'Q2', 'Q3', 'Q4'])

print(df[['age', 'age_bin_uniform', 'age_bin_quantile']].head(8))
print("\nUniform bin counts:")
print(df['age_bin_uniform'].value_counts().sort_index())
```


Uniform bins produce unequal counts; quantile bins produce equal counts. Quantile bins are
better for tree-based models because they create balanced splits.

## Log transform — taming right skew

For right-skewed positive data, $\log(x + 1)$ compresses large values and stretches small
ones, making the distribution more symmetric.

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

rng = np.random.default_rng(1)
n = 1000
income = rng.exponential(40_000, n).astype(np.float64)
log_income = np.log1p(income)  # log(1 + x) — safe when x ≥ 0

fig, axes = plt.subplots(1, 2, figsize=(10, 3))
axes[0].hist(income, bins=50, color='steelblue', edgecolor='white')
axes[0].set_title(f'income  skew={pd.Series(income).skew():.2f}')

axes[1].hist(log_income, bins=50, color='tomato', edgecolor='white')
axes[1].set_title(f'log(1+income)  skew={pd.Series(log_income).skew():.2f}')

fig.tight_layout(); plt.show()
print(f"Income  — mean: {income.mean():.0f}, std: {income.std():.0f}, skew: {pd.Series(income).skew():.2f}")
print(f"Log     — mean: {log_income.mean():.2f}, std: {log_income.std():.2f}, skew: {pd.Series(log_income).skew():.2f}")
```

```output
Income  — mean: 40189, std: 40442, skew: 2.02
Log     — mean: 10.30, std: 1.00, skew: -0.10
[Figure: before/after histogram showing right-skewed → near-normal distribution]
```

## Box-Cox and Yeo-Johnson power transforms

Box-Cox finds the optimal power $\lambda$ that best normalises the data via MLE:

$$
x^{(\lambda)} = \begin{cases} \frac{x^\lambda - 1}{\lambda} & \lambda \neq 0 \\ \log x & \lambda = 0 \end{cases}
$$

Requires $x > 0$. Yeo-Johnson extends it to handle zero and negative values.

```python
import numpy as np
import pandas as pd
from sklearn.preprocessing import PowerTransformer

rng = np.random.default_rng(2)
n = 500
# Right-skewed positive data
x_pos = rng.exponential(5, n).astype(np.float64).reshape(-1, 1)
# Data with zeros and negatives
x_mixed = (rng.exponential(3, n) - 1.5).astype(np.float64).reshape(-1, 1)

pt_bc = PowerTransformer(method='box-cox')
pt_yj = PowerTransformer(method='yeo-johnson')

x_pos_bc = pt_bc.fit_transform(x_pos).ravel()
x_mixed_yj = pt_yj.fit_transform(x_mixed).ravel()

print(f"Box-Cox λ:       {pt_bc.lambdas_[0]:.3f}")
print(f"Box-Cox output — skew: {pd.Series(x_pos_bc).skew():.3f}")
print(f"\nYeo-Johnson λ:   {pt_yj.lambdas_[0]:.3f}")
print(f"Yeo-Johnson output — skew: {pd.Series(x_mixed_yj).skew():.3f}")
print(f"Input had negatives: {(x_mixed < 0).sum()} values below 0")
```

```output
Box-Cox λ:       0.241
Box-Cox output — skew: 0.009

Yeo-Johnson λ:   0.532
Yeo-Johnson output — skew: -0.013
Input had negatives: 167 values below 0
```

Both transforms produce near-zero skew. `PowerTransformer` fits the optimal $\lambda$ on
training data and applies the same transform to test data — always use it inside a Pipeline.

## Polynomial features and interactions

Polynomial features capture non-linear structure and feature interactions:

$$
[x_1, x_2] \xrightarrow{\text{degree=2}} [1, x_1, x_2, x_1^2, x_1 x_2, x_2^2]
$$

```python
import numpy as np
from sklearn.preprocessing import PolynomialFeatures

rng = np.random.default_rng(3)
n = 6
X = rng.standard_normal((n, 2)).astype(np.float64)
X = X.round(2)

poly = PolynomialFeatures(degree=2, include_bias=False)
X_poly = poly.fit_transform(X)

print("Original X:")
print(X)
print(f"\nAfter PolynomialFeatures(degree=2): shape {X_poly.shape}")
print("Feature names:", poly.get_feature_names_out(['x1', 'x2']))
print(X_poly.round(3))
```

```output
Original X:
[[ 1.76  0.4 ]
 [ 0.98  2.24]
 [ 1.87 -0.98]
 [-0.98  0.95]
 [-0.15 -0.1 ]
 [ 0.41  0.14]]

After PolynomialFeatures(degree=2): shape (6, 5)
Feature names: ['x1' 'x2' 'x1^2' 'x1 x2' 'x2^2']
[[ 1.76   0.4    3.098  0.704  0.16 ]
 [ 0.98   2.24   0.96   2.195  5.018]
 [ 1.87  -0.98   3.497 -1.833  0.96 ]
 [-0.98   0.95   0.96  -0.931  0.903]
 [-0.15  -0.1    0.023  0.015  0.01 ]
 [ 0.41   0.14   0.168  0.057  0.02 ]]
```

With 10 input features at degree 2, you get 65 output features. At degree 3 with 10 features,
you get 285. Use polynomial features only when you have a specific hypothesis about
interactions, not as a blanket approach.

## Try it Yourself

```python
import numpy as np
import pandas as pd
from sklearn.preprocessing import PowerTransformer
import matplotlib.pyplot as plt

rng = np.random.default_rng(4)
n = 800
# Right-skewed transaction amounts
amount = rng.exponential(250, n).astype(np.float64)

# Three transforms
log_amt = np.log1p(amount)
pt = PowerTransformer(method='yeo-johnson')
yj_amt = pt.fit_transform(amount.reshape(-1, 1)).ravel()

fig, axes = plt.subplots(1, 3, figsize=(13, 3))
for ax, data, name in zip(axes,
                           [amount, log_amt, yj_amt],
                           ['Original', 'log(1+x)', 'Yeo-Johnson']):
    ax.hist(data, bins=40, edgecolor='white', color='steelblue')
    ax.set_title(f'{name}\nskew={pd.Series(data).skew():.2f}')
plt.tight_layout(); plt.show()
print(f"Original skew:     {pd.Series(amount).skew():.2f}")
print(f"Log skew:          {pd.Series(log_amt).skew():.2f}")
print(f"Yeo-Johnson skew:  {pd.Series(yj_amt).skew():.2f}")
```

```output
Original skew:     2.02
Log skew:          0.02
Yeo-Johnson skew:  -0.01
[Figure: three histograms showing progressive normalisation of skewed data]
```

## Exercise

Given a DataFrame with `price` (log-normal), `area` (normal), and `rooms` (integer 1-6):
(a) Apply log(1+price) and verify skew drops below 0.5.
(b) Create an interaction term `area_per_room = area / rooms`.
(c) Create quantile bins for `price` with 5 bins, and print the count in each.

<details>
<summary>Solution</summary>

```python
import numpy as np
import pandas as pd

rng = np.random.default_rng(0)
n = 400
df = pd.DataFrame({
    'price': np.exp(rng.normal(12, 1, n)).astype(np.float64),   # log-normal
    'area':  rng.normal(80, 20, n).astype(np.float64),
    'rooms': rng.integers(1, 7, n).astype(np.int32),
})

# (a) Log transform
df['log_price'] = np.log1p(df['price'])
print(f"price skew:     {df['price'].skew():.2f}")
print(f"log_price skew: {df['log_price'].skew():.2f}")

# (b) Interaction
df['area_per_room'] = df['area'] / df['rooms']
print(f"\narea_per_room stats:\n{df['area_per_room'].describe().round(1)}")

# (c) Quantile bins
df['price_q5'] = pd.qcut(df['price'], q=5, labels=['Q1','Q2','Q3','Q4','Q5'])
print(f"\nQuantile bin counts:\n{df['price_q5'].value_counts().sort_index()}")
```

Log transform brings skew below 0.5. `area_per_room` is a domain-motivated interaction —
a 20-room flat of 80m² is very different from a 2-room flat of 80m². Quantile bins produce
~80 rows per bin regardless of the distribution shape.
</details>
