## The project

Build a complete forecasting system for a real-world univariate time series (monthly retail sales, energy consumption, or agricultural output). Your system must include: a from-scratch exponential smoothing family (Simple, Holt, Holt-Winters), an AR model implemented via OLS on lag features, a rolling-origin backtester, a gradient-boosting model with engineered features, and calibrated prediction intervals. You compare all approaches in a model-selection writeup and justify your final recommendation.

This is the integration project for the whole course — every chapter contributes a component.

## Hard constraints (no vibe coding)

- **No statsmodels for core ETS/AR.** You may use it only for diagnostics (ACF plots, ADF test). Holt-Winters and AR must be your own NumPy/Python code.
- **No sktime / prophet / neuralforecast** for the core models. (You may use them in an optional section to benchmark your own implementation, but they cannot be your primary deliverable.)
- **No AI assistance** on the core implementation. The point is that you can do it yourself.
- Gradient boosting is allowed via sklearn/lightgbm, but the lag/calendar feature engineering must be your own code.
- Every model must go through the same rolling-origin backtester — no fitting on the full series and evaluating on the same data.

## What it must contain (and the chapter each tests)

- **Data loading and stationarity audit** (ch. 1–2): load your chosen series, plot decomposition, run ADF test, apply differencing if needed, document the decision.
- **Exponential smoothing from scratch** (ch. 4): implement Simple ES (one parameter $\alpha$), Holt (trend; $\alpha, \beta$), and Holt-Winters (trend + multiplicative seasonality; $\alpha, \beta, \gamma$) in pure Python/NumPy. Optimize parameters by minimizing MSE on a training window.
- **AR(p) from scratch via OLS** (ch. 5): build lag-matrix, solve for AR coefficients via the normal equations or `np.linalg.lstsq`, select $p$ by AIC computed from the residuals.
- **Rolling-origin backtester** (ch. 6): implement a function that slides the training window forward one step at a time, retrains/recalibrates each model, and collects one-step-ahead forecasts over a held-out window. Zero leakage — no future values ever touch training.
- **ML forecasting with lag + calendar features** (ch. 7–8): build a feature matrix (lags 1–12, rolling mean/std windows, month-of-year, quarter), train LightGBM in the backtester, tune with early stopping on a validation tail.
- **Prediction intervals** (ch. 11): produce 80% intervals for each model using either the residual-conformal method or quantile regression. Report empirical coverage and mean width.
- **Model-selection writeup**: table of MASE, SMAPE, RMSE, pinball loss, and coverage for all models; narrative justification of the winner with failure analysis of the losers.

## Acceptance criteria

- Holt-Winters implementation produces smoothed decomposition and multi-step forecasts that visually match the data's trend and seasonal pattern.
- Rolling-origin backtester produces identical results on two separate runs (deterministic) and raises an error if future data is accessed.
- MASE for your best model is below 1.0 (beats the naïve seasonal baseline).
- Prediction intervals achieve empirical coverage within ±5 percentage points of the nominal 80%.
- All three model families (ETS, AR, gradient boosting) are compared in the writeup with quantitative metrics from the backtester.
- Code is reproducible from a single entry-point script with a fixed random seed.
