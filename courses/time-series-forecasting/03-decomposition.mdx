## Decomposing a series into components

Decomposition separates the series into trend, seasonal, and residual so you can model each
independently or diagnose which component dominates forecast error.

Two main approaches:
- **Classical** (moving-average based): simple, transparent, but assumes stable seasonality
  and loses a half-period at each endpoint.
- **STL** (Seasonal-Trend decomposition via Loess): iterative, handles changing seasonality,
  robust to outliers.

## Classical decomposition

For an additive series with period $m$:

1. **Trend** $\hat{T}_t$: apply a centered $2\times m$ moving average
   (so the sum of weights is 1 and the seasonal effects cancel).
2. **Seasonal** $\hat{S}_t$: average $(y_t - \hat{T}_t)$ for each period (Jan, Feb, …)
   across all years; normalize so seasonal effects sum to zero.
3. **Residual** $\hat{R}_t = y_t - \hat{T}_t - \hat{S}_t$.

```python
import numpy as np
import pandas as pd
from statsmodels.tsa.seasonal import seasonal_decompose

rng = np.random.default_rng(42)
idx = pd.date_range('2018-01', periods=72, freq='ME')
t   = np.arange(72)

# Retail sales: upward trend + monthly seasonality (December spike)
trend    = 200 + 2.5 * t
seasonal = 30 * np.sin(2 * np.pi * (t - 3) / 12)  # peak around month 9
noise    = rng.normal(0, 8, 72)
y        = trend + seasonal + noise

series = pd.Series(y, index=idx, name='sales')

result = seasonal_decompose(series, model='additive', period=12)

print("Trend first 5 values:")
print(result.trend.dropna().head().round(1).to_string())
print("\nSeasonal (first cycle):")
print(result.seasonal[:12].round(1).to_string())
print("\nResidual std:", round(result.resid.dropna().std(), 2))
```

## STL decomposition

STL (Cleveland et al., 1990) iterates between smoothing the trend (Loess) and extracting
the seasonal component. Key advantages:
- Handles **changing seasonality** (e.g., seasonal amplitude growing over time).
- Robust option (`robust=True`) down-weights outliers so they don't distort trend/seasonal.
- Requires only one parameter: `period`.

```python
import numpy as np
import pandas as pd
from statsmodels.tsa.seasonal import STL

rng = np.random.default_rng(0)
idx = pd.date_range('2015-01', periods=96, freq='ME')
t   = np.arange(96)

# Changing seasonal amplitude: grows from 10 to 40 over 8 years
amp      = 10 + 30 * (t / 95)
seasonal = amp * np.sin(2 * np.pi * t / 12)
trend    = 100 + 1.5 * t
y        = trend + seasonal + rng.normal(0, 5, 96)

series = pd.Series(y, index=idx)

stl    = STL(series, period=12, seasonal=13)
result = stl.fit()

print("STL seasonal — first vs last amplitude:")
first_amp = result.seasonal[:12].max() - result.seasonal[:12].min()
last_amp  = result.seasonal[84:].max() - result.seasonal[84:].min()
print(f"  First year amplitude: {first_amp:.1f}")
print(f"  Last  year amplitude: {last_amp:.1f}")
print(f"Residual std: {result.resid.std():.2f}")
```

```output
STL seasonal — first vs last amplitude:
  First year amplitude: 24.1
  Last  year amplitude: 75.8
Residual std: 5.31
```

STL correctly captures the growing amplitude — the extracted seasonal component spans only
~24 in year 1 but ~76 in year 8, matching the simulated growth. Classical decomposition
would average these and miss the trend in seasonality.

## Try it Yourself

```python
import numpy as np
import pandas as pd
from statsmodels.tsa.seasonal import STL

rng = np.random.default_rng(3)
idx = pd.date_range('2020-01', periods=60, freq='ME')
t   = np.arange(60)

y = (150 + t * 1.2
     + 20 * np.sin(2 * np.pi * t / 12)
     + rng.normal(0, 6, 60))

series = pd.Series(y, index=idx)
result = STL(series, period=12).fit()

print(f"Trend range:    {result.trend.min():.1f} – {result.trend.max():.1f}")
print(f"Seasonal range: {result.seasonal.min():.1f} – {result.seasonal.max():.1f}")
print(f"Residual std:   {result.resid.std():.2f}")
print(f"Residual skew:  {result.resid.skew():.3f}")
```

```output
Trend range:    150.4 – 218.7
Seasonal range: -22.8 – 21.4
Residual std:   6.08
Residual skew:  0.039
```

## Exercise

Decompose a quarterly series (period=4) using `seasonal_decompose`. Then apply the
Ljung-Box test to the residuals to check whether they are white noise (no remaining
autocorrelation).

<details>
<summary>Solution</summary>

```python
import numpy as np
import pandas as pd
from statsmodels.tsa.seasonal import seasonal_decompose
from statsmodels.stats.diagnostic import acorr_ljungbox

rng = np.random.default_rng(7)
idx = pd.date_range('2010-Q1', periods=40, freq='QE')
t   = np.arange(40)

y = (50 + 0.5*t
     + 10 * np.sin(2*np.pi*t/4)
     + rng.normal(0, 2, 40))

series = pd.Series(y, index=idx)
result = seasonal_decompose(series, model='additive', period=4)

lb = acorr_ljungbox(result.resid.dropna(), lags=[4, 8], return_df=True)
print(lb[['lb_stat', 'lb_pvalue']].round(4))
# p-values > 0.05 → residuals are white noise
```
