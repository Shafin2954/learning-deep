## Why convexity is a superpower

A convex optimisation problem has one stunning property: **every local minimum is a global minimum**. When your loss function is convex, gradient descent is guaranteed to find the best solution, not just a good one. Understanding which losses are convex — and which aren't — tells you when you can trust your optimizer.

## Convex sets

A set $C$ is **convex** if the line segment between any two points in $C$ stays in $C$:

$$
\forall\, x, y \in C,\; \theta \in [0,1]: \quad \theta x + (1-\theta)y \in C
$$

Examples: balls $\{x : \|x\| \le r\}$, halfspaces $\{x : a^T x \le b\}$, polyhedra $\{x : Ax \le b\}$.

The intersection of convex sets is convex — this is why Ridge regression (L2 ball $\cap$ residual-fitting ellipsoid) has a unique solution.

## Convex functions

A function $f : \mathbb{R}^d \to \mathbb{R}$ is **convex** if its graph lies below the chord between any two points:

$$
f(\theta x + (1-\theta)y) \le \theta f(x) + (1-\theta)f(y) \quad \forall\, x, y,\; \theta \in [0,1]
$$

### Second-order condition

For twice-differentiable $f$, convexity is equivalent to:

$$
\nabla^2 f(x) \succeq 0 \quad \forall\, x
$$

(the Hessian is positive semi-definite everywhere).

### First-order condition (the key inequality)

If $f$ is convex and differentiable:

$$
f(y) \ge f(x) + \nabla f(x)^T (y - x) \quad \forall\, x, y
$$

The tangent plane at $x$ is a **global lower bound** on $f$. Setting $\nabla f(x) = 0$ makes the tangent plane horizontal, proving the point is a global minimum.

## MSE loss is convex

For linear regression with loss $f(w) = \|Xw - y\|^2 / n$, the Hessian is $2X^TX/n$, which is PSD. So MSE is always convex, and GD finds the global optimum.

```python
import numpy as np

rng = np.random.default_rng(0)
n, d = 100, 2
X = rng.standard_normal((n, d))
y = X @ np.array([2.0, -1.0]) + rng.standard_normal(n) * 0.5

def mse(w):
    return np.mean((X @ w - y)**2)

def mse_grad(w):
    return 2 * X.T @ (X @ w - y) / n

# Grid over w1, w2 space to verify single minimum
w1_grid = np.linspace(0, 4, 50)
w2_grid = np.linspace(-3, 1, 50)
W1, W2 = np.meshgrid(w1_grid, w2_grid)
Z = np.array([[mse(np.array([w1, w2])) for w1 in w1_grid] for w2 in w2_grid])

min_idx = np.unravel_index(Z.argmin(), Z.shape)
print(f"Grid minimum at w = ({w1_grid[min_idx[1]]:.2f}, {w2_grid[min_idx[0]]:.2f})")

# Normal equations give exact answer
w_star = np.linalg.solve(X.T @ X, X.T @ y)
print(f"Normal equation w* = ({w_star[0]:.4f}, {w_star[1]:.4f})")
print(f"Gradient at w*: {np.round(mse_grad(w_star), 8)}")
```

```output
Grid minimum at w = (2.00, -0.98)
Normal equation w* = (2.0021, -0.9891)
Gradient at w*: [-0. -0.]
```

## Cross-entropy is convex (in the linear model weights)

For logistic regression, $f(w) = -\frac{1}{n}\sum_i [y_i \log \sigma(w^T x_i) + (1-y_i)\log(1-\sigma(w^T x_i))]$. The Hessian is $X^T S X / n$ where $S = \text{diag}(\sigma_i(1-\sigma_i))$ — PSD. So logistic regression has a unique global minimum too.

## Neural network loss is NOT convex

The moment you compose two linear layers through a nonlinearity (ReLU, sigmoid), the loss becomes non-convex. Multiple local minima exist. Gradient descent finds a good — but not provably optimal — solution.

```python
import numpy as np

rng = np.random.default_rng(1)

# Simple 1D non-convex: two-layer net output f(w) = relu(w)*w (schematic)
def f_nonconvex(w):
    return np.sin(3*w) + 0.5*w**2 - 2*np.cos(w)

w_vals = np.linspace(-4, 4, 1000)
f_vals = f_nonconvex(w_vals)

local_min_idx = np.where(
    (f_vals[1:-1] < f_vals[:-2]) & (f_vals[1:-1] < f_vals[2:])
)[0] + 1

print("Approximate local minima at w =", np.round(w_vals[local_min_idx], 3))
print("Function values             =", np.round(f_vals[local_min_idx], 3))
print("Global min at w =", round(w_vals[f_vals.argmin()], 3))
```

```output
Approximate local minima at w = [-3.437 -0.787  2.011]
Function values             = [-1.694 -0.893 -1.259]
Global min at w = -3.437
```

Three local minima; gradient descent finds different ones depending on initialisation.

## Try it Yourself

```python
import numpy as np

# Verify convexity of log-sum-exp by checking the chord condition
def log_sum_exp(x):
    c = x.max()
    return c + np.log(np.sum(np.exp(x - c)))

rng = np.random.default_rng(0)
n_checks = 1000
violations = 0
for _ in range(n_checks):
    x = rng.standard_normal(5)
    y = rng.standard_normal(5)
    theta = rng.uniform(0, 1)
    mid = theta * x + (1 - theta) * y
    lhs = log_sum_exp(mid)
    rhs = theta * log_sum_exp(x) + (1 - theta) * log_sum_exp(y)
    if lhs > rhs + 1e-9:
        violations += 1

print(f"Chord violations in {n_checks} random checks: {violations}")
print("log-sum-exp is convex:", violations == 0)
```

```output
Chord violations in 1000 random checks: 0
log-sum-exp is convex: True
```

## Exercise

Classify each function as convex, concave, or neither, and verify numerically using the chord condition on random inputs.

Functions: $f(x) = x^2$, $f(x) = \log x$ (for $x > 0$), $f(x) = \max(0, x)$ (ReLU), $f(x) = \sin x$.

<summary>Solution</summary>

```python
import numpy as np

rng = np.random.default_rng(0)

def check_convexity(f, lo, hi, n=2000):
    xs = rng.uniform(lo, hi, n)
    ys = rng.uniform(lo, hi, n)
    thetas = rng.uniform(0, 1, n)
    mids = thetas * xs + (1 - thetas) * ys
    lhs = f(mids)
    rhs = thetas * f(xs) + (1 - thetas) * f(ys)
    violations = (lhs > rhs + 1e-9).sum()
    return violations

fns = {
    "x^2":   (lambda x: x**2,      -5,   5),
    "log(x)": (lambda x: np.log(x), 0.1, 10),
    "relu(x)": (lambda x: np.maximum(0, x), -5, 5),
    "sin(x)": (lambda x: np.sin(x), -5,  5),
}

for name, (fn, lo, hi) in fns.items():
    v = check_convexity(fn, lo, hi)
    tag = "convex" if v == 0 else f"NOT convex ({v} violations)"
    print(f"f(x) = {name:10s}: {tag}")
```

`x^2` and `relu` are convex (0 violations). `log(x)` is concave (many violations of the convexity condition). `sin(x)` is neither (violations in both directions).
</details>
