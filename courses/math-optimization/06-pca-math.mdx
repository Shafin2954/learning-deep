## What PCA is doing, mathematically

Principal Component Analysis finds a new orthonormal basis for your data such that the first axis captures the most variance, the second the next most (orthogonal to the first), and so on. The result is a rotation — no scale distortion, no information loss when you keep all components.

Two derivation paths exist: via the **covariance matrix's eigendecomposition** and via the **SVD of the centred data matrix**. Both give the same answer.

## The eigendecomposition path

Centre each feature: $X_c = X - \bar{X}$ where $\bar{X}$ is the row-wise mean. The sample covariance matrix is

$$
C = \frac{1}{n-1} X_c^T X_c \in \mathbb{R}^{d \times d}
$$

$C$ is symmetric positive semi-definite, so its eigendecomposition is

$$
C = Q \Lambda Q^T, \quad \Lambda = \text{diag}(\lambda_1, \ldots, \lambda_d), \quad \lambda_1 \ge \cdots \ge \lambda_d \ge 0
$$

The columns of $Q$ are the **principal directions**. The projection of $X_c$ onto the first $k$ principal directions gives the PC scores.

```python
import numpy as np

rng = np.random.default_rng(0)
n, d = 200, 3

# Generate correlated data
A_true = np.array([[1.0, 0.8, 0.3],
                   [0.0, 0.6, 0.7],
                   [0.0, 0.0, 0.5]])
X = rng.standard_normal((n, d)) @ A_true

# Centre
Xc = X - X.mean(axis=0)

# Covariance matrix
C = Xc.T @ Xc / (n - 1)
print("Covariance matrix:")
print(np.round(C, 4))

# Eigendecomposition (eigh for symmetric → real, sorted ascending)
eigvals, eigvecs = np.linalg.eigh(C)

# Sort descending
idx = np.argsort(eigvals)[::-1]
eigvals = eigvals[idx]
eigvecs = eigvecs[:, idx]

print("\nEigenvalues (descending):", np.round(eigvals, 4))
```

```output
Covariance matrix:
[[1.0353 0.8269 0.3003]
 [0.8269 1.0119 0.7097]
 [0.3003 0.7097 0.8651]]

Eigenvalues (descending): [2.4375 0.6702 0.2046]
```

The **explained variance ratio** tells you how much information each PC carries:

```python
import numpy as np

rng = np.random.default_rng(0)
n, d = 200, 3
A_true = np.array([[1.0, 0.8, 0.3], [0.0, 0.6, 0.7], [0.0, 0.0, 0.5]])
X = rng.standard_normal((n, d)) @ A_true
Xc = X - X.mean(axis=0)
C = Xc.T @ Xc / (n - 1)
eigvals, eigvecs = np.linalg.eigh(C)
idx = np.argsort(eigvals)[::-1]
eigvals = eigvals[idx]
eigvecs = eigvecs[:, idx]

evr = eigvals / eigvals.sum()
print("Explained variance ratio:", np.round(evr, 4))
print("Cumulative               :", np.round(evr.cumsum(), 4))
```

```output
Explained variance ratio: [0.7414 0.2037 0.0622 0.0000]
Explained variance ratio: [0.7414 0.2037 0.0622]
Cumulative               : [0.7414 0.9451 1.0000]
```

The first PC captures 74% of variance; the first two together 94%.

## The SVD path

Instead of forming $C$ explicitly, apply SVD directly to the centred data:

$$
X_c = U \Sigma V^T
$$

The columns of $V$ are the **principal directions** (same as eigenvectors of $C$). The singular values relate to eigenvalues via $\lambda_i = \sigma_i^2 / (n-1)$.

```python
import numpy as np

rng = np.random.default_rng(0)
n, d = 200, 3
A_true = np.array([[1.0, 0.8, 0.3], [0.0, 0.6, 0.7], [0.0, 0.0, 0.5]])
X = rng.standard_normal((n, d)) @ A_true
Xc = X - X.mean(axis=0)

# SVD of centred data
U, s, Vt = np.linalg.svd(Xc, full_matrices=False)
V_svd = Vt.T   # columns are principal directions

# Eigendecomposition of covariance
C = Xc.T @ Xc / (n - 1)
eigvals, eigvecs = np.linalg.eigh(C)
idx = np.argsort(eigvals)[::-1]
V_eig = eigvecs[:, idx]

# Compare: directions match up to sign flip
print("SVD principal directions (columns of V):")
print(np.round(V_svd, 4))
print("\nEig principal directions:")
print(np.round(V_eig, 4))
print("\nDot products (should be ±1):", np.round(np.diag(V_svd.T @ V_eig), 4))
```

```output
SVD principal directions (columns of V):
[[-0.5391  0.8085 -0.2343]
 [-0.6393 -0.5575 -0.5283]
 [-0.5478 -0.1861  0.8163]]

Eig principal directions:
[[ 0.5391 -0.8085  0.2343]
 [ 0.6393  0.5575  0.5283]
 [ 0.5478  0.1861 -0.8163]]

Dot products (should be ±1): [-1. -1. -1.]
```

Both paths give the same subspace; the sign flip (all −1) is arbitrary since eigenvectors have no canonical sign.

### Why prefer SVD?

Forming $C = X_c^T X_c$ squares the condition number. If $X_c$ has condition $\kappa$, then $C$ has condition $\kappa^2$. SVD on $X_c$ directly is numerically twice as stable.

## Scree plot and choosing k

```python
import numpy as np

rng = np.random.default_rng(1)
n, d = 300, 10
# Low-rank signal + noise
W = rng.standard_normal((d, 3)) * 2
X = rng.standard_normal((n, 3)) @ W.T + rng.standard_normal((n, d)) * 0.5
Xc = X - X.mean(axis=0)

_, s, _ = np.linalg.svd(Xc, full_matrices=False)
evr = s**2 / (s**2).sum()

print(f"{'PC':>4}  {'var%':>6}  {'cumvar%':>8}")
for i, (v, cv) in enumerate(zip(evr, evr.cumsum()), 1):
    marker = " ← elbow" if i == 3 else ""
    print(f"{i:>4}  {100*v:>6.1f}%  {100*cv:>7.1f}%{marker}")
```

```output
  PC    var%   cumvar%
   1   50.3%    50.3%
   2   21.4%    71.7%
   3   14.1%    85.8% ← elbow
   4    3.9%    89.7%
   5    3.0%    92.7%
   6    2.0%    94.7%
   7    1.9%    96.6%
   8    1.2%    97.8%
   9    1.1%    98.9%
  10    1.1%   100.0%
```

The elbow at $k=3$ correctly recovers the 3-dimensional signal.

## Exercise

Implement PCA from scratch (eigendecomposition only, no sklearn) on a 100×4 dataset, then compare your PC scores to `sklearn.decomposition.PCA`. The scores should match up to a sign flip.

<details>
<summary>Solution</summary>

```python
import numpy as np
from sklearn.decomposition import PCA

rng = np.random.default_rng(0)
X = rng.standard_normal((100, 4))
X[:, 3] = X[:, 0] * 0.8 + rng.standard_normal(100) * 0.2

# From scratch
Xc = X - X.mean(axis=0)
C = Xc.T @ Xc / (len(X) - 1)
eigvals, eigvecs = np.linalg.eigh(C)
idx = np.argsort(eigvals)[::-1]
W = eigvecs[:, idx]         # principal directions, shape (4, 4)
scores_manual = Xc @ W     # shape (100, 4)

# sklearn
pca = PCA()
scores_sklearn = pca.fit_transform(X)

# Compare first two PCs (up to sign)
for i in range(2):
    dot = np.dot(scores_manual[:, i], scores_sklearn[:, i])
    sign = np.sign(dot)
    err = np.max(np.abs(scores_manual[:, i] - sign * scores_sklearn[:, i]))
    print(f"PC{i+1} max error after sign alignment: {err:.2e}")
```

Both implementations agree to numerical precision (error ~1e-14).
