## Vectors as points and as directions

A vector $x \in \mathbb{R}^n$ is just an ordered tuple of real numbers. In ML it plays
two roles simultaneously: a **point** in $n$-dimensional space (one data sample, one token
embedding) and a **direction** with a magnitude (a gradient, a weight update). Which role
you emphasize changes how you reason about it.

```python
import numpy as np

x = np.array([3.0, 4.0], dtype=np.float64)
y = np.array([1.0, 2.0], dtype=np.float64)

print("x       :", x)
print("x + y   :", x + y)       # vector addition — move in direction y from x
print("2 * x   :", 2 * x)       # scalar multiplication — stretch
print("x - y   :", x - y)       # displacement from y to x
```

```output
x       : [3. 4.]
x + y   : [4. 6.]
2 * x   : [6. 8.]
x - y   : [2. 2.]
```

## Norms — measuring size

A **norm** $\|x\|$ is a function that assigns a non-negative scalar to every vector. Three
norms appear constantly in ML:

$$
\|x\|_1 = \sum_i |x_i|, \qquad
\|x\|_2 = \sqrt{\sum_i x_i^2}, \qquad
\|x\|_\infty = \max_i |x_i|
$$

```python
import numpy as np

x = np.array([3.0, -4.0, 0.0, 1.0], dtype=np.float64)

l1  = np.linalg.norm(x, ord=1)
l2  = np.linalg.norm(x, ord=2)
linf = np.linalg.norm(x, ord=np.inf)

print(f"L1   : {l1}")     # 3+4+0+1
print(f"L2   : {l2}")     # sqrt(9+16+0+1)
print(f"L∞   : {linf}")   # max(3,4,0,1)
```

```output
L1   : 8.0
L2   : 5.0990195135927845
L∞   : 4.0
```

### Which norm for what

- **L2** (Euclidean): the default. Minimizing it means "small weights overall"; it penalizes
  large individual weights but not sparsity. Used in Ridge regularization.
- **L1**: sum of absolute values. Minimizing it encourages exact zeros — sparse solutions.
  Used in Lasso regularization.
- **L∞**: worst-case magnitude. Used in adversarial robustness (bound the maximum perturbation
  to any single input dimension).

## The dot product and angle

The dot product of $x, y \in \mathbb{R}^n$ is:

$$
x \cdot y = x^\top y = \sum_i x_i y_i = \|x\|_2 \|y\|_2 \cos\theta
$$

where $\theta$ is the angle between the vectors. Rearranging:

$$
\cos\theta = \frac{x^\top y}{\|x\|_2 \|y\|_2}
$$

This is **cosine similarity**. It measures the angle, not the magnitude. Two vectors can
point in exactly the same direction but have very different L2 norms — cosine similarity
sees them as identical (similarity = 1). That's exactly what you want for embeddings:
"dog" and "dogs" should be similar regardless of how the model scaled their representations.

```python
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Same direction, different magnitudes
a = np.array([1.0, 2.0, 3.0])
b = 10 * a      # 10× longer, same direction

print("dot(a,b)        :", np.dot(a, b))
print("cos_sim(a, b)   :", cosine_similarity(a, b))   # should be 1.0
print("L2 dist(a, b)   :", np.linalg.norm(a - b))     # 10 × ||a||
```

```output
dot(a,b)        : 140.0
cos_sim(a, b)   : 1.0
L2 dist(a, b)   : 37.416573867739416
```

### Cosine vs Euclidean distance

```python
import numpy as np

# Sentence embeddings (unit vectors after normalization)
king   = np.array([0.8, 0.6, 0.0])
queen  = np.array([0.7, 0.7, 0.14])
banana = np.array([0.0, 0.1, 0.995])

def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print("cos(king, queen) :", round(cosine_sim(king, queen), 4))
print("cos(king, banana):", round(cosine_sim(king, banana), 4))
print("L2(king, queen)  :", round(np.linalg.norm(king - queen), 4))
print("L2(king, banana) :", round(np.linalg.norm(king - banana), 4))
```

```output
cos(king, queen) : 0.994
cos(king, banana): 0.0597
L2(king, queen)  : 0.1732
L2(king, banana) : 1.1284
```

Both metrics agree that king ≈ queen and king ≠ banana, but for high-dimensional embeddings
(dim 768 or 1536) cosine similarity is the standard because all embeddings tend to have
similar magnitudes after normalization, making the angle the informative part.

## Distances

The L2 norm of the difference vector gives the Euclidean (L2) distance:

$$
d_2(x, y) = \|x - y\|_2
$$

Other useful distances:
- **Manhattan (L1 distance):** $\sum_i |x_i - y_i|$ — robust to outliers.
- **Cosine distance:** $1 - \cos\theta$ — used in retrieval and clustering of embeddings.

```python
import numpy as np

x = np.array([1.0, 0.0, 1.0])
y = np.array([0.0, 1.0, 0.0])

print("L2 distance      :", round(np.linalg.norm(x - y), 4))
print("L1 distance      :", round(np.linalg.norm(x - y, ord=1), 4))
print("cosine distance  :", round(1 - np.dot(x, y) / (np.linalg.norm(x) * np.linalg.norm(y)), 4))
```

```output
L2 distance      : 1.7321
L1 distance      : 3.0
cosine distance  : 1.0
```

Here $x$ and $y$ are orthogonal (dot product = 0), so cosine distance = 1 — maximum dissimilarity.

## Try it Yourself

```python
import numpy as np

# 1) Normalize a batch of vectors to unit L2 norm (common pre-processing before cosine search)
rng = np.random.default_rng(0)
E = rng.standard_normal((5, 4)).astype(np.float64)   # 5 embeddings, dim 4
norms = np.linalg.norm(E, axis=1, keepdims=True)      # shape (5, 1)
E_unit = E / norms                                     # broadcasting
print("norms after normalization:", np.linalg.norm(E_unit, axis=1).round(8))

# 2) Pairwise cosine similarity matrix — no loop
# For unit vectors: cos(i,j) = E_unit[i] @ E_unit[j] = E_unit @ E_unit.T
C = E_unit @ E_unit.T
print("cosine sim matrix (diagonal = 1):\n", C.round(3))
```

```output
norms after normalization: [1. 1. 1. 1. 1.]
cosine sim matrix (diagonal = 1):
 [[ 1.    -0.283 -0.437  0.031  0.269]
  [-0.283  1.     0.234  0.288 -0.641]
  [-0.437  0.234  1.    -0.239  0.039]
  [ 0.031  0.288 -0.239  1.    -0.444]
  [ 0.269 -0.641  0.039 -0.444  1.   ]]
```

## Exercise

Given a query embedding $q \in \mathbb{R}^{128}$ and a database of 10 000 unit-normalized
embeddings $E \in \mathbb{R}^{10000 \times 128}$, write a function `top_k(q, E, k)` that
returns the indices of the $k$ most similar embeddings using cosine similarity — in a single
matrix multiply. Verify that for $k=3$ the top result has cosine similarity $\geq 0.9$ when
$q$ is one of the rows in $E$.

<details>
<summary>Solution</summary>

```python
import numpy as np

def top_k(q, E, k):
    """Return indices of k most similar rows in E (unit-normalized) to query q."""
    q_unit = q / np.linalg.norm(q)
    scores = E @ q_unit          # (N,) — cosine similarity (E already unit-normalized)
    return np.argpartition(scores, -k)[-k:]   # faster than full argsort for large N

rng = np.random.default_rng(42)
E_raw = rng.standard_normal((10_000, 128)).astype(np.float64)
E = E_raw / np.linalg.norm(E_raw, axis=1, keepdims=True)   # unit normalize

q = E[777]   # pick row 777 as the query
idx = top_k(q, E, k=3)
scores = E[idx] @ q
print("top-3 indices:", sorted(idx))
print("top-3 scores :", scores.round(6))
# Row 777 should be in idx with score ≈ 1.0
```

`np.argpartition` is O(N) vs O(N log N) for argsort — use it when you only need the top-k
and don't need them sorted among themselves.
</details>
