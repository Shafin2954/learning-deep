## The project

Build a mini optimisation library in pure NumPy with five solvers, then use it to solve three distinct machine-learning problems and verify every gradient with a numerical finite-difference check. The goal is to prove you can derive, implement, and validate the mathematical machinery behind modern ML — not call `scipy.optimize`. When you finish, you will have built the computational core of a neural network trainer from first principles.

## Hard constraints (no vibe coding)

- **No `scipy.optimize`, no sklearn, no JAX, no PyTorch, no autograd** for the core solver or gradient computation. Pure NumPy throughout.
- Every gradient function must pass a numerical finite-difference check before it is used in any solver. A gradient that fails the check is not a gradient — go back and derive it.
- Your solvers must accept the interface `optimizer(loss_fn, grad_fn, x0, **kwargs) → (x_final, loss_history)`. They are interchangeable on the same problem.
- All design decisions (learning rate, number of iterations, convergence threshold) must be justified by a measurement or a theoretical argument.

## What it must contain (and the chapter each tests)

1. **Five optimisers as callables** (ch. 9–10): gradient descent with fixed LR, GD with backtracking line search (Armijo condition), SGD with heavy-ball momentum, RMSProp, and Adam with bias correction. Each shares the same signature; all five must converge on a convex quadratic in under 500 steps.

2. **Backtracking line search** (ch. 9): a standalone function that takes `loss_fn`, `grad`, `w`, `alpha=0.3`, `beta=0.5` and returns the accepted step size. Plug it into your GD optimiser.

3. **Newton's method** (ch. 3, 12): takes a `hess_fn` in addition to `grad_fn`; solves the Newton system with `np.linalg.solve` (never `np.linalg.inv`); add Tikhonov regularisation $\lambda I$ when the Hessian is nearly singular.

4. **Linear regression** (ch. 2, 9–10): on a synthetic 1 000-sample dataset, solve with (a) closed-form normal equations, (b) GD, (c) Adam. Plot all three convergence curves on the same axes. Verify all three solutions agree to $10^{-4}$ tolerance.

5. **Logistic regression** (ch. 4, 9–10): binary classification dataset (simulate or use a real one); derive the cross-entropy gradient $\nabla_w L = X^T(\sigma(Xw) - y) / n$ step by step in your notebook; train with Adam; match sklearn's `LogisticRegression` log-loss to within $10^{-4}$.

6. **One Lagrange-constrained problem** (ch. 11): formulate a concrete optimisation with an equality constraint, write the Lagrangian, derive the stationarity conditions analytically, and solve the resulting linear system numerically. Verify that the constraint is satisfied and the objective matches the Lagrange prediction.

7. **PCA via both routes** (ch. 4–6): on a 500×20 dataset, compute PCA via (a) eigendecomposition of the covariance matrix and (b) SVD of the centred data. Show the first five principal components agree (up to sign flip) and that explained variance ratios match to 8 decimal places.

8. **Numerical gradient check** (ch. 3, 12): for every gradient function you implement, verify $\|g_\text{analytic}(w) - g_\text{fd}(w)\|_\infty < 10^{-5}$ using central finite differences $g_\text{fd,i}(w) = [f(w + h e_i) - f(w - h e_i)] / (2h)$ with $h = 10^{-5}$.

## Acceptance criteria

- All five optimisers converge to $f < 10^{-8}$ on a convex quadratic in under 500 steps.
- Linear regression: closed form, GD, and Adam solutions satisfy $\|w_a - w_b\| < 10^{-4}$ for every pair $(a, b)$.
- Logistic regression: Adam-trained model log-loss matches sklearn's `LogisticRegression` to within $10^{-4}$.
- PCA: eigendecomposition and SVD principal directions agree element-wise to within $10^{-8}$ (after sign alignment).
- Every gradient passes the numerical check with $\|\cdot\|_\infty < 10^{-5}$.
