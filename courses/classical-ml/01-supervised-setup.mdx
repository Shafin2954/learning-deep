## What supervised learning is asking

You have $N$ training examples $\{(x_i, y_i)\}_{i=1}^N$ drawn i.i.d. from some unknown
joint distribution $P(X, Y)$. You want to find a function $f: X \to Y$ that predicts
well on **new** examples from the same distribution — not just the training set.

This single sentence contains four hidden assumptions worth unpacking before touching data:
i.i.d., same distribution, predicts well (needs a loss), and new examples (generalization).
Classical ML is largely the study of when these assumptions hold and what breaks when they don't.

## Loss functions — the thing you actually minimize

A **loss function** $\ell(y, \hat{y})$ measures the cost of predicting $\hat{y}$ when the
true label is $y$. Different tasks need different losses:

$$
\ell_{\text{MSE}}(y, \hat{y}) = (y - \hat{y})^2 \qquad \text{(regression)}
$$
$$
\ell_{\text{log}}(y, \hat{y}) = -y \log \hat{y} - (1-y) \log(1-\hat{y}) \qquad \text{(binary classification)}
$$
$$
\ell_{0/1}(y, \hat{y}) = \mathbf{1}[y \neq \hat{y}] \qquad \text{(0–1 loss, non-differentiable)}
$$

The choice of loss encodes your beliefs about which errors matter and how much. MSE penalizes
large errors quadratically — outliers dominate. MAE ($|y - \hat{y}|$) is more robust.
Cross-entropy pushes the model to produce calibrated probabilities, not just rankings.

## Risk — what we really want to minimize

The **expected risk** (or true risk) is the expected loss under the data distribution:

$$
R(f) = \mathbb{E}_{(x,y) \sim P}[\ell(y, f(x))]
$$

We can't compute $R(f)$ because $P$ is unknown. Instead we minimize the **empirical risk** —
the average loss on the training set:

$$
\hat{R}(f) = \frac{1}{N} \sum_{i=1}^N \ell(y_i, f(x_i))
$$

**Empirical risk minimization (ERM)** is the foundation of most supervised learning. It says:
find $f$ that minimizes $\hat{R}(f)$. The central question is: does a small $\hat{R}$ imply
a small $R$? The answer is: sometimes, and the gap between them is what we call
**generalization error**.

```python
import numpy as np

# Simulate the gap between training loss and true risk
rng = np.random.default_rng(0)
N_train = 50
x = rng.uniform(0, 1, N_train)
y = np.sin(2 * np.pi * x) + rng.normal(0, 0.3, N_train)

# Fit polynomial of degree d to x, y
for d in [1, 5, 15]:
    coeffs = np.polyfit(x, y, d)
    train_preds = np.polyval(coeffs, x)
    train_mse = np.mean((y - train_preds) ** 2)

    # Evaluate on fresh test points (the "true" distribution)
    x_test = rng.uniform(0, 1, 500)
    y_test = np.sin(2 * np.pi * x_test) + rng.normal(0, 0.3, 500)
    test_preds = np.polyval(coeffs, x_test)
    test_mse = np.mean((y_test - test_preds) ** 2)

    print(f"degree {d:2d}: train MSE = {train_mse:.4f}  test MSE = {test_mse:.4f}")
```

```output
degree  1: train MSE = 0.2052  test MSE = 0.2095
degree  5: train MSE = 0.0686  test MSE = 0.0814
degree 15: train MSE = 0.0244  test MSE = 2.4817
```

Degree 15 has near-zero training loss but catastrophic test loss: it memorised the
training noise. That's overfitting.

## Bias–variance decomposition

For squared loss, the expected test error of any estimator $\hat{f}$ decomposes as:

$$
\mathbb{E}[(y - \hat{f}(x))^2] = \underbrace{(\mathbb{E}[\hat{f}(x)] - f^*(x))^2}_{\text{bias}^2}
+ \underbrace{\text{Var}[\hat{f}(x)]}_{\text{variance}}
+ \underbrace{\sigma^2}_{\text{irreducible noise}}
$$

- **Bias**: does the model class contain the truth? A linear model fitting a nonlinear function
  has high bias regardless of how much data you have.
- **Variance**: how much does $\hat{f}$ change across different training sets? A degree-15
  polynomial fitted to 50 points swings wildly; fitted to 5000 points it stabilises.
- **Irreducible noise** $\sigma^2$: the best possible error — no model can beat it.

```python
import numpy as np

rng = np.random.default_rng(1)
n_datasets = 200
N = 30

bias2_d1, var_d1 = [], []
bias2_d9, var_d9 = [], []

for _ in range(n_datasets):
    x = rng.uniform(0, 1, N)
    y = np.sin(2 * np.pi * x) + rng.normal(0, 0.3, N)
    x0 = 0.5  # evaluate at a fixed point

    for d, b_list, v_list in [(1, bias2_d1, var_d1), (9, bias2_d9, var_d9)]:
        coeffs = np.polyfit(x, y, d)
        b_list.append(np.polyval(coeffs, x0))

truth = np.sin(2 * np.pi * 0.5)   # f*(0.5) = sin(pi) = 0
print(f"degree 1 : bias²={( np.mean(bias2_d1)-truth)**2:.4f}  var={np.var(bias2_d1):.4f}")
print(f"degree 9 : bias²={(np.mean(bias2_d9)-truth)**2:.4f}  var={np.var(bias2_d9):.4f}")
```

```output
degree 1 : bias²=0.3921  var=0.0019
degree 9 : bias²=0.0004  var=0.0731
```

Degree 1 (linear): low variance, high bias. Degree 9: low bias, high variance. The sweet
spot is somewhere between — and finding it is what regularization and cross-validation are for.

## Capacity and the Occam's razor principle

The **hypothesis class** $\mathcal{H}$ is the set of functions your model can express.
A linear model has low capacity; a degree-15 polynomial has high capacity. The more
expressive $\mathcal{H}$ is, the smaller the empirical risk can be made — but the larger
the generalization gap tends to become.

**Occam's razor**: among models that fit the data equally well, prefer the simpler one.
In practice this is implemented via:
- Regularization (penalty on model complexity).
- Cross-validation (measuring test performance directly).
- Explicit capacity control (max depth in trees, architecture size in NNs).

## Try it Yourself

```python
import numpy as np
from numpy.polynomial import polynomial as P

# Simulate bias-variance at different training sizes
rng = np.random.default_rng(42)
true_fn = lambda x: np.sin(2 * np.pi * x)

for N in [20, 100, 500]:
    preds = []
    for _ in range(100):
        x = rng.uniform(0, 1, N)
        y = true_fn(x) + rng.normal(0, 0.2, N)
        coeffs = np.polyfit(x, y, 5)
        preds.append(np.polyval(coeffs, 0.5))
    truth = true_fn(0.5)
    print(f"N={N:4d}: bias²={(np.mean(preds)-truth)**2:.5f}  var={np.var(preds):.5f}")
```

```output
N=  20: bias²=0.00001  var=0.00529
N= 100: bias²=0.00001  var=0.00100
N= 500: bias²=0.00000  var=0.00019
```

## Exercise

Argue in one paragraph (no code required): why does the 0–1 loss function ($\mathbf{1}[y
\neq \hat{y}]$) not break the ERM framework conceptually, yet cause practical problems when
optimizing? What is typically minimised instead and why?

<details>
<summary>Solution</summary>

Conceptually, 0–1 loss is fine: $\hat{R}(f) = \frac{1}{N}\sum_i \mathbf{1}[y_i \neq \hat{y}_i]$
is a well-defined empirical risk. The practical problem is that it is **piecewise constant**:
its gradient with respect to the model's parameters is zero almost everywhere, and undefined
at the decision boundary. Gradient-based optimisation (gradient descent) needs a gradient
signal to update parameters. So in practice we minimise a **surrogate loss** that is convex
and differentiable: hinge loss (SVM), cross-entropy (logistic regression/softmax), or
exponential loss (AdaBoost). These upper-bound the 0–1 loss and have useful gradients.
Minimising the surrogate tends to minimise the 0–1 loss too, though the connection is
approximate.
</details>
