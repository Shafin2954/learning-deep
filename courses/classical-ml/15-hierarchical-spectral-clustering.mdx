## Agglomerative clustering builds a tree of merges

k-Means requires you to pick k and assumes spherical clusters. Agglomerative clustering takes a different route: start with every point as its own cluster and repeatedly merge the two closest clusters until one remains. The result is a **dendrogram** — a binary tree where the height of each merge encodes the distance at which two groups joined. You cut the tree at a chosen height to produce k clusters.

The linkage function decides what "distance between clusters" means:

- **Single linkage** — distance between the two nearest points across clusters. Produces chaining: one bridge point pulls distant blobs together.
- **Complete linkage** — distance between the two furthest points. Compact clusters, sensitive to outliers.
- **Average linkage** — mean of all pairwise distances. A middle ground.
- **Ward linkage** — merges that minimize within-cluster variance increase. Most commonly used; gives balanced, compact clusters.

```python
import numpy as np
from sklearn.datasets import make_blobs
from scipy.cluster.hierarchy import linkage, dendrogram, fcluster
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

X, y_true = make_blobs(n_samples=80, centers=4, cluster_std=0.8, random_state=42)

Z = linkage(X, method='ward')
print("Linkage matrix shape:", Z.shape)
print("First 5 merges (cluster_a, cluster_b, distance, count):")
print(np.round(Z[:5], 3))
```

```output
Linkage matrix shape: (79, 4)
First 5 merges (cluster_a, cluster_b, distance, count):
[[ 5.    68.     0.027  2.   ]
 [23.    44.     0.041  2.   ]
 [ 6.    39.     0.053  2.   ]
 [14.    58.     0.061  2.   ]
 [18.    34.     0.072  2.   ]]
```

Each row of `Z` records: left cluster index, right cluster index, merge distance, size of the new cluster. After n−1 merges you have one cluster containing all n points.

### Cutting the dendrogram to get k clusters

```python
labels = fcluster(Z, t=4, criterion='maxclust')
print("Cluster labels:", labels[:20])
from sklearn.metrics import adjusted_rand_score
print("ARI vs ground truth:", round(adjusted_rand_score(y_true, labels), 3))
```

```output
Cluster labels: [3 2 1 4 3 3 1 4 2 4 2 1 3 3 1 2 4 4 2 3]
ARI vs ground truth: 0.981
```

`criterion='maxclust'` cuts the tree so exactly `t` clusters remain. Alternatively, `criterion='distance'` cuts at a height threshold.

The long vertical gap before the final four merges is the natural cut point — it signals that those four groups are well-separated. When the dendrogram shows no obvious gap, the data probably doesn't have a clean cluster structure.

## Time and memory complexity

Ward linkage on n points costs $O(n^2 \log n)$ time and $O(n^2)$ memory for the pairwise distance matrix. For n = 10 000, that's 100 M floats (≈ 800 MB). Agglomerative clustering does not scale to millions of points without approximation.

```python
import time
from sklearn.cluster import AgglomerativeClustering

for n in [500, 2000, 5000]:
    Xn, _ = make_blobs(n_samples=n, centers=4, random_state=0)
    t0 = time.perf_counter()
    AgglomerativeClustering(n_clusters=4, linkage='ward').fit(Xn)
    elapsed = time.perf_counter() - t0
    print(f"n={n:5d}  time={elapsed:.3f}s")
```

```output
n=  500  time=0.008s
n= 2000  time=0.041s
n= 5000  time=0.312s
```

## Spectral clustering works where k-Means fails

k-Means partitions by Euclidean distance from centroids; it cannot handle two interleaved moons or concentric rings. Spectral clustering maps points into a new space where the clusters become linearly separable, then runs k-Means there.

The key insight: build a **similarity graph** where nodes are data points and edges carry a weight $w_{ij} = \exp(-\|x_i - x_j\|^2 / \sigma^2)$. Two points that are "close" (in graph-connectivity terms) get a high weight, even if they're not close in raw Euclidean distance.

### The graph Laplacian

Given the weight matrix $W$ and degree matrix $D$ (diagonal, $D_{ii} = \sum_j w_{ij}$), the unnormalized graph Laplacian is:

$$
L = D - W
$$

The normalized version $L_{\mathrm{sym}} = D^{-1/2} L D^{-1/2}$ is more commonly used. The **eigenvectors corresponding to the k smallest eigenvalues** of $L$ form a k-dimensional embedding. Points that are well-connected in the graph cluster together in this eigenspace.

```python
from sklearn.cluster import KMeans, SpectralClustering
from sklearn.datasets import make_moons
from sklearn.metrics import adjusted_rand_score

X_moons, y_moons = make_moons(n_samples=200, noise=0.08, random_state=42)

km = KMeans(n_clusters=2, random_state=0, n_init=10).fit(X_moons)
sc = SpectralClustering(n_clusters=2, affinity='rbf', gamma=5.0,
                        random_state=0).fit(X_moons)

print("k-Means ARI on moons:", round(adjusted_rand_score(y_moons, km.labels_), 3))
print("Spectral  ARI on moons:", round(adjusted_rand_score(y_moons, sc.labels_), 3))
```

```output
k-Means ARI on moons: 0.492
Spectral  ARI on moons: 1.000
```

k-Means fails because the two moons share the same centroid region. Spectral clustering recovers the true structure because the eigenvector embedding separates connected components.

### Choosing gamma (the RBF bandwidth)

`gamma` controls how rapidly similarity drops with distance. Too small: every point looks similar to every other (one blob). Too large: only nearest neighbors are connected (too many components).

```python
for gamma in [0.5, 2.0, 5.0, 20.0]:
    sc = SpectralClustering(n_clusters=2, affinity='rbf', gamma=gamma,
                            random_state=0).fit(X_moons)
    ari = adjusted_rand_score(y_moons, sc.labels_)
    print(f"gamma={gamma:5.1f}  ARI={ari:.3f}")
```

```output
gamma= 0.5  ARI=0.492
gamma= 2.0  ARI=0.864
gamma= 5.0  ARI=1.000
gamma=20.0  ARI=0.992
```

A useful heuristic: set `gamma = 1 / (2 * median_distance²)` where `median_distance` is computed over a sample of pairwise distances.

### Spectral clustering under the hood (manual)

```python
from scipy.spatial.distance import cdist
from scipy.linalg import eigh

X_small, y_small = make_moons(n_samples=60, noise=0.08, random_state=1)
gamma = 5.0
W = np.exp(-gamma * cdist(X_small, X_small, 'sqeuclidean'))
np.fill_diagonal(W, 0)

D = np.diag(W.sum(axis=1))
L = D - W

# Normalized Laplacian
D_inv_sqrt = np.diag(1.0 / np.sqrt(W.sum(axis=1)))
L_sym = D_inv_sqrt @ L @ D_inv_sqrt

vals, vecs = eigh(L_sym)
embedding = vecs[:, :2]   # 2 smallest eigenvectors (eigenvalue-0 is trivial)

labels_manual = KMeans(n_clusters=2, random_state=0, n_init=10).fit_predict(embedding)
print("Manual spectral ARI:", round(adjusted_rand_score(y_small, labels_manual), 3))
```

```output
Manual spectral ARI: 1.000
```

## Comparing agglomerative and spectral

| Property | Agglomerative (Ward) | Spectral |
|---|---|---|
| Handles non-convex shapes | No | Yes |
| Provides dendrogram | Yes | No |
| Requires k upfront | No (cut later) | Yes |
| Scales to n=100k | No (O(n²)) | No (O(n²) kernel) |
| Main hyperparameter | linkage | gamma / n_neighbors |

For large n, approximate spectral methods (Nyström approximation, sparse k-NN graph) are used.

## Try it Yourself

```python
from sklearn.datasets import make_circles
from sklearn.cluster import KMeans, SpectralClustering
from sklearn.metrics import adjusted_rand_score
import numpy as np

X, y = make_circles(n_samples=300, factor=0.5, noise=0.05, random_state=42)

km_labels = KMeans(n_clusters=2, random_state=0, n_init=10).fit_predict(X)
sc_labels = SpectralClustering(n_clusters=2, affinity='rbf', gamma=10.0,
                               random_state=0).fit_predict(X)

print("k-Means ARI:", round(adjusted_rand_score(y, km_labels), 3))
print("Spectral ARI:", round(adjusted_rand_score(y, sc_labels), 3))
```

```output
k-Means ARI: 0.0
Spectral ARI: 1.0
```

Try varying `gamma` on the circles dataset. Also try `affinity='nearest_neighbors'` with different `n_neighbors` values.

## Exercise

You have a dataset with two interleaved spirals (like `make_moons` but three-armed). Describe the steps to cluster it correctly with spectral clustering, choosing gamma from the data. What linkage would you try first with agglomerative, and why?

<summary>Solution</summary>

```python
from sklearn.datasets import make_moons
from sklearn.cluster import SpectralClustering
from sklearn.metrics import adjusted_rand_score
from scipy.spatial.distance import cdist
import numpy as np

X, y = make_moons(n_samples=400, noise=0.06, random_state=7)

# Estimate gamma from median pairwise distance
sample = X[:100]
dists = cdist(sample, sample, 'sqeuclidean')
median_sq = np.median(dists[dists > 0])
gamma_est = 1.0 / (2 * median_sq)
print(f"Estimated gamma: {gamma_est:.3f}")

sc = SpectralClustering(n_clusters=2, affinity='rbf', gamma=gamma_est,
                        random_state=0).fit(X)
print("ARI:", round(adjusted_rand_score(y, sc.labels_), 3))
```

For the gamma estimate: compute pairwise squared distances on a subsample, take the median nonzero value, set $\gamma = 1/(2 \cdot \text{median})$. This puts the "knee" of the RBF curve near the typical inter-point distance, so nearby points have similarity ~0.6 and distant points approach 0.

For agglomerative: **Ward linkage** is the first choice because spirals, while non-convex globally, have locally compact neighborhoods. Ward won't recover the spiral structure (it's not connectivity-aware), but it produces the most internally homogeneous clusters if you know roughly where to cut.
