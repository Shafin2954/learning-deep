## The project

You will implement two complete ML systems from scratch: a gradient boosting regressor (with a from-scratch decision tree base) that matches LightGBM within tolerance on a tabular dataset, and a suite of unsupervised algorithms (k-Means, GMM via EM, PCA) applied to a real clustering task with a justified choice of k. The goal is to own every algorithm end-to-end — not just call sklearn.

## Hard constraints (no vibe coding)

- **No AI assistance** during implementation. Design decisions and debugging are yours.
- **No sklearn estimators** for the core algorithms: no `DecisionTreeRegressor`, no `GradientBoostingRegressor`, no `KMeans`, no `GaussianMixture`, no `PCA` (from sklearn) for the core logic. You may use sklearn for data loading, metrics, and train/test splitting.
- **No XGBoost / LightGBM / CatBoost** for the gradient boosting implementation (you will use LightGBM only as the reference to compare against).
- **No copypaste** of reference implementations. Read the math, implement from the description.
- You may use NumPy, SciPy (for linear algebra helpers), and matplotlib.

## What it must contain (and the chapter each tests)

- **Decision tree regressor from scratch** (ch. 10): recursive binary splits on MSE impurity; `max_depth` and `min_samples_split` stopping criteria; predict via leaf mean.
- **Gradient boosting from scratch** (ch. 12): fit residuals iteratively; shrinkage (learning rate); subsample rows per tree; early stopping on a validation set; loss = MSE.
- **LightGBM reference** (ch. 13): train on the same split; compare RMSE on the held-out test set. Your implementation must be within 5% of LightGBM's RMSE.
- **k-Means from scratch** (ch. 14): Lloyd's algorithm with random restarts; track inertia per iteration; elbow-method plot to justify k.
- **GMM via EM from scratch** (ch. 14): E-step computes responsibilities $r_{ic} = \pi_c \mathcal{N}(x_i|\mu_c, \Sigma_c) / \sum_k$; M-step updates $\pi$, $\mu$, $\Sigma$; track log-likelihood; BIC to select k.
- **PCA from scratch** (ch. 16): center the data; eigen-decompose the covariance matrix; project; compute explained variance ratio. Do not use `sklearn.decomposition.PCA`.
- **Clustering task** (ch. 14–16): load a real or semi-real dataset; run k-Means, GMM, and PCA (for visualization); compare ARI or silhouette scores; write a short analysis of which algorithm wins and why.

## Acceptance criteria

- **Gradient boosting vs LightGBM**: your test RMSE ≤ 1.05 × LightGBM test RMSE on the same split. (ch. 1–13)
- **k-Means**: inertia decreases monotonically; chosen k has a visible elbow; silhouette score > 0.3 on the chosen k. (ch. 14)
- **GMM EM**: log-likelihood increases monotonically per iteration; BIC selects a k consistent with the data; responsibilities sum to 1 per sample. (ch. 14)
- **PCA from scratch**: explained variance ratios sum to 1; first two PCs match sklearn's output within floating-point tolerance (sign may flip). (ch. 16)
- **Gradient boosting convergence**: plot shows training loss decreasing; early stopping triggers within 20 rounds on the validation set.
- **Written analysis** (one paragraph each): (1) what hurt gradient boosting accuracy most — depth, learning rate, or subsampling? (2) which clustering algorithm gave the best ARI, and what does the structure of the data explain about that?
