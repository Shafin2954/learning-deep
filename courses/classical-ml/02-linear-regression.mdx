## The model and the loss

Linear regression models the output as a linear function of the features:

$$
\hat{y} = x^T w + b = \sum_{j=1}^d x_j w_j + b
$$

Absorb the bias into the weight vector by appending a 1 to each feature vector:
$\tilde{x} = [x_1, \ldots, x_d, 1]^T$, $\tilde{w} = [w_1, \ldots, w_d, b]^T$. Then
$\hat{y} = \tilde{x}^T \tilde{w}$.

For a batch of $N$ examples, write $X \in \mathbb{R}^{N \times (d+1)}$ (rows = examples)
and minimize mean squared error:

$$
L(w) = \frac{1}{N} \|Xw - y\|^2 = \frac{1}{N}(Xw - y)^T(Xw - y)
$$

## The normal equations — the closed-form solution

Setting $\nabla_w L = 0$ gives the normal equations. The gradient is:

$$
\nabla_w L = \frac{2}{N} X^T(Xw - y) = 0
\implies X^T X w = X^T y
\implies \hat{w} = (X^T X)^{-1} X^T y
$$

This is exact and requires one matrix inversion. Cost: $O(d^3 + Nd^2)$. For $N \gg d$,
the $O(Nd^2)$ term dominates — fine up to $d \sim 10^4$, expensive beyond.

```python
import numpy as np

rng = np.random.default_rng(0)
N, d = 200, 3
X_raw = rng.standard_normal((N, d))
w_true = np.array([2.0, -1.0, 0.5])
b_true = 3.0
y = X_raw @ w_true + b_true + rng.normal(0, 0.5, N)

# Augment with bias column
X = np.c_[X_raw, np.ones(N)]   # shape (N, d+1)

# Normal equations
w_hat = np.linalg.solve(X.T @ X, X.T @ y)   # more stable than inv()
print("True   w:", np.append(w_true, b_true).round(3))
print("Fitted w:", w_hat.round(3))

train_mse = np.mean((X @ w_hat - y) ** 2)
print(f"Train MSE: {train_mse:.4f}")
```

```output
True   w: [ 2.  -1.   0.5  3. ]
Fitted w: [ 1.993 -1.015  0.487  2.994]
Train MSE: 0.2498
```

Use `np.linalg.solve(A, b)` instead of `np.linalg.inv(A) @ b` — it is numerically more
stable and roughly twice as fast because it uses LU decomposition.

### When the normal equations fail

$X^TX$ is singular (non-invertible) when:
- Features are linearly dependent (one feature is a linear combination of others).
- $N < d$ (more features than samples — underdetermined system).

The fix: regularization (ch. 3) makes $X^TX + \lambda I$ always invertible.

## Gradient descent — the scalable alternative

When $d$ is large or you want to reuse the same algorithm for other models, gradient
descent updates weights iteratively:

$$
w \leftarrow w - \eta \nabla_w L = w - \frac{2\eta}{N} X^T(Xw - y)
$$

$\eta > 0$ is the **learning rate**. Each step is $O(Nd)$ — no matrix inversion.

```python
import numpy as np

rng = np.random.default_rng(0)
N, d = 200, 3
X_raw = rng.standard_normal((N, d))
w_true = np.array([2.0, -1.0, 0.5])
y = X_raw @ w_true + 3.0 + rng.normal(0, 0.5, N)
X = np.c_[X_raw, np.ones(N)]

# Gradient descent
w = np.zeros(d + 1)
lr = 0.1
losses = []
for step in range(200):
    grad = (2 / N) * X.T @ (X @ w - y)
    w -= lr * grad
    losses.append(np.mean((X @ w - y) ** 2))

print(f"GD final w: {w.round(3)}")
print(f"Final MSE:  {losses[-1]:.4f}")
print(f"Loss drop:  {losses[0]:.4f} → {losses[-1]:.4f}")
```

```output
GD final w: [ 1.993 -1.015  0.487  2.994]
Final MSE:  0.2498
Loss drop:  23.8511 → 0.2498
```

Both methods converge to the same solution. The learning rate matters: too large and the
loss oscillates or diverges; too small and convergence is slow.

### Choosing the learning rate

The optimal step size for GD on MSE is $\eta = 1 / \lambda_{\max}(X^TX/N)$, where
$\lambda_{\max}$ is the largest eigenvalue of the Gram matrix. In practice, try values
like 0.1, 0.01, 0.001 and watch the loss curve.

## The geometric view

The set of all predictions $\{Xw : w \in \mathbb{R}^{d+1}\}$ is the **column space** of $X$.
The normal equations find the $w$ such that $Xw$ is the **orthogonal projection** of $y$ onto
this column space. The residual $y - Xw$ is perpendicular to every column of $X$ — that is
exactly $X^T(Xw - y) = 0$.

```python
import numpy as np

# Verify: residuals are orthogonal to X
rng = np.random.default_rng(0)
X = np.c_[rng.standard_normal((50, 2)), np.ones(50)]
y = X @ np.array([1.0, -0.5, 2.0]) + rng.normal(0, 0.3, 50)
w = np.linalg.solve(X.T @ X, X.T @ y)
residuals = y - X @ w

print("X^T @ residuals (should be ~0):")
print((X.T @ residuals).round(10))
```

```output
X^T @ residuals (should be ~0):
[ 0.  0. -0.]
```

## Try it Yourself

```python
import numpy as np

# Compare GD with different learning rates
rng = np.random.default_rng(5)
N, d = 100, 2
X = np.c_[rng.standard_normal((N, d)), np.ones(N)]
y = X @ np.array([1.5, -0.8, 2.0]) + rng.normal(0, 0.4, N)

for lr in [0.5, 0.1, 0.01]:
    w = np.zeros(d + 1)
    for _ in range(300):
        w -= lr * (2 / N) * (X.T @ (X @ w - y))
    mse = np.mean((X @ w - y) ** 2)
    print(f"lr={lr}: final MSE={mse:.4f}  w={w.round(3)}")
```

```output
lr=0.5: final MSE=0.1563  w=[ 1.477 -0.802  2.022]
lr=0.1: final MSE=0.1563  w=[ 1.477 -0.802  2.022]
lr=0.01: final MSE=0.1564  w=[ 1.477 -0.801  2.021]
```

## Exercise

Implement **stochastic gradient descent (SGD)** for linear regression: at each step, pick
one random example $(x_i, y_i)$ and update $w \leftarrow w - \eta \cdot 2(x_i^T w - y_i) x_i$.
Run 500 passes over 100 data points and compare the final MSE with the batch GD above.

<details>
<summary>Solution</summary>

```python
import numpy as np

rng = np.random.default_rng(5)
N, d = 100, 2
X = np.c_[rng.standard_normal((N, d)), np.ones(N)]
y = X @ np.array([1.5, -0.8, 2.0]) + rng.normal(0, 0.4, N)

w = np.zeros(d + 1)
lr = 0.01
for epoch in range(500):
    idx = rng.integers(0, N)          # one random example
    xi, yi = X[idx], y[idx]
    w -= lr * 2 * (xi @ w - yi) * xi

mse = np.mean((X @ w - y) ** 2)
print(f"SGD final MSE: {mse:.4f}  w: {w.round(3)}")
```

SGD converges to the same neighbourhood as batch GD but with noise; the loss oscillates
around the minimum rather than descending smoothly. Mini-batch GD (batch size 16–256) is
the practical middle ground.
</details>
