## Maximum-margin classification

Logistic regression picks any separating hyperplane. SVMs find the one with **maximum margin** — the gap between the hyperplane and the nearest training points on either side.

Formally, for a linear classifier $f(x) = w^Tx + b$, the **geometric margin** for a correctly classified point is $y_i(w^Tx_i + b)/\|w\|$. The SVM maximises the smallest such margin.

### Hard-margin SVM (linearly separable data)

$$
\min_{w,b}\; \frac{1}{2}\|w\|^2 \quad \text{s.t.}\; y_i(w^Tx_i + b) \ge 1 \;\forall i
$$

The margin is $2/\|w\|$. The constraint $y_i(w^Tx_i+b)\ge1$ requires all points to be on the correct side, at least one margin-width away.

The points that lie exactly on the margin boundary ($y_i(w^Tx_i+b)=1$) are the **support vectors** — the only training points that determine $w$.

## Soft-margin SVM (non-separable data)

Allow some points to violate the margin using slack variables $\xi_i \ge 0$:

$$
\min_{w,b,\xi}\; \frac{1}{2}\|w\|^2 + C\sum_i\xi_i \quad \text{s.t.}\; y_i(w^Tx_i + b) \ge 1 - \xi_i,\; \xi_i \ge 0
$$

$C$ controls the trade-off: large $C$ → small margin, few violations; small $C$ → large margin, more violations tolerated (more regularisation).

```python
import numpy as np
from sklearn.svm import SVC
from sklearn.datasets import make_classification
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

rng = np.random.default_rng(0)
X, y = make_classification(n_samples=300, n_features=2, n_redundant=0,
                             n_clusters_per_class=1, random_state=3)
y = 2*y - 1  # convert to {-1, +1}
X = StandardScaler().fit_transform(X)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=0)

for C in [0.01, 0.1, 1, 10, 100]:
    svm = SVC(kernel='linear', C=C).fit(X_tr, y_tr)
    n_sv = svm.n_support_.sum()
    acc  = svm.score(X_te, y_te)
    print(f"C={C:5}: acc={acc:.3f}, support vectors={n_sv}")
```

```output
C= 0.01: acc=0.867, support vectors=80
C=  0.1: acc=0.900, support vectors=44
C=    1: acc=0.917, support vectors=24
C=   10: acc=0.917, support vectors=20
C=  100: acc=0.900, support vectors=19
```

Fewer support vectors at high C: the model relies on fewer points, more prone to overfitting.

## The kernel trick

SVMs only need inner products $\langle x_i, x_j\rangle$ in the dual problem. Replace with a **kernel function**:

$$
K(x_i, x_j) = \langle\phi(x_i), \phi(x_j)\rangle
$$

This implicitly maps inputs to a feature space $\phi$ without ever computing $\phi(x)$ explicitly. If the feature space is infinite-dimensional, this is only tractable via the kernel trick.

### RBF (Gaussian) kernel

$$
K(x, z) = \exp\!\left(-\gamma\|x - z\|^2\right)
$$

The RBF kernel corresponds to an infinite-dimensional feature space. It measures similarity by Euclidean distance: nearby points get high similarity, distant points get near-zero.

```python
import numpy as np
from sklearn.svm import SVC
from sklearn.datasets import make_circles
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

# Concentric circles — NOT linearly separable
X, y = make_circles(n_samples=300, noise=0.1, factor=0.4, random_state=0)
X = StandardScaler().fit_transform(X)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=0)

lin_svm = SVC(kernel='linear', C=1.0).fit(X_tr, y_tr)
rbf_svm = SVC(kernel='rbf',    C=1.0, gamma='scale').fit(X_tr, y_tr)

print(f"Linear SVM on circles: {lin_svm.score(X_te, y_te):.3f}")
print(f"RBF SVM on circles:    {rbf_svm.score(X_te, y_te):.3f}")
```

```output
Linear SVM on circles: 0.467
RBF SVM on circles:    1.000
```

The linear SVM can't separate concentric circles — the boundary must be curved. RBF maps to a space where the circles become linearly separable.

### Effect of gamma

```python
from sklearn.svm import SVC
from sklearn.model_selection import cross_val_score

for gamma in [0.01, 0.1, 1, 10, 100]:
    cv = cross_val_score(SVC(kernel='rbf', C=1, gamma=gamma),
                          X_tr, y_tr, cv=5).mean()
    print(f"gamma={gamma:4}: CV acc = {cv:.3f}")
```

```output
gamma=0.01: CV acc = 0.896
gamma= 0.1: CV acc = 0.983
gamma=   1: CV acc = 1.000
gamma=  10: CV acc = 0.983
gamma= 100: CV acc = 0.571
```

Large $\gamma$: only very nearby points have non-zero similarity → model memorises training data → overfits. Small $\gamma$: very smooth decision boundary → may underfit.

## Hyperparameter tuning: C and gamma

```python
from sklearn.model_selection import GridSearchCV

param_grid = {'C': [0.1, 1, 10, 100], 'gamma': [0.01, 0.1, 1, 10]}
gs = GridSearchCV(SVC(kernel='rbf'), param_grid, cv=5, n_jobs=-1)
gs.fit(X_tr, y_tr)
print(f"Best params: {gs.best_params_}")
print(f"Best CV score: {gs.best_score_:.3f}")
print(f"Test accuracy: {gs.score(X_te, y_te):.3f}")
```

```output
Best params: {'C': 1, 'gamma': 1}
Best CV score: 1.000
Test accuracy: 1.000
```

## Try it Yourself

```python
import numpy as np
from sklearn.svm import SVC
from sklearn.datasets import make_moons
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split, cross_val_score

X, y = make_moons(n_samples=400, noise=0.2, random_state=0)
X = StandardScaler().fit_transform(X)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=0)

for kernel in ['linear', 'poly', 'rbf']:
    cv = cross_val_score(SVC(kernel=kernel, degree=3), X_tr, y_tr, cv=5).mean()
    test_acc = SVC(kernel=kernel, degree=3).fit(X_tr, y_tr).score(X_te, y_te)
    print(f"{kernel:6}: CV={cv:.3f}, test={test_acc:.3f}")
```

```output
linear: CV=0.873, test=0.888
  poly: CV=0.944, test=0.963
   rbf: CV=0.972, test=0.975
```

## Exercise

Generate a 2D XOR problem (four quadrants, alternating labels). Show that a linear SVM and logistic regression both fail (near 50% accuracy). Then show that an RBF SVM achieves >95% accuracy. Explain geometrically why RBF works.

<details>
<summary>Solution</summary>

```python
import numpy as np
from sklearn.svm import SVC
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler

rng = np.random.default_rng(0)
n = 400
X = rng.uniform(-1, 1, (n, 2))
y = ((X[:, 0] > 0) ^ (X[:, 1] > 0)).astype(int)   # XOR

X_tr, X_te = X[:300], X[300:]
y_tr, y_te = y[:300], y[300:]

for name, clf in [
    ("Logistic", LogisticRegression()),
    ("Linear SVM", SVC(kernel='linear')),
    ("RBF SVM", SVC(kernel='rbf', gamma=2)),
]:
    acc = clf.fit(X_tr, y_tr).score(X_te, y_te)
    print(f"{name}: {acc:.3f}")
# RBF SVM works because in the feature space defined by the RBF kernel,
# the four XOR quadrants become linearly separable.
```
